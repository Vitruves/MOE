Logging and History Functions

Syntax

    logfile 'filename'
    logfile []
    logfile 0
    logfile 1

    cmd_list = history []

Description

logfile 'filename'
logfile []
logfile 0
logfile 1

Records the SVL outputs of the current session. The first command to be recorded is the one immediately following the call to logfile. Everything echoed to the output area, both commands and SVL return values, are written to the file filename. Recording is terminated and the current file closed by calling logfile again. If this subsequent call to logfile is passed a null argument, logging is terminated, otherwise a new log file is opened, and recording continues in the newly opened file. Logging may be temporarily suspended by calling logfile 0, and resumed with logfile 1.

Note that if the file filename already exists, it will be overwritten. You must close the current file with a call to logfile before you can examine the file's contents.

cmd_list = history []

Returns a list of the SVL commands issued at the CLI. If invoked as a procedure (i.e. its return value is not stored or used in an expression), the list of commands is pretty-printed.
