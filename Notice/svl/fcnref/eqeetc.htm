Element-wise Comparisons: eqE, neE, ltE, ...

Syntax

val = v op w               // infix notation
val = funcE [v,w]
val = funcE [v1,v2,v3,...] // reduction

Description

Element-wise comparison functions. Arguments must have the same shape (taking into account unit extension) and be composed of the same data types. Each pair of corresponding leaves is replaced by a binary result of the comparison: 1 on success, 0 on failure. Tokens are compared lexicographically by their contents.

Relational functions can accept more than two arguments, in which case the operation performed is equivalent to:

func [ v1, func [ v2,v3, ... ] ]
Function Description
eqE Equal to Element.
Shorthand notation: == 
svl> "cat" == "hat"
[0,1,1]
svl> [ 0.3, 'cart', "cat" ] == [ 0.3, 'cat', "hat" ]
[ 1, 0, [0,1,1] ]
svl> eqE ['a', 'ab']
0
geE Greater than or equal to Element.
Shorthand notation: >= 
svl> [1,3,5,7,11] >= 7 // unit extension
[0,0,0,1,1]
gtE Greater than Element.
Shorthand notation: > 
svl> 2 > 3 > 1  // error!
svl> (2 > 3) > 1
0
svl> gtE [2,3,1]      // reduction (2>3)>1
0
leE Less than or equal to Element.
Shorthand notation: <= 
svl> [1,3,5,7,11] <= 7 // unit extension
[1,1,1,1,0]
ltE Less than Element.
Shorthand notation: < 
svl> ['abc', 'aa'] < ['afa', 'aaa']
[1,1]
svl> "abc" < "afa"
[0,1,0]
svl> "a" < "b"
1
neE Not equal to Element.
Shorthand notation: <>
