Modal Dialog Functions: Warning...

Syntax

    Warning 'message'
    OKCancel 'message'
    answer = NoYes 'question'
    answer = YesNo 'question'
    answer = NoYesCancel 'question'
    answer = YesNoCancel 'question'

    Warning key
    OKCancel key
    answer = NoYes key
    answer = YesNo key
    answer = NoYesCancel key
    answer = YesNoCancel key

Description

The following functions raise a prompt window displaying a message or question token. Alternatively, a graphics key can be specified to show a graphical message (see gr_create for how to create a 2D graphics object).

The prompt is application modal which means that the entire application is suspended until the user responds by clicking on one of the prompt command buttons or by pressing Return or Esc while in one of the application windows.

Note: To display a message with a single OK button that terminates the task, use the exit function.

Warning 'message'
Warning key

Provides a single OK button. When OK is pressed, the function returns (the function which invoked it resumes).

OKCancel 'message'
OKCancel key

Provides OK and Cancel buttons. Pressing OK or Return returns null. Pressing Cancel or Esc kills the task which initiated it.

answer = NoYes 'question'
answer = NoYes key

answer = YesNo 'question'
answer = YesNo key

YesNo provides Yes and No buttons, with Yes as the default. Pressing Yes or Return returns 1; pressing No returns 0.

NoYes is the converse of YesNo, where No is now the default button. In this case, pressing No or Return returns 0; pressing Yes returns 1.

In either case, pressing Esc returns 0 and task is also killed.

answer = YesNoCancel 'question'
answer = YesNoCancel key

answer = NoYesCancel 'question'
answer = NoYesCancel key

YesNoCancel provides Yes, No, and Cancel buttons, with Yes as the default. Pressing Yes or Return returns 1; pressing No returns 0.

NoYesCancel is the complement of YesNoCancel, where No is now the default button. In this case, pressing No or Return returns 0; pressing Yes returns 1.

In either case, pressing Esc or Cancel returns null ([]), in which case the task from which the modal question was initiated is also killed.
