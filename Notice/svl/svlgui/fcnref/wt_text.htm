Text Widget

Description

Text entry field for entering numeric or token data. The allowable maximum is 1024 characters. (Use the Edit widget if more characters are required.) The field is scrollable to allow editing. Range checking is performed on numeric data when min and max values are specified and onTrigger (see core attributes) is set to 'validate'.

A shortcut list can be attached to the text widget. This is supplied as a vector of tokens. If the shortcut attribute contains at least one token, then a down arrow is displayed allowing the user to select an item from a pulldown menu. Once an item is selected, the current text is replaced with the selected text and the widget is triggered.

The text widget has the ability to display text when the field is empty. This text disappears when characters are entered in the text field, and reappears when the field is cleared. By default, the text to display is the null token so that the field appears blank.

Attributes

Text inherits all core attributes and defines the following attributes:

Trigger

Return pressed within the text entry area.

If a shortcut list is present and an item is selected from the list then the text is replaced with the selected text and the widget triggers.

Data Value

The entered data (token or number) depending on type.

Notes

The text widget redefines the default font to be 'fixedMedium'. The font in a text widget must be a fixed font. Thus, if a proportional font is used, it will be overwritten by a similarly sized fixed font.
