The Right Hand Side (RHS) Button Bar is a collection of procedures and capabilities in MOE for conducting interactive ligand modification and energy minimization in the active site of a flexible receptor. It provides for:

  1. Assigning coordinate restraints on atoms to discourage movement.
  2. Drawing molecular surfaces to highlight steric clashes.
  3. Interactive modification of ligands and geometry optimization.
  4. Estimating binding affinities and ligand properties.
  5. Control of multiple receptors and ligands.

Interactive structure-based drug design (SBDD) is generally conducted using the MOE Window, in particular the MOE Button Bar:

A typical interactive modeling session will involve loading molecular data into MOE, isolating a ligand in an active site, drawing molecular surfaces and repeatedly making modifications to the ligand and optimizing the geometry of the modifications using energy minimization.

The molecular data will typically have atomic coordinate restraints applied to ensure that the receptor or ligand scaffold atoms do not move too much. These restraints are added as part of the QuickPrep preparation procedure. MOE files are capable of saving restraint information and molecular data from a previously prepared MOE session, and so may be used without further preparation.

Ligand properties (such as molecular weight, logP and TPSA) may be displayed and Generalized Born binding affinity estimates can be calculated as part of the energy minimization procedure. Active site analysis applications like Molecular Surfaces, Contact Statistics and Electrostatic Maps may be applied and 2D protein-ligand interaction diagrams can be calculated.

Data Preparation

MOE has been designed for working with multiple macromolecular receptor and ligand molecules, which must first be loaded into MOE. Typically, receptor-ligand complexes are initially read from PDB format files, prepared for use with the application QuickPrep and stored in a MOE file (to save the preparation results). QuickPrep preparation data consists of restraint tethers on atoms, ligand identification annotations, hydrogen atoms and refined coordinates. Receptors and ligands can be read from different files (e.g. a receptor from a PDB file and a ligand from a 3D MOL file). It is worthwhile to expend some effort in preparing a receptor-ligand complex and saving the results to a MOE file for subsequent use; in this way, a good initial starting point can be used repeatedly without having to prepare the structure each time.

Open, Close and Save Files

The MOE | RHS | Open button raises the Open panel. Select a file in the list (or change directories prior to selecting a file) and press OK. Typically MOE, PDB, Tripos MOL2 or MDL MOL files are relevant for structure based design. Recent directories are available from the pull-down in the file name text field.

Note! The MOE Button Bar is customizable and in some cases the Open button may raise a custom panel different from the default.

The MOE | RHS | Close button will delete all data in MOE (after a confirmation prompt). Use MOE | RHS | System to delete a subset of the system (including related surfaces).

Warning! Once cleared, molecular data cannot be recovered (unless it was previously saved to disk).

Use MOE | File | Save to save the contents of the MOE Window to disk. The data can be saved in one of a number of file formats; however, if the data is not saved to a MOE file then QuickPrep annotations and preparations such as atom position tethers and ligand identifications may be lost. See Saving the System for more information.

QuickPrep – Prepare Structures for SBDD

Before performing calculations on receptor-ligand complexes in an interactive structure-based design session, the molecular data must be prepared. If a structure is already prepared then this preparation procedure can be skipped. A prepared structure will have all of its hydrogens present; use MOE | RHS | Hydrogens to check for the presence of explicit hydrogens. Unprepared files, such as PDB files, may contain missing atoms, alternate geometry or other crystallographic artifacts. See Preparing Protein and Protein Complex Data for more details on preparing PDB files and correcting problems (e.g., with ligand topology). With reasonably good quality starting data, the following procedure will be adequate for structure-based design:

  1. Verify Ligand. Often the ligand topology in PDB files is not correct (due to coordinate errors and the lack of such information in PDB files). Use MOE | RHS | Ligand | 2D Molecules to calculate 2D depictions of the ligands in MOE. Use SiteView to isolate the ligands (stick) and their receptor environments (line) in the MOE Window (Make sure that there are no atoms selected before pressing SiteView.) If bond orders, charges or connectivity of a ligand of interest are not correct then the Builder must be used to make corrections.
  2. Choose Ligand. QuickPrep can handle multiple ligands; however, many receptor-ligand complex files contain molecules that are not of interest or are copies of a ligand of interest. In such cases, the SiteView command will isolate and display all such molecules. Use MOE | RHS | Ligand | Choose Ligand to manually select the ligand of interest. To do this, clear the atom selection and select the atoms of the ligand of interest. (Ctrl+click will select all of the atoms of a ligand provided that it is contained in a single residue.) Alternatively, the System Manager can be used to select subsets of the atoms. Then, issue the Choose Ligand command.

    Note! Choose Ligand is implemented by setting all other ligands to be considered as Receptor atoms. They will still be part of the system but will be considered as non-ligands by MOE. If multiple complexes are present, then "other ligands" means other ligands in the same tag grouping as defined in the System Manager.

  3. QuickPrep. Use MOE | RHS | QuickPrep to raise the QuickPrep preparation application. QuickPrep will delete distant solvent, add hydrogens, install tethers, calculate charges and perform initial refinement of the system (see below). QuickPrep can handle multiple independent complexes; however, it is usually best to prepare each complex separately and save them to separate files.

QuickPrep – Prepare Structures for Biologics

Preparing initial protein data is a critical step in a biologics workflow. If the structure is already prepared, this preparation procedure can be skipped.

A prepared structure will have completely populated heavy atoms in the regions of interest, and all of its hydrogens present. Unprepared files, such as PDB files, may contain missing atoms, alternate geometry or other crystallographic artifacts. Running QuickPrep assumes reasonably good quality starting data. For structures with more serious problems, or for in-depth analysis, see Preparing Protein and Protein Complex Data.

For macromolecular studies, QuickPrep is used to cap termini for large loops and endgaps, or to build small loops. It uses Protonate3D for adding and placing hydrogens, and can assign tethers and perform structure refinement.

Running QuickPrep

The purpose of MOE | RHS | QuickPrep is to prepare one or more systems (defined by their tag in the System Manager) for structure-based drug design involving energy minimization or biologics studies involving refinement. The following panel will be raised:

Prepare If Use Structure Preparation is enabled, the Structure Preparation application is run with default settings. In this case, when a structure is missing atoms, the Structure Preparation build loop protocol is used to build small loops — breaks of up to 10 residues and terminal outgaps of up to 5 residues. For larger gaps, there are two possible treatments; these options only have effect if Use Structure Preparation is enabled:
  • Preserve Sequence and Neutralize. The endpoints adjoining empty residues are neutralized, and the empty residues are preserved.
  • Cap Breaks and Delete. Capping groups are added to endpoints adjoining empty residues, and the empty residues are deleted.
Most macromolecular structure data do not contain hydrogen atoms. Additionally, the ionization state of protein residues is often in question. If Use Protonate3D for Protonation is enabled, the Protonate3D [Labute 2009] methodology is used to determine ionization state and add hydrogens. Otherwise, hydrogen atoms are added in extended conformations according to the current atomic ionization states. If Allow ASN/GLN/HIS "Flips" in Protonate3D is also enabled, then terminal amides, sulfonamide and imidazole groups will be permitted to flip to optimize the hydrogen bond network.
Delete Often, receptor-ligand complexes contain many water molecules. Some of these water molecules may be important to maintain receptor structure or to mediate ligand interactions. The remaining water molecules are generally uninteresting and may be deleted. If this option is enabled, water molecules farther than 4.5 Å from any receptor or ligand atom will be deleted.
Tether Typically, receptor atoms are constrained during geometry optimization so that they do not deviate too much from their initial (experimental) coordinates. QuickPrep will add tether restraints to receptor, ligand, and solvent atoms of the system according to which of the Receptor, Ligand, and Solvent checkboxes have been enabled. If a checkbox is off, no corresponding tethers will be created. If the corresponding Hydrogens checkbox to the right is enabled, then hydrogen atoms will also be tethered, otherwise the hydrogen atoms will not be tethered.

The Strength and Buffer parameters control the strength and shape of the tether restraint on an atom. The greater the Strength, the less the atom will deviate from its initial coordinates. The Buffer value specifies the extent of the flat bottom of the flat-bottom restraint, described in Atom Restraints and Tethers.

Note that the tethers remain in the system after QuickPrep has been run. They can be viewed in the Potential Setup panel Restraints page, and can be cleared using MOE | RHS | Constrain | Free.
Fix Even if tethers are applied to a system, generally atoms far away from the binding site can be held fixed (for efficiency). If enabled, atoms farther than the specified distance will be constrained to be fixed (whether or not they have been assigned tethers). A value of 0 will cause all non-ligand atoms to be fixed.
The pulldown option can be used to specify how the binding site is defined. By default, all Ligands will be used. Select Dummy Atoms for example to use the results of Site Finder, or Selected to use a custom definition. In all cases, the site definition will be split by Tag and restricted to active atoms. Dummy atoms, if present, are always excluded from refinement. Use the ? button to the right to verify the selection – the atoms will briefly "flash" in the MOE Window.
An exception can be made for hydrogens near the binding site: if Hydrogens Close to Binding Site will not be Fixed is enabled, then receptor hydrogen atoms within 4.5 Å of the binding site will be free to move.

Note! All atoms matching the binding-site selection (e.g. if Ligands is selected and there is more than one ligand) will be treated equally for the purposes of calculating the receptor atoms "far away". Use Choose Ligand to focus on a particular ligand of interest (and treat the remaining ligands as the receptor).
Refine If enabled, the results of the previous calculations will be subjected to an energy minimization using the current Potential Setup, including the tethers specified in the panel. Energy minimization will be terminated when the root mean square gradient falls below the specified value. If Retain QuickPrep Minimization Restraints is on, the restraints used for energy minimization will remain in place after the calculation has completed. Otherwise, any restraints that were originally in effect before QuickPrep calculations were applied are restored. If Overwrite Temperature Factor with Displacement is on, the temperature factor value of each atom will be overwritten by the distance it was moved during Quickprep, multiplied be a factor of 30.0.
Save Settings
Standard Settings		 Before pressing the OK button, the current settings of the QuickPrep panel can be saved for use as default values on future runs of QuickPrep. Press Save as Default to store the values. Press Restore Defaults to restore the values to the previously saved values. Press Standard Settings to restore the values to "factory defaults".

Note! The values will be retained even if Cancel is pressed. In this way, the QuickPrep button can be used as a kind of configuration panel. The tether values and RMS gradient test are used by Minimize and the Constrain menu commands.

Press OK to begin the calculation. If there are multiple tag groups in MOE, each will be prepared separately from the others. Once the calculation completes, use MOE | File | Save to save the results.

In summary, a prepared molecular system will have one or more identified ligands with correct topology, reasonable ionization states and proton coordinates, tether restraints on the heavy atoms of the receptor and no unimportant water molecules. This prepared state configuration is assumed for optimal display by the SiteView rendering command and energy Minimize procedures of QuickPrep as well as other applications in MOE.

Visualization

The MOE Button Bar provides a number of commands for visualizing the ligand and receptor active site. The visualization commands are designed to analyze active sites, protein-ligand interactions and to provide information about which atoms have tether restraints (normally added in the QuickPrep preparation steps). For this section, we will assume that a prepared receptor-ligand complex is in the MOE Window. Also, a good working knowledge of the operation of the MOE Window is assumed. The angle of view and zoom of the MOE Window is controlled with the mouse (see Rotating, Translating and Zooming Atoms). For more information on selecting atoms, see Selecting Atoms Using the Mouse.

The System Manager – Control Multiple Complexes

One of the most important buttons on the MOE Button Bar is the System button: it raises the System Manager which is used for rapid control of:

  • Visibility of atoms and surfaces.
  • Active, Inert and Selected state of atoms (independent of visibility).
  • Coloring and rendering mode of atoms.
  • Names of chains and chain groupings.

For detailed information, please refer to the documentation on the System Manager.

The SiteView Button

The MOE | RHS | SiteView is a kind of "reset" or "home base" for structure-based drug design. SiteView renders the molecular system according to a convention which is designed to provide feedback regarding which atoms belong to the receptor and which atoms are ligand atoms as well as to focus on the active sites of receptor-ligand complexes. First ensure that no atoms selected. Pressing SiteView will

  1. Isolate the ligand in the active site by hiding distant receptor and solvent atoms. Distant backbone ribbons will be hidden.
  2. Render the receptor atoms in line mode and ligands in stick mode.
  3. Hide non-polar hydrogens and show polar hydrogens.

Use the System Manager to color the receptor carbons differently from the ligand (e.g., dark green). Note! In previous versions of MOE, this color was integrated with the Active Site button. This coloring is not performed by SiteView in order to support multiple receptor-ligand complexes.

SiteView will affect all atoms in the system and take all atoms into account. The active site (within a System tag group) is defined by default to be those residues with atoms (Receptor or Solvent) within 4.5 A of any ligand. Note! This cutoff value is defined in the SiteView Cutoff text field of the QuickPrep page of the Configuration Options panel.

When a single ligand is to be modified with the small molecule Builder, explicit display of the ligand hydrogens is required for substitution. Select the ligand with Ctrl+Left (to select the residue) or by using the System Manager and then press the MOE | RHS | Hydrogens to cycle through the hydrogen display until all hydrogens are visible. Note! If there are no explicit hydrogens then the system should be prepared with QuickPrep button.

Warning! If there are selected atoms in the system, then these, and only these, atoms will be considered "ligands".

Surfaces: Receptor – Ligand – Interaction (VDW)



 The MOE | RHS | Surface menu provides commands to create surfaces around receptors and ligands in the system. These "Quick" surfaces are those most useful in structure-based drug design and are created with coloring designed to show the color of atoms in the system:
  • Receptor. Creates a Connolly-type molecular surface using receptor atoms only near any ligand atom in the system. The surface is colored by nearest atom with hydrogen bond information (direction of polar hydrogens and lone pairs) overlaid.
  • Ligand. Creates a Connolly-type molecular surface using ligand atoms only. The surface is colored by nearest atom with hydrogen bond information (direction of polar hydrogens and lone pairs) overlaid.
  • Interaction (VDW). Creates an accessible surface (line mode) using receptor atoms only near any ligand atom in the system. This surface shows the onset of van der Waals clash (a 0-potential energy contour of the van der Waals interaction energy with a water molecule probe). The color of the surface is the same as that shown in the System Manager.
  • Protein. Creates a Connolly-type molecular surface using all of the receptor atoms. The surface is colored by nearest atom with hydrogen bond information (direction of polar hydrogens and lone pairs) overlaid.
  • Protein Patches. Creates both electrostatic (excess charge) and hydrophobic surface patches to show regions of significance with respect to protein-protein interactions.

More sophisticated surfaces such as Electrostatic Maps and Contact Statistics can be created with the Surfaces and Maps application, which provides more coloring options and transparency settings. Surfaces and Maps can also be used to change the coloring, transparency and material properties of the surfaces created in the Surface menu. This application can be started with the Surfaces and Maps command in the Surface menu.

Note! Inactive receptors and ligands are ignored and no surface will be created. A receptor or ligand is considered inactive if it has no active atoms (as shown in the System Manager).

Note! Each active receptor or ligand is treated separately and a separate surface will be created for each one.

Note! The surfaces do not automatically update if the atomic coordinates change. Issue the command again to update the surface (which will delete the old surface).

To delete Quick surfaces:

  • Use the Clear command in the Surface menu.
  • Use Surfaces and Maps.
  • Use the X button in the System Manager.

Measure Interatomic Distances, Angles and Dihedrals

The MOE Footer can be used to measure distances, angles and dihedrals. When 2, 3 or 4 atoms are selected, the MOE Footer will adapt accordingly and display distance, angle and dihedral measurements (and means to change them). See MOE Footer for more information.

The MOE | RHS | Measure menu and the equivalent MOE | Edit | Measure menu provides commands to measurement meters that are displayed in the MOE Window and continuously updated as atomic coordinates change. Choose one of

  1. Distance to measure and display the distance between 2 atoms.
  2. Angle to measure and display the angle between 3 atoms.
  3. Dihedral to measure and display the dihedral angle between 4 atoms.

The SVL Commands line in the MOE Window will change into a prompt (see figure) providing for selection of the measurement type (distance, angle or dihedral), then click on either 2, 3, or 4 atoms as required. If you make a mistake in the process, click on the selected atom to deselect it. Alternatively, the current selection can be used to specify a meter. Once created, the meter appears in the MOE Window as a colored line with a measured value. Clicking on the X closes the Measure menu and restores the MOE menu bar.

The last meter created can be removed by pressing Remove Last and previously created measurement meters can be removed via the Remove items in the Measure menu. Select the atoms for which meters are to be removed (or the meter lines themselves) then select one of Remove Distances, Remove Angles, Remove Dihedrals or Remove All from the menu. Warning! If no atoms are selected, then all measurement meters of the selected type will be removed from the MOE Window. Meter removals and additions cannot be undone.

Ligand Properties

The MOE | RHS | Ligand | Ligand Properties command toggles the display of calculated ligand properties. Issue the command once to enable the property display and again to disable it. Properties will be automatically calculated if there is a single active ligand in the system. If there is more than one (or none at all) a message will be displayed and the properties will not be calculated. Use MOE | RHS | System to activate or de-activate ligands so that there is only one active ligand.

The calculated 2D ligand properties are displayed in the top left corner of the MOE Window. Use MOE | Render | Setup to configure the color of the properties text (if, for example, they cannot be seen on the current background). See Visualization Setup. The font properties (size, bold, etc.) can be controlled in MOE | Render | Draw.

The properties divide into two categories: those that depend only on the ligand connectivity and topological state (2D) and those that depend on coordinates and interactions with the receptor (3D). The 2D properties are continuously updated as the ligand is modified with the Builder; the 3D properties are updated after each energy minimization using the MOE Button Bar or the Builder.

The calculated 2D properties are MOE Descriptors relevant to structure-based drug design:

  • toxic – An assessment of the toxicity or mutagenicity of the ligand using a rule-based method [Kazius 2005].
  • weight – The molecular weight calculated with values from the CRC Handbook [CRC 1994].
  • TPSA – The topological polar surface area [Ertl 2000]. Hydrogen bond donor and acceptor counts are to the right.
  • Hueckel Theory Descriptors [Gerber 1998, Wang 2007]
    • logP – The log octanol/water partition coefficient.
    • logD – The log distribution coefficient at pH=7.
    • logS – The log solubility coefficient.

Additional 3D properties are calculated after each Minimize. If the ligand is subsequently modified then these additional properties will no longer be displayed and another energy minimization must be performed so that they can be recalculated. The energy minimization will optimize the potential energy of the receptor-ligand complex. From the resulting coordinates, several properties are calculated:

  • strain – The local strain energy of the ligand conformation. The ligand is subjected to energy minimization, isolated from the rest of the molecular system. The strain energy is the difference between the ligand conformation energy in the bound state and the energy of the isolated local minimum. The value in parentheses to the right is the strain energy divided by the number of heavy atoms in the ligand (the strain efficiency). The rmsd at the far right is the superposition distance between the bound conformation and the isolated local minimum conformation.
  • solvation – The molecular mechanics generalized Born interaction energy is the non-bonded interaction energy between the receptor and the ligand and comprises van der Waals, Coulomb and generalized Born implicit solvent interaction energies [Labute 2008]. Ligand and receptor strain energies are not taken into account. The parenthetical value to the right is the interaction energy divided by the number of heavy atoms in the ligand (interaction efficiency). Solvent molecules are ignored in the calculation.
  • affinity – The estimated binding affinity is that of the GBVI/WSA dG scoring function, reported in units of kcal/mol. The parenthetical value to the right is the affinity divided by the number of heavy atoms in the ligand (the efficiency). Solvent molecules are ignored in the calculation.

Note! It is possible to customize the Ligand Properties display by using custom SVL functions. In such a case, the number and type of the properties may be different from the description above.

Ligand R-Vectors

Modifications to ligands such as the addition of functionals may result in poor steric interactions with the receptor. The purpose of the R-Vector display is to provide information about potential steric clash at substitution locations on a ligand.

The MOE | RHS | Ligand | Ligand R-Vectors command toggles the display of "exit vectors" on active ligand molecules. These graphical arrows are drawn along visible Q–H bonds when a methyl substitution is likely to be free of steric clash. R-Vectors will only be displayed along visible explicit hydrogens. Select a ligand of interest and use MOE | RHS | Hydrogens until all hydrogens are visible. For each hydrogen attached to a ligand, a virtual methyl group is positioned along the heavy atom to hydrogen bond with the appropriate bond length. A van der Waals interaction energy is calculated with every active atom in the system (leaving out 1-2, 1-3 and scaling the virtual 1-4 interactions). If the resulting energy is less than 2 kcal/mol, the hydrogen is deemed to be a candidate attachment point. This energy threshold can be adjusted on the Builder page of the Configuration Options panel.

Note! All active atoms will interact with the virtual methyls, even overlaid ligands. Use the System Manager to activate or deactivate ligands and receptors.

The information displayed by the Ligand R-Vectors is similar to that of the Interaction (VDW) surface and the same parameters are used (van der Waals radii and well depths). However, it is important to keep in mind that a virtual methyl group may clash with the ligand itself (not just the receptor). A clash with the ligand means that some sort of conformational change is required. Thus, the lack of an R-Vector may also mean that some sort of ligand movement (translation, rotation, conformational) is required to accommodate the methyl group.

Ligand Torsions

Ligand conformations, whether in crystal structures or as a result of modifications, may have torsion angles in poor geometry. The purpose of the Ligand Torsions display is to provide information about whether the dihedral angle of rotatable bonds are consistent (statistically) with angles observed in small molecule crystal structure data.

The MOE | RHS | Ligand | Ligand Torsions command toggles the display of torsion quality cylinders on active ligand molecules. These graphical cylinders drawn along visible rotatable bonds are color coded to indicate quality: green for good, orange for warning and red for bad.

A colored bar at the top of the Dihedral Profile plot shows the torsion quality when rotating the selected bond. By default, this bar is hidden if Mogul is available.

The underlying software uses the Torsion Library jointly developed by the University of Hamburg, Center for Bioinformatics, Hamburg, Germany and F. Hoffmann-La-Roche Ltd., Basel, Switzerland. The methodology is more fully described in [Schärfer 2013] [Guba 2016] [Penner 2022]. The torsion library is available in the supplementary information to [Penner 2022].

Duplicate a Ligand

Use MOE | RHS | Ligand | Duplicate to create a copy of a ligand. A single connected ligand must be selected in the MOE Window or an error will result. The ligand atoms, residues and chains are duplicated along with membership in any atom sets, restraints, tethers, etc. The chain name of the newly created ligand will have a numeric copy number appended; for example, a copy of MyLigand will be given the chain name MyLigand (2). The original ligand will be hidden and de-activated (made inert) and the created copy will have all of its atoms selected.

Minimize in Isolation

Use MOE | RHS | Ligand | Minimize (Vac) to create a duplicate of a ligand and energy minimize its coordinates in isolation (without any receptor or other atoms). A single connected ligand must be selected in the MOE Window or an error will result. The ligand is duplicated in manner similar to the Duplicate command. The newly created copy is energy minimized and superposed back onto the original. The chain name of the newly created ligand will have (Vac) appended; for example, MyLigand will become MyLigand (Vac). The newly created minimized copy will be de-activated (made inert).

When the minimization is complete, a message will be displayed at the top left of the MOE Window for ~5 seconds and printed to the SVL Commands window. The message has the form

Minimize (Vac): Estrain: ss.sss kcal/mol (e.eee)  rmsd: d.ddd

where ss.sss is the strain energy (energy of original conformation less the energy of the minimized conformation), e.eee is the strain efficiency: strain energy divided by the number of heavy atoms, d.ddd is the root mean square deviation between the original conformation and the minimized conformation.

Ligand Modification and Constrained Energy Minimization

The MOE Right Hand Side Button Bar provides a number of commands for performing energy minimization calculations and ligand modifications. It is assumed there is a receptor-ligand complex in the MOE Window that has been prepared with the QuickPrep preparation procedure. Press the SiteView button to isolate the ligand in the active site. The MOE | RHS | Ligand | Ligand Properties command can be used to display and monitor ligand properties in the MOE Window. The protocol for ligand modification is

  1. Decide which receptor and ligand atoms will be fully flexible.
  2. Use the small molecule Builder to modify the ligand.
  3. Use MOE | RHS | Minimize to perform constrained energy minimization to optimize the receptor-ligand complex.

The MOE | RHS | Constrain menu is used to change the constraint status of selected atoms in the MOE Window. An atom can be either tethered, fixed or free. Tethered atoms will be subject to artificial forces during energy minimization to prevent gross movement; fixed atoms will not move at all; free atoms are not restrained in any manner. The strength and other parameters of the tether restraints are specified in the QuickPrep panel.

The Render command in the Constrain menu can be used to render the tethered and free atoms differently to reflect their state:

tethered

free

Heavy atoms (non-hydrogen) are rendered in Line mode if they are tethered, or Ball and Stick mode if they are free. Ligand hydrogen atoms are rendered in Line mode if they are tethered and Ball and Stick mode if they are free. Receptor hydrogen atoms are rendered in the same style as the heavy atom to which they are attached. Alternatively one can Constrain | Label the atoms with their constrain status: (F): Fixed, (T): Tethered, (I): Inert.

To remove tether restraints from atoms, select the atoms in the MOE Window and use Constrain | Free on the button bar. This can be used to allow receptor active site atoms free movement; for example, to allow a fully flexible residue sidechain.

To add tether restraints to atoms, select the atoms in the MOE Window and use Constrain | Tether on the button bar. Generally, hydrogen atoms need not be tethered, allowing them full flexibility to form hydrogen bonds. It is also good practice to tether a few ligand heavy atoms if certain interactions are to be ensured.

To fix atoms to atoms, select the atoms in the MOE Window and use Constrain | Fix on the button bar. Use Constrain | Unfix to remove the fixed status without affecting any assigned tether.

Use MOE | RHS | Builder to start the small molecule Builder which is used to make modifications to the ligand; it can also be used to make minor modifications to the receptor. The MOE | Footer is selection-sensitive and can also be used to edit bond lengths, angles and dihedrals: select 2, 3 or 4 atoms and use the controls provided. Alternatively, the Protein Builder/Rotamer Explorer can be used to explore alternative sidechain conformations or make residue mutations. Modifications to ligands are made by selecting atoms in the MOE Window and changing their properties in the Builder. Substitutions are made by selecting hydrogen atoms in the MOE Window and selecting a chemical group in the Builder. See Building and Editing Molecules for information on how to use the Builder. Use MOE | RHS | Sketch to start an external chemical drawing program. This allows the selected component in the MOE Window to be edited in 2D; the changes can be brought back into MOE once editing is complete. A 2D sketcher is most suitable for small edits to small molecules and is a complementary tool to the 3D small molecule Builder. The MOE | RHS | Minimize button will optimize the coordinates of the molecular data in the MOE Window. The Minimize button in the Builder is equivalent. Typically, the MOE | RHS | System Manager is used to activate or de-activate receptors and ligands so that there is only one set of active receptor, ligand and solvent atoms.

The QuickPrep prepare procedure is typically used to set up for the following protocol:

  1. Receptor and solvent atoms far from the ligand are held fixed (constrained to not move at all). Typically, an atom is held fixed if it is at least 8 Å from any ligand atom.
  2. Atoms not held fixed in Step 1 will be subject to any restraints that have been set by the prepare procedure or the Constrain commands. The Constrain | Render button can be used to visualize the tethered status of atoms.
  3. A forcefield energy minimization is performed which will stop when the root mean square (RMS) gradient of the potential energy falls below a certain threshold, by default 0.1 kcal/molÅ (configurable in the QuickPrep panel). The current Potential Setup determines the forcefield, solvation model and non-bonded interaction cutoff.

The Cancel menu in the top right corner of the MOE Window can be used to terminate an energy minimization before it has completed.

If the energy minimization calculation terminates normally, and the Ligand Properties are displayed (using the MOE | RHS | Ligand | Ligand Properties button) then binding affinity estimates are calculated and the display updated. Note! If an energy minimization is performed but the Ligand Properties are not displayed, then enable their display and then press Minimize again.
