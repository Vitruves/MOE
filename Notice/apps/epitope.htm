Overview

The MOE Epitope Analysis module uses the results of a protein-protein docking simulation to calculate the most likely residues to have a key role in the protein-protein interface. Epitope Analysis uses Protein Interaction Fingerprints (PLIFs) calculated using Surface Patch Contacts to cluster the docked poses of the "ligand" protein. Clustering is performed using single-link hierarchical agglomeration with the Tanimoto similarity of the fingerprints as the distance metric.

Pose clusters are ranked using an ensemble free-energy score

where N is the number of fingerprint bits in a pose j in cluster i, S is the docking score of pose j, and the sum is over all poses in cluster i. The scaling factor β corresponds to an effective inverse temperature for calculating the cluster entropy. Small values of β will favor clusters with the most member poses and poses with the most contacts whereas large values will favor contacts in poses with the best docking scores. The default value has been validated to provide the best overall results on a diverse test set of protein complexes.

Epitope residues are selected by picking the residues with the highest-occupancy fingerprint bits in each cluster. The population of a bit is calculated using Boltzmann weighting

where the sum is over all poses in the cluster and ρij is equal to one if bit j is set in pose i and zero otherwise. Fingerprint bits which correspond to strong interactions are selected preferentially, followed by weak interactions. Compact epitopes are generated by applying a distance cut-off such that a new residue is only added to an epitope if it is within the specified distance of an existing residue. The total number of residues in each epitope is set to a fraction of the average number of fingerprint bits per pose in the ensemble, such that depending on the size of the interface the number of contacts will be correct on average.

The Epitope Analysis panel provides a convenient way to configure and carry out the calculation, as well as a tool for browsing the resulting epitopes and annotating the corresponding structures in MOE.

Running Epitope Analysis

The Epitope Analysis panel can be opened from the Database Viewer with DBV | Compute | PLIF | Epitope or from the PLIF Analyze panel by pressing the Epitopes button in the panel footer. The panel will open initially as a panel in the Database Viewer by default. Press on the Detach button to detach the panel as a separate window as shown below. Press the Attach button to re-attach the panel to the Database Viewer.

To perform the calculation, select the database field containing the docking scores (or another numerical field to be used in ranking clusters) from the Score pulldown menu. Check Use Site Constraints to apply additional sequence-based scoring terms in the epitope analysis as described in Applying Constraints below. Press the adjacent Configure icon to open the site configuration panel. The number of sites in the current constraint specification will be indicated.

For databases resulting from docking calculations there will be additional fields in the database, usually called mseq and rseq, which indicate the different pairs of proteins, for example different antibodies docked with the same antigen. In that case a Receptor field will be available in the Epitope Analysis panel along with an option to select the integer field to be used for indexing. This enables restricting the calculation to only those poses with the specified index. Select zero or blank to use all poses in the database as is the case by default.

Epitope Configuration Panel

The calculation parameters can be configured by pressing the Configure () icon in the upper right corner of the panel. This will open up the Epitope Configuration panel with the following options.

Score Scale Factor The value of the effective inverse temperature used in cluster scoring.
Cluster Threshold The cutoff value for the Tanimoto similarity above which two poses are assigned to the same cluster.
Epitope Fraction The fraction of the average number of fingerprint bits used to determine the target number of residues in each epitope.
Distance Cutoff The maximum distance (in Å) between a residue in an epitope and its closest neighbor in the same epitope.
Epitope Limit The maximum number of epitopes to calculate.

Press OK in the Epitope Configuration panel to save the parameters or Cancel to discard the changes. Press Defaults to set the parameters to standard values, or Last to restore the values used in the previous calculation (if applicable). To run the calculation press Calculate in the lower left corner of the panel.

Tip Selecting Last in the Configuration panel is a convenient way to inspect the parameters which were used when re-opening a database which already contains epitope data.

Applying Constraints

Epitope Analysis can be directed using experimental information such as Hydrogen/Deuterium exchange spectrometry [Iacob, Huang]. Regions of the protein which are more likely to be involved in the protein-protein interactions can be specified in the site configuration panel shown below. Each segment is given a type which corresponds to the weight parameter used in the analysis (e.g. 1=strong, 2=weak, etc.) Docking pose clusters which contain interactions with a high weight will be ranked higher, and those interactions will also be ranked higher among the residues present within the cluster.

Each row of the constraint table corresponds to a segment defined by a chain ID, beginning and ending residues, and interaction type. Multiple segments can be assigned the same type. Pressing brings up the option menu, allowing the user to enter Site Constraints. Selecting an existing constraint allows the user to edit or remove it. The option menu is described below:

Residue Expression

Create a new segment from the residues specified by a selection expression applied to the specified chain. Set the weight Type and press the Accept icon to add the Constraint or the Discard icon to discard the entry.

Selected in MOE

Create a new segment from the selected residues in MOE. The chain ID and residue numbers will be filled in automatically. Set the weight Type and press the Accept icon to add the Constraint or the Discard icon to discard the entry. A description of the selection is provided.

Load from File

Load a site constraint from a file. Enter the filename in the text field or press the Browse icon to select the file. The file format is four space- or comma-separated values per line (duplicating the list format), specifying the chain, first and last residues and the weight Type. Press the Accept icon to add the Constraint or the Discard icon to discard the entry.

Tip Cut-and-paste can be used to create a valid file for later use.

Edit

Edit the selected row by entering new values directly as shown. Press the Accept icon to add the Constraint or the Discard icon to discard the changes. Press to delete the selected constraints. Note that editing the residue expression is disabled if multiple rows are selected.

Five Types are provided which can be used to create different classes of constraints (strong, weak, etc.) with unique weights. The Constraint Weights can be specified by entering a new value in the appropriate text box. These are used for selecting epitope residues and ranking pose clusters. Larger values correspond to stronger constraints.

When the site configuration is complete, press OK or Cancel to exit the configuration panel and return to the Epitope Analysis panel. If OK was used to save the data, it can be used to recalculate the epitope clusters by pressing Calculate. The Site Constraints are saved in the MDB file and will appear in the configuration panel if it is reopened from the same database.

Tip The site configuration panel can be used to inspect the constraints and save the data at a later time.

Viewing Epitopes

After the calculation has been performed, the Epitope Analysis panel will be populated as shown. One or more epitopes can be selected for display by clicking on items in the list or by scrolling using the up and down arrow keys. Other navigation options correspond to those of the standard Listbox with style 2 multi-selection.

The bottom toolbar controls how the selected epitopes are displayed. The protein sequences shown for the epitopes in the list correspond to the structures in the database field selected in the Database pulldown menu. This should normally be the same as the field used as the receptor during the fingerprint calculation.

Use the checkboxes to turn on or off showing the selected epitopes in the main MOE window. Either unchecking the checkboxes or deselecting the epitopes will remove structures which were loaded from the MOE database.

Annotate MOE Show epitope residues mapped onto the structure in the MOE window. If the MOE window contains other protein chains besides those to be annotated, use the pulldown menu to specify which chains to include. This will apply a ribbon-only representation with the epitope residues highlighted in yellow. Named sets (Epitope_1, etc.) will also be created for convenience in carrying out further operations on the epitope residues. Pressing ? will temporarily highlight the receptor that will be annotated.
Note: The protein sequences must be aligned to those of the receptors used in the docking and fingerprint calculations, otherwise the results may not be reliable. For multi-chain receptors the chains should also appear in the correct order in the System Manager and Sequence Editor.
Note: This operation will not be performed if the chains to be annotated are in a different reference frame from those in the database. In that case, perform the required superposition before annotating.
Database Import the receptor structures from the MOE Database into the main MOE window. The database field containing the receptor structures used in docking will be used automatically for annotation, however if the database contains multiple molecule-type fields a pulldown menu can be used to select the field in the MOE Database containing the structures to annotate. For each epitope, the first entry in the database for that epitope will be used. The structures will have the same representation as described above for annotating a single structure, however the epitope sets will each be part of a different chain in this case.
Note: If the receptor structures for each epitope have a common backbone the imported structures will be superposed and not all annotations may be visible.
Show Poses Show the docked poses of the interacting protein in the MOE window. The database field containing the structures used as the "ligand" in docking will be shown automatically, however if the database contains multiple molecule-type fields a pulldown menu can be used to select the field in the MOE Database containing the poses to display. All entries in the database corresponding to the selected epitopes will be displayed, with a translucent "spaghetti" representation to permit viewing the ensemble of poses contributing to the epitope. Poses corresponding to different epitopes will be sorted into separate groups and tags to simplify manipulation in the System Manager.
Note:If the selected epitopes contain a large number of poses a warning will appear and the display in MOE will be limited to the first 50 poses matching the selection. To view all poses (which may take some time), use Select Entries and import the structures into MOE from the Database Viewer.

Tip In order to retain the epitope annotations in MOE, close the Epitope Analysis panel while the desired items are selected. De-selecting all epitopes will remove the annotations before closing.

If any problems are detected which might cause the Epitope Analysis panel to display incorrect or missing annotations for the indicated display mode and selection a warning button will appear to the left of the bottom toolbar as shown.

To see an explanation of potential problems press the warning button to bring up a Warning panel containing further details. Not all warnings are necessarily fatal, but particular attention should be paid to ensure that the results are as expected.

Additional buttons in the panel footer allow the selected epitopes to be displayed in other MOE applications.

Select in DBV Pressing this button will open the corresponding MOE database in a Database Viewer if it is not already open. All entries corresponding to the selected epitopes in the Epitope Analysis panel will be selected in the Database Viewer. This selection will be applied each time the button is pressed, but will not have any further effect if the selections are changed explicitly in either of the two panels until it is re-applied.
Tip In order to annotate all receptor structures corresponding to a given epitope, first use Select in DBV to select the corresponding entries and then use the DBV Browser to view Selected Entries. Select all chains already in MOE and choose Unselected Chains in the Epitope Analysis panel. The annotation will thus be automatically applied to each browsed structure as it appears.
PLIF Analyze Pressing this button will open the PLIF Analyze panel associated with the corresponding MOE database if it is not already open. This is equivalent to opening the MOE database and selecting DBV | Compute | PLIF | Analyze. Regardless of which is opened first, if the PLIF Analyze and Epitope Analysis panels are both open simultaneously the selections in the two panels will be synchronized (unlike the case with the Database Viewer.) Selecting in the Epitope Analysis panel will select all corresponding entries in the PLIF Analysis panel. Conversely, the selection in the Epitope Analysis panel will include all epitopes for which any entries are selected in the PLIF Analysis panel, even if that does not include all members of the epitope cluster.
Tip Selecting an entry in the PLIF Analysis panel and then clicking on the selected epitope in the Epitope Analysis panel will select all entries in the same cluster as the original selection.

Database Browser

The Epitope Analysis panel can be used to view previously-calculated epitope analysis data. When browsing the Summary Database from a Protein-Protein Docking calculation, carried out for example on multiple pairs of input protein structures, the Epitope Analysis panel is automatically integrated into the Database Browser.

To open the Database Browser plugin version of the Epitope Analysis panel, open the docking Summary Database in a Database Viewer, then select DBV | File | Browse. The DBV Browser will show the following data in the header section:

Subject When set to Epitope Analysis, the text field will show the name of the docking output database corresponding to the current entry.
Receptor Shows the value of rseq for the current entry.
Ligand Shows the value of mseq for the current entry.
Show DBV Press to open the database shown in the Subject in a separate Database Viewer.
Select Entries Performs the same action as Select in DBV described above for the stand-alone viewer.
Keep Press to retain the current annotation in MOE after closing the Database Browser.

In the Database Browser plugin, the Listbox and Annotate functions are identical to the stand-alone Epitope Analysis panel, with one exception: since the Database Browser already uses the keyboard arrow keys, to navigate between database entries, the keyboard navigation between epitopes is accomplished by holding the Shift key while pressing the arrow keys.

Example: Interleukin-13

  1. This example makes use of the results of docking Interleukin-13 to the CNTO607 Fab antibody domain (PDB: 3G6D). Load the structure into MOE.

    MOE | File | Open | $MOE/sample/mol/3g6d.pdb.gz

    View the antibody by showing the ribbons only and coloring the CDR loops.

    MOE | RHS | Hide | All Atoms
    MOE | Protein | Annotate | Antibody | CCG

  2. Open the MOE database in a Database Viewer.

    MOE | File | Open | $MOE/sample/mol/3g6d_dock.mdb

    Make a local working copy.

    DBV | File | Save As | temp.mdb

    Open the Epitope Analysis panel.

    DBV | Compute | PLIF | Epitope

  3. Notice that in this example the top cluster contains most of the top poses. To generate smaller clusters, repeat the calculation with a higher cutoff value. First, press the gear icon in the upper right corner and change the value of Cluster Threshold from 0.4 to 0.5. Press OK to save the change and then Press Calculate at the bottom left corner of the Epitope Analysis panel. The epitope list should be repopulated with a new set of epitope clusters. Select the first row in the Epitope Browser.

  4. Color the Interleukin-13 ribbon by selecting MOE in the Annotate control and selecting Selected Chains in the adjacent pulldown menu. Activate the receptor annotation by checking the Annotate checkbox. Because no chains have been selected at this point nothing will happen and the warning icon will appear to the left. Open the Sequence Editor (MOE | SEQ) and select the chain 3G6D.A. The ribbon should now be colored, with the epitope residues highlighted in yellow. Now close the Epitope Analysis panel by pressing Close at the bottom right.

  5. To view the interacting protein sidechains, open the MOE System Manager (MOE | RHS | System) and expand the Sets list. Select the epitope atoms by clicking the item labeled Epitope_1 and display them.

    MOE | RHS | Show | Selected

    Deselect the atoms by clicking anywhere in the MOE window. The MOE window should now appear similar to the image.

  6. Return to the original epitope data by repeating Step. 2. Alternatively, reopen the Epitope Panel and press the gear icon to raise the configuration pop-up. Press Defaults, then OK to save the values, and then Calculate to refresh the Epitope Browser.

  7. To see the effects of the site constraints, select residues A:85-91 in the Sequence Editor. The highlighted loop is not part of the top-ranked clusters, but we can see what would happen if it was known to be involved. Check Use Site Constraints in the Epitope Panel and press the adjacent Configure icon. This opens the Epitope Site Constraints panel. Press the icon, change the Input Method to Selected in MOE and press the Accept icon to add the Site Constraint. Increase the weight for Type 1 by entering 200 in the text box and press OK to save the Constraint and close the panel.

  8. Press Calculate to apply the constraints. Notice that now the poses which contain interactions in the selected loop are ranked first in the Epitope Browser. Select the top three epitopes and then open the PLIF panel by pressing PLIF Analyze.... The top poses are selected, and it can be seen that they are the only entries with interactions for residues 90 and 91, in the selected loop.
