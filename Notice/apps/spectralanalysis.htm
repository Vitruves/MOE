Introduction

The Spectral Analysis application supports computational studies of NMR, IR, VCD, UV, ECD and ORD spectra for small molecules and peptides. In addition, chemical shift, coupling constant and/or pairwise distance data can be used to determine the relative populations within a set of conformers, yielding the in-solution conformer distribution. This information can be used for compound analysis as well as for understanding the energetics of binding.

NMR spectroscopy measures the magnetic properties of atomic nuclei, and can reveal detailed information about molecular structure. On the NMR Analysis page, NMR shielding tensors and spin-spin coupling constants (J) calculated by a quantum mechanical program [Bally 2011] can be converted into chemical shifts, which can be displayed and (automatically) matched to experimental data.

Vibrational Circular Dichroism (VCD) measures the differential infrared (IR) spectrum produced with left– and right–circularly polarized light as it interacts with vibrational transitions in a molecule [Stephens 1985, Dukor 2000, Vargek 1997]. Chiral centers will exhibit optical activity in the IR range analogous to that seen in other types of spectra such as polarimetry. Calculating the VCD response involves determining the dipolar response of the perturbed Hamiltonian (ℋ') due to the incident electric (E) and magnetic (H) fields according to

ℋ'(E,H) = -(μel · E) (μmag · H).

These calculations require that both the polarizability and magnetic shielding tensors be calculated, which is handled by third-party quantum mechanics (QM) software.

Electronic Circular Dichroism (ECD) measures the differential Ultraviolet (UV) spectrum produced with left– and right–circularly polarized light as it interacts with electronic transitions in a molecule [Trander 2017]. Calculating the ECD involves modeling excited electron states using time-dependent density functional theory.

Optical rotation is the rotation of the orientation of the plane of polarization of linearly polarized light as it travels through certain materials. It occurs as a consequence of different refractive indices for left and right circularly polarized light. Optical rotation dispersion (ORD) measures the dependence of optical rotation on wavelength. Quantum mechanics computation of ORD does not involve time dependent DFT and may be faster than the computation of ECD. Experimentally ORD is measured at a given wavelength, and to compute it in MOE a set of wavelength values must be specified.

ECD and ORD are related by the Kramers–Kronig transformation [Polavarapu 2006]. If experimental ECD data is available the Spectral Analysis application transforms it to ORD using the Kramers–Kronig transformation. Mollar ellipticity [θ] is given by 3298.2 * ε. And ORD, expressed as specific rotation, is given by [φ] * 100 / molar mass. Experimental information for a chemical structure can be estimated by generating a set of conformers and constructing a weighted sum of the appropriate calculated property:

pcalc = Σi wi pi,

where wi is the weight and pi is the calculated property for conformer i. The best-fitting set of conformers can be determined by calculating the weights via a least-squares fit to a penalty function, Χ2:

Χ2 = Σj ((Jexp,j - Jcalc,j) / σJ,j)2 + Σj ((rexp,j - rcalc,j) / σr,j)2 + Σj ((δexp,j - δcalc,j) / σδ,j)2,

where j refers to the individual experimental (and calculated) measurements and the denominators, σp,j, are the errors associated with individual measurements of type p.

Currently, MOE supports Gaussian 09 and Gaussian 16 for QM calculations. Conformers can be generated by either forcefield or QM-based methods.

NMR Analysis

Reference Method

Calculated isotropic shielding constants, σ, can be converted to chemical shifts, δ, by a number of Reference Methods. We will use 13C NMR as an example, but any atom that has a calculated shielding and a Reference Method can be used. As with experiment, the 13C chemical shift can be obtained by subtracting the isotropic shielding constants for the carbon nuclei in the molecule of interest, {σi}, from those of tetramethylsilane (TMS), σref:

δi = σref - σi

While there can be significant errors with any calculation, using the same QM parameters for both will likely reduce them. Thus, chemical shifts calculated in this way often benefit from cancellation of errors.

Alternate reference compounds can be chosen that are more similar to the molecules being studied. This can result in improved results due to greater cancellation of errors. If the experimental chemical shift for the reference compound is known, the isotropic shielding constants can be converted into chemical shifts relative to TMS:

δi = σref - σi + δref

where δi is the chemical shift relative to TMS, σref is the computed shielding constant for the reference molecule, σi is the computed shielding constant for the nuclei in the molecule of interest and δref is the experimental chemical shift for the reference compound. A logical extension to using a single reference compound is to use one reference for sp3-hybridized carbons and a different reference for sp- and sp2-hybridized carbons. Typically, methanol is used for saturated and benzene is used for unsaturated atoms. In the case of hydrogens, the hybridization of the adjacent carbon is used.

Alternatively, the reference compound(s) can be replaced by linear regression models constructed from sets of compounds. If a large set of data is used, the effect of mismatch between calculated and experimental chemical shifts for any single compound is reduced. As above, all calculations must be performed with the same QM parameters. The chemical shift can be calculated by:

δi = (intercept - σi) / −slope

Slope/intercept values can be tabulated for sets of molecules and QM parameters; the CHESHIRE chemical shift repository provides one such example [Lodewyk 2011].

Reference Data

Calculated isotropic shielding constants are converted into chemical shifts by applying a transformation rule that is dependent on the QM parameters used for the calculation. These Reference Methods are collected into a MOE database, where each entry specifies the QM parameters (method, basis set, solvent model and solvent), the element, a transformation rule and a description that appears in the Method menu of the NMR Analysis panel. A sample portion of a Reference Database is provided below:

Description This is a description of the Reference Method defined in the database entry. Examples are TMS, indicating that tetramethylsilane is used as a reference, Linear Scaling, indicating that a regression model relates shielding to chemical shifts and Two Reference, indicating that one model is used for saturated atoms and another for unsaturated atoms. These appear in the Method menu of NMR Analysis.
QM Method This is the QM method used to calculate the isotropic shielding constants. Typical methods would be a DFT method such as B3LYP, Hartree-Fock (HF) or post-Hartree-Fock methods such as MP2.
Basis Set This is the complete description of the basis set used to calculate the isotropic shielding constants.
Solvent Model This is the solvent model used to calculate the isotropic shielding constants. If omitted, a gas-phase calculation was performed.
Solvent This is the solvent used (with the above mentioned Solvent Model) used to calculate the isotropic shielding constants. It can be the name of a solvent supported by the external QM program or the value of the dielectric constant that was used. If omitted, a gas-phase calculation was performed.
Element The atomic symbol of the element the Reference Method applies to. In this case carbon, indicated by C.
Transform This specifies the one or two algebraic transformations used to convert the calculated isotropic shielding constant into a chemical shift. x is the value of the chemical shielding and the result is the chemical shift. When two transformations are used, they must be separated by a semicolon. The first equation is used for saturated atoms and the second is used for unsaturated atoms. All equations must be valid SVL expressions.

Ideally, all entries in a Reference Database should be unique. If redundant entries are present, NMR Analysis will use the first matching entry to calculate chemical shifts. If, for example, multiple regression models are available for a particular set of QM parameters, the Description field must be used to differentiate them. Note that the QM parameters are set when QM Refinement is performed in Conformational Search, which writes them to the output database.

Running NMR Analysis

To perform an NMR Analysis, open a database containing isotropic shielding constants calculated via QM Refinement in the Database Viewer and select DBV | Compute | Molecule | Spectral Analysis. If the required fields are present, the Spectral Analysis panel will appear. The name of the associated database will be included in the window header. Select the NMR page using the buttons at the top of the panel.

Element Specify the element of the current molecule to be analyzed.
Method Specify the method used to convert calculated isotropic shielding constants to chemical shifts. This menu contains all of the entries in the Reference Database that share the element being analyzed and the QM parameters used in generating the isotropic shielding constants. Typical methods would be TMS, Linear Scaling or Two Reference, with the last referring to separate transformations for saturated and unsaturated carbons (or hydrogens connected to such carbons).
Chemical Shifts Specify the name of the file containing the experimental chemical shifts. This is a whitespace-delimited or CSV file. The Experimental Data can be specified by pressing the Browse icon and selecting the appropriate file. The results are compared to the chemical shifts for the current element being analyzed. Recently-used experimental data files are provided as a shortcut.
Action Menu This menu allows the user to Load experimental values from a text file, Save the currently-loaded values to a text file, View the experimental values for the current atom, Clear the currently-loaded filename or Clear all loaded experimental information. Values currently assigned to individual atoms will be retained when the text file is cleared. When loaded, the user selects values from the Expt Data option menu. When no file is loaded, experimental values can be entered manually in a text field.
Reference DB Specify the name of the Reference Database, which contains a set of transformation rules used to convert calculated isotropic shielding constants to chemical shifts. Each transformation is specific to the QM parameters (method, basis set, solvent model and solvent) used in generating the isotropic shielding constants, the element being analyzed, and a description field that describes each method. The Reference Database can be specified by pressing the Browse icon and selecting the appropriate database.
QM Parameters Specify the QM parameters used in generating the isotropic shielding constants, read from the input database. For example, B3LYP/6-31+G** would be a gas-phase DFT calculation using the B3LYP method and the 6-31+G** basis set. These parameters are set in the QM Refinement configuration panel of Conformational Search.
Atom List This list contains all of the available atom-centric information. Label is the user-specified alphanumeric label assigned to each atom, defaulting to the atom name in the input structure. Expt is the experimental chemical shift associated with each atom. Calc is the calculated chemical shift, determined using the current Reference Data and Method. Corr is the empirical correction applied to the calculated chemical shift value. Corr Calc is the corrected value of the calculated chemical shift. Delta is the difference between calculated and experimental chemical shift values. Shielding is the calculated isotropic shielding constant, determined when the conformer database is created.

Selecting one or more atoms in this list highlights them in the 2D depiction and the chemical shielding and shift plots. The Up and Down arrow keys and the Home and End keys can be used to navigate this table. The selected entries can be cleared by right-clicking on the table and pressing Clear Selection.
Label This text field contains the current alphanumeric label for the selected atom in the Atom List. This label can be specified by changing the current value and pressing Apply.
Expt Data If an experimental data file has been read, this is an option menu containing the sorted chemical shifts. Otherwise, this will be a text field containing the currently-assigned experimental chemical shift for the selected atom in the Atom List. This can be specified by changing the current value and pressing Apply.
Correction This text field contains the current empirical correction for the selected atom in the Atom list. This correction can be specified by changing the current value and pressing Apply.
Structure When the input database contains conformers of multiple molecules, pressing the Structure icon displays the 2D depictions of each structure: If there are more than nine molecules in the input database, the Page text field and the forward and backward buttons can be used to navigate the display. Selecting a structure will use the associated conformers for the analysis, resetting the Atom List, the 2D depiction and all plots as appropriate.
Weighting Specify how the calculated chemical shifts are constructed from the set of isotropic shielding constants. None uses the shielding, chemical shifts and coupling constants (if present) of individual conformers. Boltzmann uses an energy-weighted sum while Fitted uses the weights from the Conformer Distribution panel for all properties. Individual conformers are shown in the MOE window. Conformers with negative frequencies are ignored when doing the weighting. The Back and Forward buttons can be used to navigate the individual conformers, and the conformer text field can be used to specify individual conformers.
2D Depiction This contains a 2D depiction of the current structure being analyzed. The label of each atom being analyzed is specified by the View menu, and reflects the current information in the Atom List. In the picture above, the atoms are labeled by their alphanumeric label. A highlight is drawn for each selected atom in the Atom List. If the Delta column is defined, the size of the highlight reflects the magnitude of the delta value (the difference between individual experimental and calculated chemical shifts). Coupling constant information can also be displayed if present and a single atom is selected in the Atom List.
View This specifies the atom label to use in the 2D Depiction. Label is the user-specified alphanumeric label for each atom, Index is the atom index in the input structure, Expt is the experimental chemical shift, Calc is the calculated chemical shift, Corr Calc is the corrected chemical shift, Delta is the difference between calculated and experimental chemical shifts, Shielding is the calculated isotropic shielding constant and Coupling is the calculated coupling constant (vs. the selected atom). Both Shielding and Coupling reflect values obtained from the QM calculation.
Coupling This specifies how coupling constant information should be displayed if available and if a single atom is selected in the Atom List. The options are to show couplings to the same element (C-C, if carbon is being analyzed), All, which shows the coupling constants associated with all of the atoms in the structure and None which prevents display of coupling constants. Positive values are shown as blue disks and negative values as orange disks. The radius of each disk is proportional to the absolute value of the coupling constant with a dark circle indicating that the value exceeded the maximum radius.
Table Shows the coupling constant matrix for all atoms as a table in a separate window.
Information Plot This provides a graphical representation of the information contained in the Atom List, read from the Experimental Data file (if present) and determined from association of calculated and experimental chemical shifts. The particular representation is specified by the View menu as one of the following options: Setting View to Shielding shows the calculated isotropic shielding constants, determined using the current Reference Data and Method. In this case, a single entry in the Atom List is selected, and the corresponding shielding constant is highlighted. The scroll wheel can be used to zoom or translate the Shielding plot. Setting View to Shift shows the calculated chemical shifts (in black) and the chemical shifts read from the from the Experimental Data file (in blue). As calculated and experimental peaks have been associated, a graphical representation of the Delta column of the Atom list is provided in red (the empirical correction is applied if present). As above, a single entry in the Atom List is selected, and the corresponding chemical shift is highlighted. The scroll wheel can be used to zoom or translate the Shift plot.

The Display checkboxes control what chemical shift values are shown. If Calculated is on, calculated chemical shifts will be shown. If Experimental is on, experimental chemical shifts will be shown. If a single experimental chemical shift is selected in the plot, its value will be shown. Setting View to Delta shows the difference values (Delta) between experimental and calculated chemical shifts (the empirical correction is applied if present). In this case, there are seven experimental/calculated chemical shift associations with errors ranging from roughly 7 to 15. The Label associated with each row of the Atom List is included with each histogram column. Selecting entries in the Delta plot will select (or highlight) the appropriate entries in the Atom List and 2D Depiction. Setting View to Abs Delta shows the absolute Delta values between experimental and calculated chemical shifts (the empirical correction is applied if present). This uses the data in the above plot, with positive Deltas for all histogram entries.
View This specifies the particular data representation to be shown in the Information Plot. The four possibilities are shown above.
Reset Zoom This button is used to restore the Shift or Shielding plots to their original view, removing the effect of any zoom or translation that has been applied.
RMSD Limit If two conformations have a root mean square distance between heavy atoms less than the specified value, then they are considered duplicates and the conformation with higher energy is filtered out.
Auto-Assign When pressed, unassigned experimental and calculated chemical shifts will be associated such that the associated Deltas are a minimum. Magnetically equivalent atoms, defined as topologically equivalent atoms whose isotropic shielding constants are within 5 ppm, will be determined. Such atoms, if present, are treated as a single atom when associating calculated and experimental chemical shifts. This process will not change manually-assigned atoms.
Reset Assignments When pressed, all experimental and calculated chemical shifts will be disassociated and all Expt and Delta columns cleared. This can be used as a "global reset" of the chemical shift assignments.
Create Report When pressed, an HTML report reflecting the current contents of the NMR Analysis panel will be created and loaded into the default web browser. All images are saved in a subdirectory drawn from the report filename. This report can be modified in any editor capable of working with HTML files.

NMR Examples

In this section we will present a number of example NMR Analyses using the Spectral Analysis application. Sample files are contained in $MOE/sample/mol and $MOE/lib.

Cresol Structural Isomers

  1. Open the database to analyze.

    First, clear the system:

    RHS | Close

    Open the MDB file:

    RHS | Open

    In the Open panel, select $MOE/sample/mol/cresols_dft_qm.mdb and press OK.

    Save a local copy to disk:

    DBV | File | Save as...

    In the Save panel, enter cresols_dft_qm.mdb and press Save. This will be used for the remainder of this example. This database contains o-, m- and p-cresol, each with a single conformer. It was created with QM Refinement in Conformational Search.

  2. Start the Spectral Analysis application.

    DBV | Compute | Molecule | Spectral Analysis...

    As the required fields are present in the database, the NMR Analysis application panel will be shown.

  3. Specify the Reference Database.

    The default Reference Database is $MOE/lib/NMR_ref.mdb. To select a different database, press Browse and in the Open panel, select the database that will be used.

  4. Specify the Experimental Data.

    If the Chemical Shifts text field is empty, press Browse to select the experimental data file.. In the Open panel, select $MOE/sample/mol/p_cresol_data.txt and press OK. At this point, the NMR Analysis panel should look like this:

    In the Atom List, the Shielding column (offscreen to the right) contains the calculated isotropic shielding constants read from the input database. The Calc column contains the calculated chemical shifts obtained by applying the Reference Model (TMS) to the shielding values. The Expt and Delta columns are empty because no pairing of experimental and calculating data has been performed. The Corr and Corr Calc columns are empty because no empirical corrections have been entered.

    The atoms in the 2D Depiction are labeled by the Label column of the Atom List, which defaults to the atom name in the mol field of the input database. The plot shows the calculated chemical shifts (in black) and the experimental chemical shifts (in blue). When a row is selected in the Atom List, the corresponding atom in the 2D Depiction and the plot is highlighted.

  5. Assign atoms to Experimental Data.

    Select the last atom in the atom list (roughly 125 ppm) and select the smallest entry in the Expt Data option menu (roughly 20 ppm). This is a mismatch, which will generate a large Delta. Select the third atom in the atom list and the similar-value entry in the option menu (both roughly 155). This is a match with a small Delta. Select both entries in the Atom List; the NMR Analysis panel should look like this:

    Note that the radius of the selection marks on the 2D depiction corresponds to the values of the Delta column, as do the representations of the absolute Delta on the Shift plot. The dark ring on the larger circle indicates that the actual value is larger than the maximum radius value. Two rows in the Atom List are selected, so two calculated and two experimental chemical shifts are highlighted in the Shift plot. As the experimental/calculated shift associations were assigned manually, they are marked with a *.

  6. Label individual atoms in the Atom List.

    Select the third entry in the Atom List, enter C1 in the Label text field and press Apply. This changes the label for the third atom. Repeat this process to change the label of the fifth atom to C2. Note that the labels of the 2D Depiction change as the Labels are changed.

  7. Enter an empirical correction

    Select the third entry in the Atom List, enter 10.0 in the Correction text field and press Apply. The correction appears in the appropriate column of the selected entry, which defines the Corr Calc column value. Adding the correction decreases the Delta column value.

  8. Automatically assign atoms to Experimental Data.

    Press Reset Assignments to clear the two manual assignments, and press Auto-Assign to automatically associate calculated and experimental chemical shifts with minimum Delta. The absolute Delta representations on the Shift plot are now quite small, and reflect the presence of the empirical correction. The magnitude of the Delta values can be easily seen by changing View to Delta to display a histogram plot:

  9. Examine alternate structures.

    Press the (Structure) icon to display the 2D depictions of each structure in the input database. As the number of experimental peaks is less than the number of carbon atoms, some of them must share chemical shifts or be magnetically equivalent. Pick the third structure, p-cresol, as it is the only symmetric structure. Change plot View to Shift to show the calculated and experimental chemical shifts. The NMR Analysis panel should now look like this:

  10. Automatically assign atoms to Experimental Data

    Press Auto-Assign to automatically associate calculated and experimental chemical shifts with minimum Delta. Carbons in the 2 and 6 positions and the 3 and 5 positions of the phenyl ring are topologically equivalent. As their calculated chemical shieldings are quite similar (much less than the 5 ppm cutoff), they are considered magnetically equivalent and the same experimental chemical shift is assigned to both atoms of each pair. The chemical shifts group well on the Shift plot and all Delta values are small, suggesting that p-cresol corresponds to the experimental shifts.

  11. Analyze elements other than carbon.

    Change Element to H to analyze the hydrogen atoms of p-cresol. As experimental results are tied to individual elements, changing the element clears the previously loaded information. There is an entry for hydrogen in the default Reference Database, so calculated chemical shifts are available. The panel should look like this:

    Note that hydrogens have appeared in the 2D display. They will also appear when examining coupling constants for all atoms.

  12. Analyze Coupling Constants.

    NMR spin-spin coupling constants (J) are calculated as a matrix of values, expressing the coupling of each atom with the other atoms in the molecule. The coupling constants can be visualized for individual atoms by selecting one of the atoms in the Atom List and changing the Coupling menu to either the same-element (H-H in this case) or All. If we select the hydrogen labeled H1, and select H-H the following depiction shows the couplings with that atom:

    Positive values for J are represented in blue, negative values in orange and the size of the disk is proportional to the value (in Hz). The ring around the central carbon indicates that the actual value is larger than that of the maximum radius. The specific values of the coupling constants can be seen by selecting Coupling in the View menu. If Boltzmann weighting is turned off, a similar scheme is used to color the atoms of the conformer shown in MOE. Pressing the Table button brings up a table containing the full matrix of J values:

    Hydrogen H1 is selected in the Atom List, so its entry in the table is highlighted with an asterisk. As with the other displays in NMR Analysis, the display is synced to the settings of the panel, reflecting the current structure, conformer, etc. If the atom labels are not sufficient to identify rows in the table, the atom indices can be shown in the 2D display by setting the View menu to Index.

Calculate TMS Reference Shielding

This example will use Gaussian to calculate the gas-phase Reference values for TMS to be used with B3LYP/6-311+G** calculations.

  1. Load the TMS starting structure.

    First, clear the system:

    RHS | Close

    Open the file containing TMS:

    RHS | Open

    In the Open panel, select $MOE/sample/mol/TMS_qm.moe and press OK. Starting structures for methanol and benzene are also provided in $MOE/sample/mol.

  2. Calculate isotropic shielding constants

    1. Start Conformational Search:

      MOE | Compute | Conformations | Search...

    2. In the Conformational Search panel, select Refine with QM.

    3. Press the configure icon to bring up the QM Calculation Options panel.

    4. Set the QM parameters for the calculation:

      Set Engine to Gaussian
      Set Method to B3LYP.
      Set Basis Set to 6-311G.
      Set Polarization to **.
      Set Diffuse to +.

    5. Enable Geometry Optimization, Keep QM Files, and NMR Shielding.

    6. Enter symmetry=loose in Keywords. This instructs Gaussian to use loosen cutoffs during symmetry determination in the first optimization cycle of the calculation.

      If running MOE on a multi-core machine, specifying %nproc=n (where n is the desired number of cores) in Keywords can be used to run Gaussian in parallel.

      As Gaussian is memory-intensive, it is recommended to explicitly allocate sufficient memory: specify %mem=8gb in Keywords.

      The QM Calculation Options panel should look similar to this:

    7. Press OK to close the QM Calculation Options panel and OK in the Conformational Search panel to run the MM calculation and then launch the QM calculation locally.

      Two files will be created: csearch.mdb, which contains the forcefield-minimized structure, and csearch_qm.mdb, which will be used to hold the QM-minimized structure and the NMR shielding values in the NMR field. While the isotropic shielding constants can be extracted from csearch_qm.mdb, it might be easier to edit the Gaussian output file in the run-time directory that was created.

    8. When the QM calculations are complete, use

      DBV | Compute | QM | Import

      to import the QM results into csearch_qm.mdb if this was not already done automatically.

    The directory csearch_qm_conffiles will have been created in the current MOE directory. This should contain a single directory as only one conformer for TMS was generated. This directory contains the input and output files for Gaussian when the job completes. The output file, gaussian.out, includes the isotropic shielding constants if the calculation was successful. As TMS is highly symmetric, the full point group of the calculation should be Td (i.e. all four carbons are equivalent).

    Upon successful minimization, the isotropic shielding constants are calculated in the second job step. Search for SCF GIAO Magnetic shielding in the gaussian.out file. Successive lines contain the Isotropic value for each atom; atoms 2, 6, 10 and 14 are carbons and their shielding constant should be equal to or very close to 189.7772 (depending on Gaussian version, hardware, operating system, etc.).

  3. Create an updated Reference Database

    Open the Reference Database:

    RHS | Open

    In the Open panel, select $MOE/lib/NMR_ref.mdb and press OK.

    Save a local copy to disk:

    DBV | File | Save As

    In the Save panel, enter local_NMR_ref.mdb and press Save.

    Add an entry for the gas-phase B3LYP/6-311+G** calculation by bringing up the Add Entry panel:

    DBV | Edit | New | Entry

    and entering the QM parameters into the panel. When complete, the panel should look like:

    Press OK to create the new entry. Note that the specific value used in Transform is that provided in the Gaussian output file for the carbon isotropic shielding constant. The bottom of the Reference Database should now look like this:

Perform QM Refinement in MOE

This example uses Gaussian to calculate and visualize NMR shielding constants. It demonstrates running QM calculations in MOE.

  1. Create the sample molecule.

    First, clear the system:

    RHS | Close

    Start the Builder:

    RHS | Builder

    and select C to create a new molecule (press OK) and then C, N, C to complete the structure. Press Close to close the Builder.

  2. Generate conformations.

    1. Start Conformational Search:

      MOE | Compute | Conformations | Search

    2. Select Refine with QM and press the Configuration button. This brings up the QM Calculation Options panel.

    3. Set the QM parameters for the calculation:

      Set Engine to Gaussian.
      Set Method to HF.
      Set Basis Set to STO-3G.
      Set Polarization to *.
      Set Diffuse to none.

    4. As Gaussian is memory-intensive, it is recommended to explicitly allocate sufficient memory:

      Set Keywords to %mem=8gb.

    5. Enable Keep QM Files and NMR Shielding, which also turns on Vibrational Frequencies.

    The Options panel should look like this:

    Press OK to close the Options panel, and OK in the Conformational Search panel to perform the forcefield-based search (using LowModeMD) and then launch the QM calculations locally. This creates csearch.mdb containing the classical results, and csearch_qm.mdb, which will be used to hold the QM-minimized structures. Also created in the current directory will be the sub-directory csearch_qm_conffiles, which will contain all of the files used in the QM calculation.

  3. When the QM calculations are complete, use

    DBV | Compute | QM | Import

    to import the QM results into csearch_qm.mdb if this was not already done automatically.

  4. Start the Spectral Analysis application.

    DBV | Compute | Molecule | Spectral Analysis...

    As the required fields are present in the database, the NMR Analysis application panel will be shown. It can be analyzed as with the first example.

Running VCD and ECD Analysis

Spectral Analysis has separate tabs for VCD and ECD analysis which are designed to be similar for user convenience. To start the VCD or ECD Analysis application, open a database containing the required data from a QM calculation in the Database Viewer and select DBV | Compute | Molecule | Spectral Analysis. The name of the associated database will be included in the window header. Use the buttons at the top of the panel select the VCD tab to perform VCD analysis or ECD tab to perform ECD analysis.

QM Parameters The method, basis set, and solvent parameters (if present) used to generate the data in the QM Database. For example, B3LYP/6-31G* PCM(78.4) would indicate that the hybrid B3LYP density functional was used with the 6-31G* basis set and solvation modeled with the Polarizable Continuum Model using a dielectric constant of 78.4 (water). These parameters are set in the QM Refinement configuration panel of Conformational Search.
Experimental Data The name of the file containing the experimental chemical shifts. This is a whitespace-delimited or CSV file.
The Experimental Menu allows the user to load or clear experimental data. Select Load Data to open a file browser to select the desired file. The data is read into the analysis panel and recorded in the database environment for later use. Select Clear to remove the experimental data from the analysis panel and environment.
Conformer Table The table on the left-hand side shows the list of stereoisomers and their associated conformers imported from the QM Database.

A +/- in the far left column indicates if the conformer list has been expanded (-) or if only the Boltzmann-weighted data is currently displayed (+). Conformer Distribution Methodology Overview

An important application of NMR and NOE studies is the elucidation of dominant conformers of a compound in solution. Knowledge of the dominant conformers can be used to understand and optimize the compound's biological activity. The population distribution of individual conformers, and thus the dominant solution conformations, can be determined by comparing calculated to known experimental values [Navarro-Vazquez 2018], and then finding the distribution of conformers that yields the best agreement with the experimental measurements.

A set of conformers can be computationally generated for a compound via forcefield or QM-based methods based on structure and energetics. Calculated measurements can then be obtained for every conformer through simulation: pairwise distances can be obtained via classical or quantum chemical methods, while chemical shifts and coupling constants can be calculated via quantum chemical simulation.

The problem of finding a compound's conformer population distribution can be cast as a Least Squares Error optimization, in which the sum of squared residuals – differences between weighted calculated values and experimentally measured values – is minimized over all conformations by adjusting the weights. Measurement error is taken into account to reduce the impact of less reliable measurements, and experimental measurement distance bias is alleviated by applying vibrational analysis.

Least Squares Error Optimization

The Least Squares method used in the Conformer Distribution application seeks to minimize the overall difference or error between the experimental measurements and a weighted average of the simulated measurements. The weights in this weighted average represent each conformer's contribution to the simulated measurement values, and are meant to reflect the relative proportion of each conformer expressed within the in-solution conformer population. When the weights accurately reflect the true conformer distribution, the observed (experimental) measurement should in principle correspond closely with the simulated weighted-average values.

The experimental measurement error is used when available, otherwise, an estimated value is applied.

Data Requirements for Accuracy

The Least Squares method used by the Conformer Distribution application relies on non-degenerate measurement data that cover key areas of the molecule under study, and on accurate error estimates.

  • The calculated set of conformers used to perform the Least Squares analysis should avoid including conformers with highly similar measurement values, otherwise, degeneracy can result, in which multiple possible optimal solutions present themselves, all very similar. When this happens, it becomes difficult to obtain a meaningful interpretation of the results.
  • The calculated (AT) and experimental (b) values are normalized by the standard deviation of the measurement errors. Because this calculation is sensitive to the values of σ, an accurate estimate of the measurement error is vital to obtaining an accurate computed conformer distribution.
  • The pairwise NOE values must cover the areas of interest on the target molecule, and it is best to have as many input data values as possible. The addition of coupling constant or chemical shift experimental results (or both) will increase the likelihood of arriving at a correct solution.

  4. Create a subset plot.

    The effect of varying weights of a conformer subset can be determined by creating a subset plot. Using the Conformer list created above, select the fourth and sixth conformers when sorted by Weight. Click on the 6 conformers solution on the conformer plot. This should align the vertical red line with the 6 on the x-axis. Switch the Mode of Operation to Subset and press the Subset button to create a 2-conformer subset using the previously selected rows. Press Calculate to start the calculation, which will take a few seconds to complete.
