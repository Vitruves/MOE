Many aspects of the MOE interface can be configured and customized. Many customizations are performed through the Settings menu, MOE | , with changes being made primarily through the Configuration Options panel. The settings are stored in an RC (resource configuration) file, $HOME/moefiles/moe-rc.

Note: The user resource configuration file is rewritten each time a MOE session ends normally. Any changes made during the session are saved at this time. This means that if you wish to edit your configuration file manually, you must do so after closing MOE to avoid losing any changes. Note that comments in a user configuration file are not saved.

The menu is available in the MOE, Sequence Editor, and Database Viewer window header line. The three windows have different options in their menus:

Window Options
MOE Appearance, Builder, QuickPrep, Align/Superpose, Protein Defaults, Java/JDBC Database, Web Settings, Mouse, Function Keys, Miscellaneous, User Interface, Draw Settings, Stereo Mode
Sequence Editor Appearance, Align/Superpose, Protein Defaults, Java/JDBC Database, Web Settings, Function Keys, Miscellaneous, User Interface
Database Viewer Database Viewer, Function Keys

Tip! Changes in the Configuration Options panel are saved and put into effect by pressing Apply or OK (in this latter case, the panel will close after saving). Note, however, that only changes to settings on the currently displayed page will be saved. Each page must be saved separately, to avoid saving unseen changes. Unsaved changes to settings are displayed in blue font.

Builder

The Controls section is used to enable or disable the display of the hybridization, SMILES input and atom name controls in the molecule builder.

The Options section contains checkboxes used to turn on or off the Create New Molecule popup to set chain and tag names of new molecules, the option to adjust the view while building so that newly created atoms are visible, and the automatic labeling of atoms with incorrect topology. When the keyboard shortcuts are enabled, one can build molecules by pressing keys when the mouse focus is in the MOE 3D Window:

Key(s) Description
1,2,3 Create or change to single/double/triple bond
x Remove bond
c,n,h,o,s,p Change Element to C, N, H, O, S, or P
f,l,b,i Change Element to F, Cl, Br, or I
0,-,+ Set Charge to 0, -1, +1

The Fragments section is used to enable or disable the Combinatorial Builder option, whether the conformation space of free rotors will be sampled prior to structure generation and the R-group library that will be used.

The Sketcher section allows to set the preferred sketcher(s) to be used with MOE and to specify the class path for Java sketchers such as MarvinSketch (see 2D Sketcher Programs for more details).

You must press Apply or OK to put any changes into effect.


QuickPrep

The rendering style (Solid, Line, Dot), material property, and coloring mode of "Quick" surfaces (Receptor Surface, Ligand Surface, Interaction Surface and Protein Surface) are set in the top part of the page. The color widget is used for color selection when Constant color is chosen. Quick surfaces can be created via MOE | RHS | Surface or the System Manager. If Clear Surface is enabled MOE | RHS | Surface | Clear will remove all surfaces. Otherwise only molecular, interaction, and patch surfaces are deleted. The Replace existing option controls whether newly created Quick surfaces will replace existing surfaces of the same type without confirmation. When disabled, a confirmation dialog is raised.

R-Vector specifies the van der Waals interaction energy threshold and color used to draw exit vectors.

The Duplicate options specify properties of the molecule object that are created using MOE | RHS | Ligand | Duplicate or MOE | Edit | Duplicate.

  • Activate+Show. If on, the duplicate will be shown and all atoms will be active (the original molecule/ligand in all cases will be hidden and made inert). If off, the active state and visibility will be that of the original ligand atoms. The newly created atoms will be selected.
  • Copy Render Style. If on, the atom and ribbon render style will be also duplicated.
  • Ligand Stick. If on, duplicated ligands will be rendered in stick mode. If off, it will be rendered in the same way as the original ligand atoms.
  • Element Color. If on, the duplicate atoms will be colored by element, duplicate ribbons by secondary structure. If off, they will be colored in the same way as the original atoms and ribbons.

The Minimize (Vac) options are used to control the MOE | RHS | Ligand | Minimize (Vac) operation. If the Display message option is on, a message reporting the minimized strain energy and RMSD will be displayed for the specified number of seconds. If off, no message will be displayed. If Color newly created carbons is on, the duplicate of the ligand will have its carbon atoms colored by the specified color. If off, the carbon atom coloring will be the same as that of the original ligand.

The SiteView Cutoff distance is used to identify which nearby residues to show when using MOE | RHS | SiteView. Any residues which contain an atom within this distance of the ligand and/or selected atoms will be shown. All other residues are hidden. If the Toggle Site option is enabled, the SiteView button will toggle between showing the ligand site and reshowing the structure outside of the ligand binding site. If Ignore inert is enabled, residues which have all atoms hidden and inert are ignored by SiteView, and neither shown nor hidden. If Preserve render style is enabled, current atom render style settings will not be changed by SiteView.

You must press Apply or OK to put any changes into effect.

Align/Superpose

The parameters used for protein alignments and superpositions with the Protein Align and Superpose panel are controlled via the Sequence Alignment and Structural Superposition options. The Score Matrix specifies the directory in which the amino and nucleic acid similarity score matrices reside, which are selected via the dropdown menus. MOE is distributed with 70 score matrices derived from the literature. The chosen matrix is also used by the Sequence Editor similarity coloring and plots.

Gap Start specifies the gap penalty used during sequence alignment, whereas Gap Extend is the penalty for extending a gap. The maximum number of iterations allowed during the refinement procedure is given by Iteration Limit. Failure Limit sets the number of consecutive failures that will cause termination of the refinement procedure. Here a failure is defined as an iteration that does not improve the alignment score. Build-up selects the strategy for use in generating the initial estimate of the sequence alignment.

The default subset of atoms used for structural alignment is controlled with the Atom Sets option. By default, alpha carbons are used. The Custom button may be used to choose any subset available via the checkboxes, which will activate when the button is selected. These atom sets are also used by the RMSD coloring and RMSD plots in the Sequence Editor.

Two default weighting options for protein structure superposition can be selected. Secondary Structure will upweight residues with matching secondary structure. The Refine by Distance option enables an iterative algorithm with dynamic residue weights based on residue pair distances during the superposition calculation.

When enabled, Result Output will cause the pairwise sequence identity or RMSD values to be printed in the SVL command window.

You must press Apply or OK to put any changes into effect.


Protein Defaults

The Project Path field specifies the default search path to be used by the Project Search panel, as well as for the SE | Annotate | Auto-Annotate and SE | Alignment | Constraints | Auto-Constraints menus. It is also used to populate the MOE | Protein | Search and SE | Protein | Search menus. If the path includes $MOE/project, then databases installed on the Default MOE/web Server, if any, will be included as well (with local databases of the same name taking precedence). If Include Remote PSILO Families is enabled, any databases installed on the configured PSILO server will also be included.

The Rotamer Library is used by multiple applications. The Rotamer Library is used to defined the standard library residues in Structure Preparation, and the rotamers and amino acids available for rotamer exploration in the Protein Design, Protein Builder, DNA/RNA Builder and the Homology Modeler. Multiple libraries can be specified in a semicolon separated list.

The Loop Library is used by the Loop Modeler application. It is also used for the loop building stage of the Homology Modeler. Multiple libraries can be specified in a semicolon separated list.

Click the Validate button to validate both the Rotamer Library and Loop Library specified above the button. A validation report will be opened in a text editor.

Render Protein is used to enable or disable the highlighting of key structural features when reading from PDB or mmCIF files. For more information, see Rendering PDB and CIF files.

Ligand/Peptide specify the maximum number of residues allowed in a peptide/nucleotide chain to be considered as a ligand in e.g. the System Manager. When working with large peptide ligands it might be beneficial to increase this setting.

You must press Apply or OK to put any changes into effect.


Java/JDBC

The Java Executable text field sets the path of the Java Executable that is used by MOE. However, if Java is not found, or a different version is preferred, this field can be used to modify the default setting. The text shortcut shows possible locations for the Java executable. The Test button will test the specified Java executable and display its version when successful.

The JDBC Database Viewer and other applications that use the JDBC relational database access library generally require server and user specification.

Up to four connection configurations may be specified.

The fields and commands in this window are as follows:

  • Alias. String used for the Database Alias option menu in the JDBC Database Viewer. If no alias is given, then the URL itself will be used in the menu.
  • URL. The URL specifying the database to which to connect. This is a JDBC URL that begins with the protocol string jdbc:. The specific form of this URL will be defined in the documentation of the JDBC driver used.
  • Driver. The JDBC driver class file to be used to communicate with the database server. Refer to the documentation of the JDBC driver used for the exact string to enter.
  • Username. The user account to use when connecting to the database server.
  • Password. Defines the password to use when logging-in to the database server. This must be either the mpu_ encoded password for the given Username or the non-encoded password. If the non-encoded password is entered, then mpu_password[] will be run on the text when Return is pressed. If the desired password is an empty string, press Return in the textfield when the textfield is blank. A pre-encoded password can be pasted into this field.

Validate Connection calls jdbc_open with the currently displayed values to determine if the data allows for a valid connection to be established. To clear each entry in the form, press Clear Form. OK or Apply must be pressed before the changes will take effect.

Warning!   The JDBC Database Viewer must be closed and reopened to have new connections recognized by the viewer.


Web Settings



The MOE/web field specifies the name and listening port (hostname:port) of the default MOE/web server to be used. Recent choices are available in the shortcut of the text field. The Test button will contact MOE/web and report its state; if successful, the MOE/web version is shown as well. To start a MOE/web server on the local machine, please refer to Start the Web Server.

Connecting to PSILO requires setting the PSILO URL, user name, and password. If you have access to a PSILO server, you can configure MOE to connect to it, for example using the Open panel. Enter the PSILO server address (host address and port), Username, and Password, each time hitting enter in the text field to acknowledge the data entry. The Test button allows to validate the connection to the PSILO server and the login credentials.

RCSB is the host address of an RCSB mirror site (or other compatible site). Connecting to the RCSB requires no username or password. By default, MOE will connect to the main RCSB site https://files.rcsb.org, as maintained by the World Wide Protein Data Bank consortium.

PDB Archive specifies an HTTP or FTP address of a valid PDB FTP Archive. This URL will be used to update the entry list on the RCSB page of the Open panel.

If your network uses an internet proxy server or if your machine requires a proxy server to access the internet or intranet, you may need to configure the Proxy server. There are a number of modes to choose from, but in all cases, if the environment variables http_proxy, https_proxy, and/or ftp_proxy are set, they will take precedence over any other setting. If used, these variables should contain the information in the form host:port. See Using a Proxy Server for more information. Possible modes include:

  • System: This will inherit the settings from the Internet Option Control Panel on Windows, or the Network Control Panel on macOS computers. It has no further effect on other UNIX platforms.
  • Auto-Detect: This option is only available on Windows and macOS. MOE will attempt to automatically detect the location of a proxy configuration script, and use it. If this fails, it will use the configuration script specified, if any, under the following option. If none is specified, the manual proxy configuration specified will be used.
  • URL Script: This option is only available on Windows and macOS. Your network administrator will give you the address of the Script to fill in here. This script tells MOE, for each URL, whether the proxy should be used or not, and which proxy to use (there may be multiple). If the script cannot be accessed, it will default to the manual proxy configuration from this panel instead.
  • Manual: Specify the proxy host and port for each protocol (HTTP, HTTPS and/or FTP), Use HTTP Proxy Server Setting for all Protocols to copy the information from the HTTP boxes to the HTTPS and FTP ones. A SOCKS proxy, if any, will be used for any protocols other than HTTP, HTTPS and FTP, as well as for any of those which do not have an explicit proxy server specified. Bypass proxy for Local Addresses, if enabled, means the proxy will not be used to any URL where the hostname does not contain a period (.). An additional semi-colon (;) separated list of hosts, domains or IPs for which the proxy should not be used can be specified below. These are usually machines on the local network.
  • None: Do not use a proxy.

You must press Apply or OK to put any changes into effect.



Database Viewer

Database plots, such as the data and correlation plot, can be attached to the database viewer window or created as separate windows. The option Plot Location specifies the default behavior.

The Modifier fields contain SVL commands to execute when the left mouse is double-clicked in any cell of the database viewer. None means a plain double click, Control means a double-click with the Ctrl key held down and similarly for Shift and Alt.

Row Color sets the color of alternate rows in the Database Viewer.

Invert text color on dark background controls, whether the default cell text/foreground color will be inverted when the cell has a dark background color. This improves the readability of the cell content when field coloring is used.

Show popup menu shortcut markers, when selected, will cause a popup menu to appear in the cell the mouse currently points to: . Clicking on the triangle brings up the same menu as a right-click in the cell.

Bubble Help controls the display of bubble help on molecule cells in the Database Viewer. Note that the bubble help is only shown if the database rows are minimized to their minimum height. Additionally a modifier key can be set which needs to be pressed before the mouse hovers over the molecule cell to pop up a 2D diagram of the molecule in the cell.

The Browser option sets the default rendering style and carbon color of ligand structures when browsing molecule fields in the Database Browser.

You must press Apply or OK to put any changes into effect.


Mouse (MOE Window)

MOE allows the user to completely reconfigure the mouse behavior in the MOE main 3D window. This is useful, if e.g. only a two button mouse is available or the user is accustomed to a specific mouse mapping. To change the mouse configuration, select the mouse event to change (e.g. Left Button Drag, Right Button Click, etc.) and then choose the action from the pulldown list which MOE should perform in case of released mouse events. To unset an action select the empty line on the bottom of the pulldown list. If the chosen action is not the default action for this mouse event a red exclamation will appear beside it, and the bubble help for the exclamation mark will describe the default action for this mouse event. At the top of the panel the Modifier Key can be selected for the mouse events to be configured. A Status line at the bottom of the panel summarizes the number of changes and additions with respect to the MOE default mouse configuration.

SVL commands can also be assigned to mouse single click and double click events. When Execute SVL Command is selected in the pulldown list a Set button will appear on the event line. It opens a popup with a text field, in which the SVL expression to execute can be entered. To assign the expression the Enter key must be pressed. In this expression, the $ character is replaced by the atom key of the clicked atom (or zero if no atom is clicked). For example, to toggle the label setting of selected atoms using the shift right mouse button in the MOE Window, you would enter the following SVL expression in the Shift + Right Button Click text field:

if $ then aSetLabelName[$, not aLabelName $]; endif

The Sensitivity controls the sensitivity of rotation and translation as a function of mouse movements. Higher values provide more precise control in the rendering area than lower values. The sensitivity is a floating-point number with default value of 250; this value provides fine control without significant loss of speed. Values ranging from 100 to 1000 produce good results.

Buttons is used to specify the number of mouse buttons. Normally the default Auto Detect will work correctly. When a 2-button mouse is detected or selected, the functions of the middle button are mapped to the right mouse button. In this case, the Alt key may be used to access popup menus.

The mouse configuration changes are applied right away (Note that the changes will be written to the user moe-rc file when MOE is closed). To discard the changes and return to the settings when the Mouse Configuration panel was opened, press Undo Changes. If you want to return the default mouse configuration of MOE, press MOE Default.
Note! Returning to the MOE Default configuration will still retain any Execute SVL Command mouse actions that do not interfere with the default mapping. To remove such actions as well, each of them have to be unset explicitly in the panel.

The Manage button allows to retain and manage multiple different mouse configurations. It opens a popup which allows to name and save the current mouse configuration, as well as to load previously saved mouse settings.


Function Keys

MOE normally does not make use of the F1-F11 keyboard function keys. However, these may be programmed to invoke SVL expressions. To configure these keys enter the expression to be associated with the function key, or the Ctrl+, Alt+ or Shift+ modified function key. Press OK or Apply (the latter leaves the panel open). The function key configuration will be written to the MOE configuration file.

Note: On some operating systems, certain function key combinations have been appropriated by the system for keyboard shortcuts and cannot be overridden by the Configuration Options panel. For example, Alt+F4 is commonly used to exit programs and on GNOME, Alt+F7 is used to move the focused window.


Miscellaneous

Bubble Help Mode controls the display of bubble help in all MOE panels. To enable the display of bubble help, select on. To disable the display of bubble help, select off. Select key to display bubble help only when the Ctrl key is held down for over a second.

3D Window and Sequence Editor toggle the bubble help over atoms, bond, or ribbons in the main 3D window and residues/chains in the Sequence Editor. Additionally a modifier key can be set, which needs to be pressed while the mouse hovers over the atoms and residues, to pop up the bubble help with the object information.

Contextual Help enables, on all application panels, help icons which directly link to the corresponding manual pages or videos. MOE must be restarted for a change of this option to take effect.

SVL Output History controls the number of text lines to maintain in the scrolling area of the SVL Commands window. SVL Color Annotation enables color annotation in the SVL output history; if on, SVL commands are displayed in blue, errors in red. SVL Command History controls the number of previous commands to keep in the command line (CLI) history buffer. Previous commands are retrieved at the CLI with the up/down arrow keys.

The Warning Beep checkbox determines whether or not the operating system will produce a sound when a warning or error panel is produced within MOE. Note that on Linux and macOS systems, the sound may not be produced regardless of this setting, due to its dependence on the installed implementation of X11.

The Start In Folder text field sets the initial working directory for MOE. The "." character (dot) sets the initial working directory to the directory from which MOE was launched. The "~" character (tilde) sets the initial working directory to your home directory. In addition to the dot and tilde characters, you can enter any file path here. The CWD button will populate the text field with the path to the current working directory.

The Course Download Folder text field sets the directory where Courses and Tutorials downloaded through the MOE Help menus will be saved to. Environment variables, such as $HOME, may be included in the path.

The MOE installer will set File Associations for several file formats to the MOE applications on Microsoft Windows and Linux, so that these files can be opened directly in MOE from the Desktop or File Explorer with a simple double click. These associations can also be set or unset from within MOE with the File Associations: Setup popup: Select the file types to be associated to MOE and press Set. Unselected file types will get disassociated from MOE. On Microsoft Windows only, a new extension that is not part of the default extensions can be associated to MOE by clicking the Add New button in the File Associations: Setup popup.

Note! The selection state upon opening the popup reflects the current file association state. Enabled and disabled extensions reflect set and unset extensions, respectively, while a red extension signifies that the extension is not associated with the current version of MOE.

The MOE installer will also create a desktop shortcut to MOE on Microsoft Windows and Linux, if desired. The ability to create additional desktop shortcuts is available from within MOE by using the Desktop Shortcut: Create popup. Existing shortcuts can be updated using this panel (on Windows only).

You must press Apply or OK to put any changes into effect.


User Interface

The Window Positions buttons allow the user to:

  • Save: Saves the position of all named windows in MOE.
  • Restore: Moves all windows to their saved positions.
  • Reset: Reset stored positions to default values.

If multiple monitors are available, one can specify whether the monitor assignments of each window should be stored and restored the next time they are opened. The dropdown list defines which monitor will be used when new windows are opened:

  • Primary monitor. As defined by the operating system.
  • MOE. The monitor in which the MOE Main Window resides.
  • Mouse. The monitor in which the mouse is currently located.

On Microsoft Windows, the behavior of the MOE Input Focus (i.e. which window receives the keyboard input) can be configured. If Focus follows mouse is enabled, the window under the mouse will be automatically activated without the need of a mouse click. This mode is common on Unix systems. The option Click on title to raise windows controls whether clicking anywhere into a MOE panel or just on its title bar activates and raises it. Selecting Activate windows on show will make sure that newly opened application windows will get activated and have the input focus. For this option to have an effect Focus follows mouse must be deactivated.

MOE has enhanced support for Multi-Monitor systems and can configure the UI Zoom on a per Monitor basis:
A schematic drawing shows the available monitors and their arrangement as defined by the operating system. One can set for each monitor the scaling factor, to be applied to all MOE windows on it, in order to fine-tune the display according to the actual screen resolution. For example, on some high-definition screens it may be necessary to select a higher magnification value to make the default fonts and icons appear as expected. By default the zoom is set according to the OS recommendations.

By default the window scaling is done by MOE separately for each monitor; this is the recommended setting. If set to None all monitors will use the same zoom factor; this should be only used if all available monitors are identical. If set to OS, the Windows OS will do internal scaling, so that windows have the same physical size on all monitors, but this can produce a slight fuzzy display.

You must press Apply or OK to put any changes into effect. If the requested UI zoom changes are significant one has to confirm the changes in a popup dialog, otherwise the previous zoom settings will be restored after 15 seconds.

Draw Settings

The Draw menu is used to control the rendering quality and font of the MOE Window rendering area. The display of the system coordinate axes is also set here.

Command Description
Coordinate Axes
Displays X-Y-Z axes. The origin of the axes is drawn at the center of rotation of the system, which is, by default, the center of mass of all atoms in the system.
High Quality Rotation When off, lower quality graphics is used to display the molecular system any time the view is translated or rotated, or the zoom is increased or decreased, and similarly when atoms of the molecule are moved (using the Alt key). The use of lower quality graphics accelerates view transformations when large numbers of graphic objects and space-filling atoms are present. When on, high quality graphics rendering is used throughout.
High Quality Anti-Alias Enables Full-Scene Accumulation Anti-Aliasing, which produces high quality graphics with smooth lines and polygons. On some low-end graphics cards this option might not be supported or very slow.
Font The font used for drawing labels or for displaying messages in the MOE Window are controlled by the options here. The font size can be set to one of Small, Medium or Large. The default font is a proportional font, but can be changed to Fixed Width. The font style can also be set to Bold.

To save the current values for future MOE sessions, use the Save As Default option.

Stereo Mode

MOE supports a wide variety of stereo display methods. Select the default stereo mode, which is used when enabling stereo display:

Mode Description
Green-Magenta In this anaglyph stereo mode, the left and right images are drawn in complementary colors. The stereo effect is achieved by wearing green-magenta tinted glasses. To enhance color perception, the Anaglyph Stereo visualization theme can be used.
Besides the tinted glasses no other special display equipment is needed.
Red-Cyan Same principle as the Green-Magenta anaglyph stereo mode for use with red-cyan tinted glasses.
Quad Buffer Stereo mode using special hardware of high-end graphics cards. The video card you are using must support OpenGL quad-buffered stereo, such as e.g. NVidia Quadro graphics cards or similar workstation class graphics systems. Additionally you must also be equipped with special shutter glasses, a passive stereo projection system, or other stereo devices supported by the graphics card drivers.
Interlace In this mode, the left and right images are drawn on alternate scan lines (to support certain stereo display technologies such as Zalman and SeeReal 2D/3D monitors, as well as DLP projectors).
Left-Right Displays split stereo images of left/right eye simultaneously for freeviewing. Configuration parameters for this option can be found in the Stereo Settings panel
Over-Under Displays split stereo images in the top and bottom half of the fullscreen monitor. This mode is designed to be used in fullscreen mode and can be used with some Stereo TV systems. Configuration parameters for these options can be found in the Stereo Settings panel
Side-By-Side Displays split stereo images side by side on a fullscreen monitor. This mode is designed to be used in fullscreen mode and can be used with some Stereo TV systems.
zSpace Enables zSpace compatible stereo mode. A zSpace display and 3D glasses are required to view the images properly. This mode uses head tracking data to calculate accurate parallax movements.

To set additional stereo mode related settings such as the eye location, eye separation distance, or swapping the left and right eye, open the Stereo Settings panel.
