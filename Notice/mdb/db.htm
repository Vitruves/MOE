Introduction

Databases are useful for storing the output of any process that generates a large amount of data. Some examples in MOE are: Molecular Dynamics, Conformational Search, Docking, Pharmacophore Search, Protein Homology Modeling and PDB Search. MOE's QuaSAR and Diversity Analysis tools also use databases. MOE applications commonly use databases to provide input and return output. Should you need to write an SVL application, it is likely that you will need to use a database.

The Database Viewer displays database files. MOE database filenames typically end with the extension .mdb. Files of various database formats can be imported and exported. The Database Viewer is also used to create and modify databases within MOE.

The MOE interface provides several ways of opening the Database Viewer. Choose File | Open in the MOE Window or Sequence Editor to open an existing database. Choose File | New | Database or DBV | File | New in the Database Viewer to create a new database. The Database Viewer DBV | File | Open command can be used both to open an existing database and to create a new one.

A database file can be viewed in a single Database Viewer at any given time. Trying to view a database already open in another viewer simply brings this Database Viewer window to the foreground.

Warning! The Database Viewer is a direct view of the data on disk. A change made to data in the Database Viewer immediately updates the disk file. No backups are made, and there is no undo. Previous data can be irrevocably lost.

Warning! When MOE opens a database for write access, it locks the database to prevent concurrent writing from another MOE session which could result in data corruption. A database which is locked for read/write access cannot be opened by any other MOE session. However, in rare cases, the system file locking can fail, therefore it is still strongly recommended not to attempt to open the same database in two different sessions of MOE. Note that a database opened and locked for read-only access can be viewed (but not written to) by other MOE sessions.

The menu bar heading the Database Viewer provides direct access to most commonly used commands, applications and tools. The window includes a command line interface (CLI), which is used to enter SVL commands and also serves as a prompt area for editing commands such as DBV | Edit | New | Field. The Cancel menu in the top right corner of the window allows you to halt any tasks running in MOE.

Alternate rows in the Database Viewer can be colored to enhance visualization. The coloring is controlled by the Row Color option menu in the Configuration Options panel.

A MOE molecular database is a binary file that stores a list of data records, called entries. Rows represent the entries, where the leftmost column indicates entry numbers (i.e. row number). Each entry is composed of a list of values, called fields, that have an associated name and data type. Columns represent the fields. Each field has a name, or label, which is unique within the database. The Database Viewer provides the option of displaying numerical fields in a plot area below the data area (DBV | Footer | Plot icon).

Data cells store character or numeric data values as well as 3D conformations of molecules. When a molecular field is a single character high, MOE displays the compound name instead of the line drawing. Cells can be resized by dragging the left mouse button within a cell. Modifying the width of a cell modifies the width of the whole field without affecting other fields. Conversely, modifying the height of a cell modifies the height of every cell (i.e. every row) in the Viewer.

The font size of the data area in the Database Viewer can be changed with the zoom wheel in the right corner of the footer. The Reset button will restore the default size.

The Database Viewer Footer also includes a selection status line, displaying how many entries/fields are present/selected/visible in the database. The status text can be clicked with the mouse and corresponding menus will open with commands which operate on the corresponding items. E.g. to delete the currently selected entries use DBV | Footer | Selected Entries | Delete.

To move within a large database, use the attached scroll bars or the mouse wheel. Alternatively, one can press the PageUp and PageDown keys to move one page up or down each time; the Up and Down arrow keys can be used to move by one entry row at a time. To jump to the next or previous selected entry, press Ctrl+Down or Ctrl+Up respectively.

(Use the right button if you are using a two-button mouse.) The functionality of the mouse depends on the location of the cursor in the database.

Inverting and Clearing Field Selection

The invert and clear field selection commands operate on all fields. To invert the selection means to deselect fields that are currently selected and select those that are not selected. To clear the selection means to deselect all fields in the database. There is also a command to select all fields. The commands are duplicated in the DBV | Edit and Field Popup | Selection menus and are summarized in the following table:

Clear field selection Clear | Field Selection Clear

The top left corner button, which connects the field and entry labels, can be also used to clear the field selection or select all fields. If pressed, any field and entry selection will be cleared and if nothing is selected the press will select all fields and entries.

Moving Fields

Single fields can be repositioned in the Database Viewer, as follows:

  1. Position the cursor over the field to move.
  2. Press and hold the middle mouse button and drag the field to the new position. During the move, the field label is marked by a red rectangle.

Or

  1. Position the cursor over the field to move (the field need not be selected).
  2. Choose Field Popup | Move.
  3. The cursor changes to a crosshair, and the CLI prompts you to select the new position of the field.
  4. Click on the new position. The field is moved to the new location. Other fields are shifted to the right to make space for the new field.

Multiple fields are moved in a similar manner:

  1. Select the fields to reposition.
  2. Choose Field | Popup | Selection | Move.
  3. Click on the new position. All selected fields are inserted at the new location and the displaced fields are moved right.

Note: Fields in read-only databases cannot be moved.

Coloring Fields

The menu item DBV | Display | Color or the Field Popup | Color command opens the field coloring popup. It can be detached in a separate window by pressing the Detach button.

Database Path and filename of the database to operate on (only visible in detached window).
Color Specify whether the color is to be applied to the background or foreground of the cells of the specified field; the default is Background. For molecule fields, only background coloring is available. Foreground refers to text coloring.
of Field Target field to which the coloring rule will be applied.

Coloring scheme The color scheme can be specified in two ways, either by selecting and configuring one of the prepared Cell color options or specifying the Color Rule expression directly:
Cell The coloring of each cell depends on the value of the cell contents. Options for coloring all cells or just the empty ones are available for both numeric and character fields. If the field contains numeric data, options for coloring based on comparison to a specified value are available.

In addition, a gradient option provides the ability to calculate a color gradient over multiple values. The end- and midpoints of the color gradient and their associated colors can be specified. Values beyond the endpoints are assigned the endpoint colors. For logarithmic data the log gradient option can be used, which interpolates the color logarithmically instead of linearly between the end- and midpoints.

Another scheme is the ranges option which allows specifying adjacent color bins, such that values within the bin ranges show the same color. Values beyond the range endpoints are not colored.

The number of gradient and range color bins can be specified between 1 and 7. Plausible end- and midpoint values are chosen on the value range in the database field, but can be changed as needed. The colors will be recalculated on the fly. To reset the values to the range in the database use the button

With the inherit option the same coloring scheme as that of another field is used. This is the only color option available to molecule fields. The coloring scheme cannot be inherited from molecule fields or from other fields which already inherited a color scheme.

If the field contains character data, coloring can be based on having a pattern match or not match.
Color Rule The coloring expression which is currently in effect is displayed in the text field. If this option is enabled, the coloring expression can be edited or a new expression entered. If the expression is supported by the Cell color option controls, their values are updated, although not all color expressions can be represented by the Cell color option.

Molecule fields only support the expression {field} to inherit the colors from another field.

The bubblehelp on the text field offers a few representative examples of coloring expression syntax. When there is a syntax error, the text color turns red and its bubblehelp will show the error message.
Apply Puts the coloring rules or changes into effect.
Clear Clears the coloring rule for the foreground or background, as specified in the Color options, of the specified field. The foreground or background will revert to the default MOE foreground or background color.

If no foreground color is set, the DBV cell foreground color will be either black or white depending on the cell background color to maximize readability. This inversion of the text color on dark background can be disabled in the Configuration Options.

Undo Undo the last color change.
Custom The current color scheme/rule can be saved for future use as a favorite with the Custom button. It opens a popup to save the current coloring as a New custom rule under a descriptive name. Once saved, it can be selected from the Rules option menu and loaded or deleted with Load and Delete buttons.
Detach Detach the color popup panel from the menu into a separate window.

Renaming Fields

A field can be renamed in the Database Viewer using either DBV | Edit | Rename or Field Popup | Rename. When renaming using the Edit menu, you will be prompted to pick the field to rename. Otherwise the field from which the popup was issued will be renamed. The CLI area will display a prompter where the new name can be entered. Press Apply or Enter to put the new name into effect. The new name must be unique.

Adjusting the Precision of Numeric Data

You can adjust the precision of numeric data displayed in a given numeric field by using that field's Field Popup | Display menu controls. Note that the precision is for display purposes only and does not modify the values stored in the database. The precision display information is retained and saved with the database, unless it was opened in read-only mode.

Displaying Multiple Lines per Entry

Numeric fields can contain one or more data items, provided data is of the same type. Multiple numeric entries are entered into a single cell as space-separated values. For example, to add a value to an existing cell, double-click in the cell then in the Edit prompter insert a space and enter the new value. In a new entry (e.g. in the Add Entry panel), enter the values with spaces in between them.

The data items can be displayed all on one line or one per line using the Field Popup | Display | Multi-line command. This option also allows long character strings to wrap instead of being truncated.

Use the left mouse button to resize the data cell to see all the data lines. The multi-line attribute is retained and saved with the database, unless it was opened in read-only mode.

Deleting Fields

The deletion of multiple fields is controlled using the DBV | Edit | Delete | Selected Fields command or the Field Popup | Selection | Delete Selected command. These commands use field selection to determine which fields to delete. The Field Popup | Delete command can be used to delete individual fields. In all cases, you will be asked to confirm the operation.

Command Description
DBV | Edit | Delete | Selected Fields Delete selected fields. This command is unavailable when no fields are selected.
Field Popup | Selection | Delete Selected Delete selected fields. This command is unavailable when no fields are selected.
Field Popup | Delete Delete the current field (i.e. where the popup was issued). The field need not be selected.

Caution! These commands delete fields from the database file, and are not to be confused with the Hide Field command. Deleted fields cannot be recovered as no temporary backups are made.

If Field Popup | Select is used, Search: All Fields is set to the specific field name. Fields with character data can be searched with text queries, numeric data fields with numberical comparisons. Molecular fields allow Substructure, Descriptor, and Selection Expression searches. The following sections describe in detail how to specify the selection criteria.

Selecting Based on Numeric Data

Select in the Field Menu All Fields, Number Fields or a numeric data field. Use a Selection Criteria of the form op number, where op is one of =, <>, <, >, <=, >= or ! and number is a numeric value. If op is omitted, it is assumed to be =. ! is the not operator, equivalent to <>. A range of values can be defined by using a Selection Criteria of the form [low,high], where low and high are numeric values with low < high.

Multiple numerical comparison criteria can be specified, separated by spaces. If the default mode And is selected, all of the criteria must be satisified, otherwise in Or mode any one of them.

Note: If a numeric cell entry has multiple data items, any one of its values must satisfy the Selection Criteria for the comparison to be successful.

Selecting Based on Character Data

Select in the Field Menu All Fields, Text Fields or a character data field. Use a Selection Criteria of the form op text, where op is one of =, <> or !. If op is omitted, it is assumed to be =. ! is the not operator, equivalent to <>. If numbers or spaces are present, text must be enclosed in single-quotes.

Multiple text comparisons can be specified, separated by spaces. If the default mode And is selected, all of the criteria must be satisified, otherwise in Or mode any one of them.

The values of Partial, Wildcards and Case are used to control how the character matching occurs. This is shown in the following example using a Search Criteria of Ala:

Partial on and Case off (Default) Matches Ala, ala and ProAla.
Partial on and Case on Matches Ala and ProAla but not ala.
Partial off and Case off Matches Ala and ala but not ProAla.
Partial off and Case on Matches Ala but not ala or ProAla.

The following wildcard expressions can be used in the pattern if the Wildcards option is enabled:

* Wildcard used to match zero or more characters in the text. For example:
ZO* matches ZOPICLONE
ZORUBICIN
? Wildcard used to match any single character in the text. For example: 
alpha?rhamnopyranose
 matches alphaDrhamnopyranose
alphaLrhamnopyranose
@ Wildcard used to match any character in {a...z,A...Z,_}. For example:
@zulene matches Azulene
# Wildcard used to match [0, ..., 9]. For example:
poly(#*) matches poly(1-butene)
poly(1-decene)
[...] Wildcard used to match any character in a set. In the following example, the search matches any string that starts with an S or U:
[SU]* matches Sulfur
Ultem
[^...] Wildcard used to match any character that is not present at this position.
For example:
* [^2] matches Polycarbonate 1
Polycarbonate 3
Resin F 		but does not match Polycarbonate 2
Polyetherimide 2

When Wildcards is off, all of the wildcard characters above are taken literally. An alternative way to search for partial strings is to use the * wildcard character at the start and/or end of the search pattern. E.g. the Wildcards search *Ala* is equivalent to the search Ala with the Partial option on.

  • For more on wildcards, see findmatch.

Selecting Based on Substructure Search of Molecular Data

Select in the Search Field Menu All Fields, Substructure, a Molecular or a MOE data field. Substructures queries can be defined via SMARTS, or a Sketcher can be used. SMARTS strings must be contained in double-quotes: c1ccccc1 is invalid while "c1ccccc1" is valid. The Selection Criteria can be of the form:

op SMARTS SMARTS is a valid SMARTS string and op is one of =, <> or !. If omitted, = is assumed.
SMARTS op number SMARTS is a valid SMARTS string, op is one of =, <>, <, >, <=, >= or ! and number is a numeric value representing the number of matches.
SMARTS [low,high] SMARTS is a valid SMARTS string and low and high are numeric values with low < high.
op {sketch} op is one of =, <> or !. If omitted, = is assumed.
{sketch} op number op is one of =, <>, <, >, <=, >= or ! and number is a numeric value representing the number of matches.
{sketch}[low,high] low and high are numeric values with low < high.
op number This assumes a query has been defined via a Sketcher. op is one of =, <>, <, >, <=, >= or ! and number is a numeric value. If omitted, = is assumed.
[low,high] This assumes a query has been defined via a Sketcher. low and high are numeric values with low < high.

Selecting Based on Descriptor Search of Molecular Data

Select in the Search Field Menu Molecule Fields, a Molecular or a MOE data field. A Descriptor Filter query can be specified to search all molecule fields in a database entry. The descriptor values are calculated on the fly on the molecules. If there are multiple molecule fields in an entry only one of them has to match, but all descriptor values (if multiple descriptors are specified in the query) must fall into the specified ranges for that molecule.

Selecting Based on Atom Expression Search of Molecular Data

Select in the Search Field Menu Molecule Fields, a Molecular or a MOE data field. A Selection Expression query can be specified to search all molecule fields in a database entry. If the Selection Expression would select any atom, residue or chain of the molecules in the database entry the selection criteria for that entry is satisfied.

A Selection Expression query can not be mixed with other query types (e.g. a Descriptor Filter query) in the same search query.

Selecting Based on Multiple Fields

Select All Fields in the Search Field Menu. The Selection Criteria can be more complex, possibly including multiple fields and/or numeric or text searches. If the default mode And is selected, all of the criteria must be satisified, otherwise in Or mode any one of them. Options include:
Field op number Field is a visible data field, op is one of =, <>, <, >, <=, >= or ! and number is a numeric value.
Field [low,high] Field is a visible numeric data field and low and high are numeric values with low < high.
Field op text Field is a visible character data field, op is either = or <> and text is character string. If numbers or spaces are present, text must be enclosed in single-quotes.
Field op SMARTS Field is a visible molecule data field, op is either = or <> and SMARTS is a valid SMARTS string.
Field op {sketch} This assumes a query has been defined via a Sketcher. Field is a visible molecule data field and op is either = or <>.

These can be combined to search multiple data fields. E.g. assuming logP and Weight are fields in the database,

logP[-2,2] Weight>300 "c1ccccc1">2

will search for all entries whose logP value is between -2 and 2, Weight value is above 300 and with molecules that contain more than 2 phenyl rings. If there are Multiple molecule or MOE fields, only one needs to satisfy the substructure search for success.

Using More Than One Selection Criterion

To select entries based on more than one criterion that cannot be performed using All Fields, successive selections must be done using the Add To, Intersect, Remove, and Replace Mode menu options. The first three perform a logical operation between the existing entry selection and the results of applying the current selection criteria. Replace is equivalent to first clearing the selection and then using Add To mode.

  • When Add To is used, unselected entries satisfying the selection criteria become selected, and entries already selected remain selected. In effect, a logical "OR" operation is being performed: an entry is selected if it was already selected or it satisfied the latest selection criteria. In this mode, the set of selected entries is either enlarged or remains the same.
  • When Intersect is used, previously selected entries will become deselected if they fail to match the selection criteria, and unselected entries remain unselected. In effect, a logical "AND" operation is being performed: an entry is selected only if it was already selected and it satisfied the latest selection criteria. In this mode, only previously selected entries are considered for selection; the set of selected entries is either diminished or remains the same.
  • When Remove is used, previously selected entries will become deselected if they successfully match the selection criteria, and unselected entries remain unselected. In effect, a logical "AND NOT" operation is being performed: an entry is selected only if it was already selected and it did not satisfy the latest selection criteria. In this mode, only previously selected entries are considered for deselection; the set of selected entries is either diminished or remains the same.
Example. Suppose you wish to select all entries with a molecule field containing "c1ccccc1" exactly once and whose pKa value is greater than 3. To do so, you can perform two separate selection operations: the first based on the substructure, the second based on the pKa value. Use the Replace mode for the substructure comparison and the Intersect mode for the pKa comparison.

Selecting Using the Field Popup

The Field Popup | Select Entries commands perform entry selection based on data in a given field using special pre-defined selection criteria. Entries whose data satisfy the selection criteria are added to the current entry selection.

The selection criteria vary depending on the field data type. In all cases, Empty, First Occurence, and Not Unique are available:

  • Empty selects entries with no data in the given field.
  • First Occurence selects the entries corresponding to the first instance of each new unique field value encountered (the first entry encountered with a given data value is selected; subsequent entries with the same value are not selected). This applies only to the visible entries, hidden entries are ignored.
  • Not Unique selects entries that have duplicate values within a given field. All entries with duplicated values are selected; entries whose field value is unique across the whole field are not selected. This applies only to the visible entries, hidden entries are ignored.

Character data fields have two additional selection criteria:

  • Numeric selects entries with text which can be converted into numerical values.
  • Not Numeric select entries with text which are not numerical values.

Numeric fields also allow for selecting entries whose field data are Not Finite, i.e. NaN or Inf.

Molecular data can be subject to the following selection criteria:

Field Popup | Select Entries Select Entry if Molecular Data:
Druglike Passes Lipinski's Rule of Five [Lipinski 1997].
Leadlike Passes Oprea's lead-like test [Oprea 2000].
Reactive Contains reactive groups.
Inorganic Contains atom elements outside of the set [LP,H,C,N,O,F,P,S,Cl,As,Se,Br,I].
Bad A[k] Is an invalid R-group label.

Inverting and Clearing Entry Selection

The invert and clear entry selection commands operate on all entries. To invert the selection means to deselect entries that are currently selected and select those that are not selected. To clear the selection means to deselect all entries in the database. There is also a command to select all entries. The commands are duplicated in the DBV | Edit and Entry Popup | Selection menus and are summarized in the following table:

The top left corner button, which connects the field and entry labels, can be also used to clear the entry selection or select all entries. If pressed, any entry and field selection will be cleared and if nothing is selected the press will select all entries and fields.

Filtering Entries

Entry filtering based on numerical and textual properties can be performed using the DBV | Edit | Filter and the Field Popup | Filter commands, both of which open the Filter Database panel. The panel can also be conveniently opened with the button at the top. By default the filter panel is attached to the left of the Database Viewer, but it can also be detached into a separate window. The Filter Database panel allows the user to choose multiple filters to do a multi-variable analysis of molecular properties in a database. Each of the filters will define a search criterion on a specified molecular property stored in the database fields for the molecule entries. The filters act on numerical data and text data and then select or show only the entries in the Database Viewer which satisfy the filter criteria. More details about the filter features can be found in the Filter Database section.

Moving Entries

Single entries can be repositioned in the Database Viewer as follows:

  1. Position the cursor over the entry to move.
  2. Press and hold the middle mouse button and drag the entry to the new position. During the move, the entry label is marked by a red rectangle.

Or

  1. Position the cursor over the entry to move (the entry need not be selected).
  2. Choose Entry Popup | Move.
  3. The cursor changes to a crosshair, and the CLI prompts you to select the new position of the entry.
  4. Click on the new position. The entry is moved to the new location. Other entries are shifted down to make space for the new entry.

Multiple entries are moved in a similar manner:

  1. Select the entries to reposition.
  2. Choose Entry | Popup | Selection | Move.
  3. Click on the new position. All selected entries are inserted at the new location and the displaced entries are moved down.

Note: Entries in read-only databases cannot be moved.

Sorting Entries

The Database Viewer provides two ways of sorting entries in the database.

The Field Popup | Sort menu includes the following sort commands:

  • Ascending. Sorts all entries in the database based on the ascending order (from lowest to highest value) of the field over which the popup is opened. Text is sorted alphabetically and molecules are sorted by SMILES string.
  • Descending. Sorts all entries in the database based on the descending order (from highest to lowest value) of the field over which the popup is opened.
  • Advanced. Opens the Sort Database panel, which allows to specify extended sorting options, such as multi-criteria sorting.

MOE asks you for confirmation before sorting the database.

The DBV | Compute | Sort command opens the Sort Database panel. Sorting can be done on all fields, i.e. numeric, character, and molecule. Molecule fields are sorted based on their unique SMILES strings.

Deleting Entries

The deletion of multiple entries is controlled using the DBV | Edit | Delete | Selected Entries command or the Entry Popup | Selection | Delete Selected command. These commands use entry selection to determine which entries to delete. The Entry Popup | Delete command can be used to delete individual entries. In all cases, you will be asked to confirm the operation.

Command Description
DBV | Edit | Delete | Selected Entries Delete selected entries. This command is unavailable when no entries are selected.
Entry Popup | Selection | Delete Selected Delete selected entries. This command is unavailable when no entries are selected.
Entry Popup | Delete Delete the current entry (i.e. where the popup was issued). The entry need not be selected.

Caution! These commands delete entries from the database file, and are not to be confused with the Hide Entry command. Deleted entries cannot be recovered as no temporary backups are made.

Specifying Molecular Data

Molecular data is displayed in the Database Viewer by default as 3D line drawings, but small molecules can also be viewed as 2D depictions. For proteins the sequence can be shown as a string of single letter codes, which is the default for proteins with sequence-only data. When the field is displayed with a height of a single character, MOE displays the compound name instead of the line drawing. To see the molecule in this situation press the Shift key and hover with the mouse over the molecule cell, a bubble help with the 2D depiction will pop up.

The 3D line drawing of molecules can be rotated and zoomed in a similar fashion to that in the MOE Window:

  • Rotate by dragging the middle mouse button within the molecule cell.
  • Zoom by holding down the Ctrl button and dragging the middle mouse button within the molecule cell.

The 2D depiction of molecules can be translated and zoomed:

  • Translate by dragging the middle mouse button within the molecule cell.
  • Zoom by holding down the Ctrl button and dragging the middle mouse button within the molecule cell.

Minimizing Molecular Structures

The DBV | Compute | Molecule | Energy Minimize command opens the Energy Minimize panel with Input Data set to MDB File and the MDB filename set to the active database. This command is used to minimize the energy of molecular structures using a classical forcefield or quantum chemical method. It can also be used to calculate the energy of the current geometry (MOPAC/QM mode only) by unselecting Geometry Optimization before starting the calculation. The database must contain at least one field of molecular data for this computation to available.

Displaying Images in the Database

Image data can be stored and displayed in character fields. The following example shows ligand interaction diagram images stored and displayed in the character field Ligand Interactions.

Database fields of the char type can store arbitrary text strings, including binary data. So it is possible to store the binary data of a .png file in a cell. The Database Viewer will automatically recognize if PNG image data is stored in a cell and then show it as an image. In addition to the PNG image bitmap format, the Database Viewer also supports the internal image/grob vector image format of MOE, which is available as an option in the Export Image panel accessible in MOE applications which create images.

Images in database cells can be zoomed with Ctrl - drag middle and panned when zoomed in with drag middle.

Supported image data can be transferred into char type cells via Copy/Paste. E.g. the PNG image of the structure in MOE can be put onto the clipboard via MOE | Edit | Copy | Picture, and then retrieved in the Database cell with DBV Cell Popup | Paste. This works also from other applications as long as the application puts the image in the format image/png onto the clipboard and no additional text string is on the clipboard (when pasting into DBV character cells and multiple different data types are on the clipboard, text strings get preference on paste). In the same way one can also copy image data from a database cell onto the clipboard to be retrieved by other applications.

Tip! Images from .png files can also be imported into the database with the following SVL commands. The example fetches the PNG image data from the file image.png and writes it into the field image at the 10th entry of the database file database.mdb:

data = freadb ['image.png', 'char', INT_MAX];
ent  = db_Entries 'database.mdb';
db_Write ['database.mdb', ent(10), tag ['image', [data]]];

Database Plots

Numeric database data can be visualized using various plotting and specialized applications, available through the DBV | Compute | Analysis menu. The plotting applications are by default displayed in the plot area at the bottom of the Database Viewer. To open the plot area in the Database Viewer, press the plot icon in the footer of the Database Viewer and choose the plot type (Data Plot, Correlation Plot, Field Histogram, ...) for your data.

Note: The plot area can be resized vertically by dragging its upper boundary up and down with the left mouse button.

The plots can be detached into separate windows with the Detach button in the bottom button bar of the plot area; and consequently reattached with the Attach button. The default behavior determining whether new plots should be attached or created as separate windows can be configured in the Configuration Options. See details about the features of the various plot application in the Database Plots section.

The data in the Database Viewer and the plot visualizations are linked. If data in the database is changed the plots will update as well.

In the Data Plot and Correlation Plot the entry numbers of data points are obtained by positioning the cursor and holding it motionless on or near that point. The entry number, the plotted field values and the associated small molecule depiction will be displayed as bubble help next to the cursor. If the data point bubblehelp do not appear, check in the Configuration Options if bubble help has been turned off or requires the Ctrl key.

The functionality of the mouse inside the plot area depends on the location of the cursor. In general, the left button is used to select entries, and the middle button is used to modify the view of the plotted data. The data plot can be panned and zoomed.
