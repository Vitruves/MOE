The Database Browser

The Database Browser is used to view fields within a MOE database, one entry at a time. Usually, molecule fields are viewed individually using the default field viewers, but specially-created databases may install additional viewing modes.

To open the browser, choose DBV | File | Browse in the Database Viewer.

Database Browser, Viewer and MOE main window

In most cases, the first molecule field is the default Subject, as shown above. The 3D structure will be displayed in the MOE main window and a 2D depiction in the Database Browser window. In addition to the molecule views, one can also select to monitor the values of up to eight data fields in the browser. The browser is linked to the Database Viewer window; browsing the Database Viewer will scroll to the current entry and mark it. Selecting an entry in the Database Browser will also select it in the viewer and vice versa. The name of the currently browsed database is shown in the title of the browser window.

The Database Browser supports the following generic features in all browsing modes:

Subject Specify the Database Browser viewing mode. Select a molecule field name in the Subject option to browse molecules in 2D and 3D or Data fields to browse data fields only. The resulting databases from certain applications provide additional customized views, which are selected by default: E.g. when browsing domain motif searches, analyzing docking results, or animating molecular dynamics simulations, etc.
Entry:
 The entry number in the database may be selected by using the slider control or by entering its number in the text field located in the lower right corner of the Database Browser panel. The navigation buttons at the bottom of the panel can also be used. Additionally, the Up and Down arrow keys on the keyboard can be used to browse the entries, as well as the mouse scroll wheel while hovering over the slider.

Note: If the Database Viewer is open during browsing, the current entry row is marked by a blue rectangle and the database view is scrolled up or down, in order to keep the current entry visible within the Database Viewer window.
Click to step to the next entry. Alternatively, press the Down arrow key.
Click to step to the previous entry. Alternatively, press the Up arrow key.
Jump to the first entry in the database.
Jump to the last entry in the database.
Start or stop automatic database browsing. If on, the browser will automatically step to the next/previous entry. This is useful to: e.g. visually analyze docking poses or animate molecular dynamics simulation trajectories.
Configure automatic database browsing:
Speed Set speed of automatic browsing in steps per second.
Direction Choose direction of automatic browsing.
Loop Enable/Disable loop mode. In normal (Yes) loop mode, the browser will jump to the first entry after the last entry is reached. In Reverse mode, the browsing direction is switched when the first or last entry is reached.
Browse Only Selected Limit browsing to the entries which are currently selected in the database viewer. All unselected entries will be skipped.
Select Click to toggle selection state of the current entry in the database viewer. The current selection state of the entry in the database viewer is shown by the checkbox.
Open or raise browsed database in Database Viewer window.
Mark Click to mark/unmark entry in the specified database field. Alternatively, press Ctrl+Space to toggle the mark state. The current mark state is shown by the checkbox. The name of the database field which will be modified can be selected in the option menu next to the Mark button. Only fields of type int can be selected.
Note: Marking is only possible if the database can be written.

Browsing Molecules

When browsing molecules, the name of the molecule is shown in the text field to the right of the Subject option. If the database is opened in read-write mode, one can change the molecule name directly in the text field or by pressing the edit button next to it. Molecules can be browsed in 3D in the main MOE window and/or as a 2D depiction in the browser window. Molecule browsing is available for database fields of type molecule and moe. Several render and control options are available:

3D Enable or disable 3D display of browsed molecules in the main MOE window.
Atoms Change atom render style of the currently browsed 3D molecule.
The default render style and carbon color of ligand molecules when browsing molecule database fields can be set in the Configuration Options panel.
Ribbon Change backbone render style of the currently browsed 3D molecule.
Options Set 3D molecule center and render options. (see below)
2D Enable or disable 2D display of browsed molecules in the Database Browser window.
Use standard 2D depiction display for browsed molecules.
Use the Symbol Nomenclature for Glycans (SNFG) representation for carbohydrate molecules. All other molecules use the standard 2D depiction display.
Options Set 2D molecule display options. (see below)
Keep Ordinarily, when browsing molecules in 3D, the browsed molecule is only present in the MOE window while the Database Browser is open, and it is replaced when a different entry is selected. Pressing the Keep button will create the molecule in the MOE window as a permanent copy, so that it remains when another database entry is selected or the database browser is closed.

3D Molecule Center and Render Options

If the checkbox Browse | 3D is on, the molecule of the current entry will be shown in the MOE main window. The Browse | 3D | Options popup panel provides options to control the centering and rendering of the molecules during browsing.

Center Choose center mode for browsed molecules:
Mode Description
None Do not translate/rotate browsed molecules.
Cell Apply view settings stored in database cell.
Note: This option is only available when browsing fields of type 'moe'.
Molecule Center browsed molecule in the middle of the main window.
Reorient Center molecule and its molecules along its principal axes.
SiteView Isolate browsed ligand molecule visually, as if the RHS | SiteView button is pressed.
Graphics Control visibility of graphics objects from 'moe' field cells.
Note: This option is only available when browsing fields of type 'moe'.
Mode Description
Cell Apply visibility setting, which is stored in the database entry's cell.
Show Show all graphics objects from current entry.
Hide Hide all graphics objects from current entry.
Retain Try to retain current visibility when stepping from entry to entry.
Render Molecules stored in fields of type 'moe' retain, by default, the rendering style which was saved in the entry's database cell. If enabled, this stored rendering style is overridden with the current molecule rendering style during browsing, unifying the rendering of all browsed molecules. This option is only available when browsing fields of type 'moe'.
Ligand If enabled, only the ligand atoms in the entries are shown, otherwise all atoms are shown. This option is useful if the database contains Receptor ligand complexes, while interest lies solely in the ligand conformations.
Data Toggle display of field data of the current entry in the MOE Window. The entry number, molecule name, and the names + values of the currently selected data fields are shown in the upper left corner of the MOE main window.

2D Molecule Options

If the checkbox Browse | 2D is on, the molecule of the current entry will be depicted as a 2D molecular diagram in the Database Browser window. The Browse | 2D | Options popup panel provides settings to control the 2D display of the molecule.

Depict Choose the layout mode of the molecule 2D depiction:
Mode Description
No Do not re-depict the molecules. Use the original X,Y coordinates of the entry.
Maybe Checks if the entry coordinates are already a 2D depiction by checking the Z coordinates. If the molecule coordinates are flat, it is assumed that the entry already has 2D coordinates and they are used for the 2D diagram. All other entries are re-depicted.
Yes Re-depict all entry molecules, regardless of their current coordinates.
Match3D Depict entry to match the current 3D view of the molecule.
Font Choose relative font size of element labels.
Color If enabled, color the hetero atoms by element type.
Ligand Force depiction and display of the entry's ligand atoms only; receptor and solvent atoms are not shown. If this is option is disabled, an attempt is made to depict all atoms, but if there are too many atoms, still only the ligand atoms will be depicted.
Chirality Toggle display of atom stereochemistry in the 2D depiction. The stereochemistry of chiral atoms or atoms with chiral constraints is shown by wedge and hash bonds.
Mode Description
None Do not show any stereochemistry in the molecule's 2D diagram.
Constraints Show stereochemistry according to chiral constraints defined on the atoms.
Geometry Show stereochemistry according to actual 3D geometry of the molecule's atoms.
Both Show stereochemistry according to actual 3D geometry and chiral constraints of the molecule's atoms.
Labels Toggle display of R/S labels on chiral atoms. The label colors are used to indicate presence of a stereochemistry constraint, and whether the constraint and the actual 3D geometry match.
Color Description
Green The molecule's 3D geometry indicates a chiral atom center, but no stereo constraint is set on that atom.
Blue The stereochemistry constraint matches the atoms chirality from the 3D geometry.
Red The atom constraint does not match atom's chirality indicated by the current 3D geometry of the molecule.

Browsing Packed Conformations

When browsing molecule fields that contain packed conformations, the Conformation slider becomes available to browse through the conformations of each entry. The Energy of each conformation is shown next to the conformation slider, if the energy value is part of the molecule conformation information.

When browsing conformations, the navigation button at the bottom of the panel also navigates through the molecule conformations, jumping to the next molecule entry when the last conformation of an entry is reached. The alternative arrow key navigation (Up and Down keys) will also apply to the conformations. The button switches the button navigation between conformation and entry browsing. Note that the Ctrl+Up and Ctrl+Down keys will always browse the entries, regardless of the conformation navigation mode.

Browsing Data Fields

If the Browse | Subject | Data Fields view is selected, the browser panel will only show the names + values of data fields (types: int, byte, float, double, char) from the database, and no molecules will be shown in the MOE main window or the Database Browser window.

Up to eight data fields at a time can be set up to be monitored during browsing of the database. The names of the data fields are selected in the data option menus. The current entry's value is then shown right next to it.

When browsing conformations, if a browsed data field entry contains a vector of numerical values with the same length as the number of conformations for that entry, the numerical values will be browsed one at a time, with only the value associated with the current conformation being displayed.

If the database is opened in read-write mode, the data value can be modified directly from the browser's text field or through the edit button next to it. Multi-line text can only be modified through the edit button popup. Note! The data value change is permanent and stored immediately in the database.

The data field monitoring is also available when browsing molecules. The names + values can be also shown directly in the MOE main window when browsing 3D structures by checking the option Browse | 3D | Options | Data

Docking/Ligand-Pocket Browser

Output databases created by the Docking application support a special viewing mode in the Database Browser. In such databases, the Database Browser opens, by default, with Subject set to mol + receptor if there is a receptor column in the output database (i.e. there was a refinement stage and the receptor was allowed to move). If the receptor was the same for all poses and stored in the database environment, it is referenced by the value of Rec_index and this is indicated by showing it in the Subject with parentheses. In that case it is not displayed by default, but can be restored in MOE using the Keep Receptor button as described below. In the Docking viewing mode, since the browsed system can consist of multiple chains it is shown in a separate Tag with Browser appended to the original name.

In addition to the ligand-pocket specific options, the generic features of the Database Browser are still available. It is also possible to browse the molecules similar to non-docking databases by setting Subject to the specific molecule field. The ligand-pocket specific view adds a 2D diagram of the docked ligand and four text fields to display the docking score and three configurable data field values for each entry.

This ligand-pocket browsing mode is also available to databases created by Scaffold Replacement, MedChem Transformations or BREED.

In Ligand-Pocket mode, the panel appears as above and the additional information and options are:

Pocket Options
Enable If enabled, the receptor residues in the pocket will be displayed. This option is on by default if the receptor is taken from a field in the database, but off if the receptor is the same for all poses.
Keep Residues Visible Pocket residues may differ from pose to pose, and if this option is enabled as the database is browsed from entry to entry a residue will not disappear once it has appeared. Uncheck this option to display only the pocket residues of the current entry.
Cutoff The pocket comprises residues within the Cutoff distance from the ligand.
Pocket Carbon The color of the residue carbon atoms displayed in the pocket can be customized with the color selection.
Ligand The display style of the ligand carbon atoms can be customized with the rendering selection.
The default render style and carbon color of browsed ligand molecules can be set in the Configuration Options panel
Ligand Carbon The color of the ligand carbon atoms can be customized with the color selection.
Receptor PTM Residues If on, PTM and other small molecules in the receptor field are deemed to be part of the receptor.
2D Ligand Toggle on/off 2D diagram display of ligand and set 2D diagram render options.
Keep Receptor/Ligand Retain the currently browsed Receptor or Ligand molecule in the MOE Window.

Tip! If you like to use the ligand-pocket browsing mode on MOE databases generated by other applications, you need to set the field Environment dbvbrowse_Dock of the ligand and the receptor fields to ligand and receptor respectively. For example, if the database is named mdb and the fields corresponding to the ligands and receptors are named lig and protein, issue the following commands:

svl> db_SetFieldEnv ['mdb', 'lig', 'dbvbrowse_Dock', 'ligand']
svl> db_SetFieldEnv ['mdb', 'protein', 'dbvbrowse_Dock', 'receptor']

Then, when you open the Database Browser, you will be able to browse the ligand and the receptor columns simultaneously. If there is a score column, setting its field Environment dbvbrowse_Dock to score will cause the score to be displayed automatically on the browser panel.

Domain Motif Search Browser

Output databases created by the Domain Motif Search application support a special viewing mode in the Database Browser. In such databases, the Database Browser opens, by default, with Subject set to MotifSearch. The basic mode features of the Database Browser are still available, as appropriate, when Subject is changed to some other field. Note that the first entry in the database is always the query itself.

In MotifSearch mode, the panel appears as follows:

The additional information and options are:

Query The name of the query which generated the database, followed by its code in parentheses.
Hit The name of the current hit being browsed, followed by its code in parentheses.
Domains A comma-separated list of domains from PFAM. Only domains which overlap the aligned portions of the hit are displayed. The E-values shown in parentheses indicate the confidence in the domain assignment.
Score The score of the current hit being browsed. Higher scores are more similar to the query. For details, see the Domain Motif Search documentation.
Alignment Details of the alignment between the query and the current hit. Residues aligned out of the total number in the query are given, along with percent identity and RMSD. The alignment is based mostly on structural superposition rather than sequence, so a large fraction of aligned residues does not necessarily imply a good sequence alignment.
Color By A shortcut to toggle ribbon coloring in the MOE Window. Chain colors ribbons based on the chain to which they belong, while Segment colors ribbons such that aligned gapless segments receive the same color. In the former mode, colors of the query and current hit can be specified with the color selector boxes.
Show Vectors If on, shows 3D arrows for the secondary structures in the MOE Window. These may help visualize the similarity between query and target.

In addition to the panel, when browsing the database, the current record as well as the query will be shown in the MOE Window. They will be superposed based on the secondary structure vectors used during the search. Additionally, the Sequence Editor will display the alignment of the current record to the query, with like colors indicating gapless aligned segments. Note that because motif alignment is not always sequential, the Sequence Editor may be unable to display a meaningful sequence alignment, but where alignment is linear in sequence space, the alignment displayed will be accurate.

Animating Molecular Dynamics

Databases created by the Molecular Dynamics application support a special dynamics animation viewing mode in the Database Browser. In such databases, the Database Browser opens, by default, with Subject set to Dynamics. The generic features of the Database Browser are still available, as appropriate, when Subject is changed to some other field.

In Dynamics mode the panel appears as above. The text field right next to the Subject selection shows the time index. Additional data fields can be specified to show their values in the browser window while browsing the dynamics trajectory. The MOE main window shows the 3D structure of each dynamics simulation step and the values of t, T, P, V, and U are displayed in the upper left corner as well. To start the trajectory playback, push the Automatic Browse button. The playback speed and browse direction can be adjusted in the automatic browsing configuration popup.

Alternatively molecular dynamics trajectories can also be visualized with the Dynamics Player.
