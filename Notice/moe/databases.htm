Databases Distributed with MOE

A number of useful databases are distributed with MOE. Some are designed for use with specific MOE applications while others are for more general use.

Small Molecule Databases

Leadlike Conformer Database. The MOE Conformation Database contains 3D conformations in MOE Molecular Database (.mdb) format for approximately 650,000 (Oprea) lead-like chemical structures. These are ready to be used for pharmacophore searching, fingerprint analysis, and so on. The structures were obtained from chemical suppliers' public catalogs, including the following:

The conformations were generated using Conformational Search with the default forcefield in MOE. A maximum of 500 conformers were generated for each molecule, resulting in a limit of roughly 4 kcal/mol strain energy. The results were filtered using a heavy-atom RMSD tolerance of 1.0Å to create the final set of conformers. These databases (leadlike_conf_###.mdb) can be obtained from the MOE Auxiliary Databases download page.

RECAP Fragments. $MOE/opt/recaplib.mdb is the output of RECAP analysis on a collection of ~750,000 diverse lead- and drug-like chemical structures from vendor catalogs and medicinal chemistry journals. It can be used with the RECAP Panel.

Protein Databases

Non-redundant PDB. $MOE/project/pdb.mdb is a database of non-redundant domains derived from the PDB. Domains were clustered such that all pairs of domains within a cluster had 90% pairwise sequence identity as well as similar structure. One representative was chosen from each cluster based on resolution, R free, and number of missing residues. Nearby ligands and solvent atoms are included in each entry. This database may be searched with the MOE-PDB page of the Open panel.

Protein Families. The non-redundant set of PDB chains is further clustered structurally to form families. These structure-based sequence alignments are stored in the $MOE/project/pdb.fam.gz file in (compressed) plain text. Small families are enhanced with additional sequences from UniRef 50. This file can be used during Homology Searching to identify possible templates to a query.

Antibody Modeling Databases. The central building block for knowledge-based antibody homology modeling is the antibody MOE Project database, which constitutes the information pool of the antibody homology modeling procedures. The current antibody structure collection (MOE 2018) is composed of over 4200 VL:VH complexes, as well as several hundred monomers, stored in $MOE/project/Antibody.mdb.

For more information, see the MOE Antibody Database section.

Kinase Modeling Databases. The MOE kinase database is part of the MOE Project databases, and is found at $MOE/project/Protein Kinase.mdb. After their identification, kinase chains are further classified based on their sequence identity within the kinase core. If this is at least 45% within one of the classes in the reference file $MOE/lib/kin_ref.fam.gz, that class is assigned; otherwise a check is conducted to verify if the sequence is sufficiently close to one of the known pseudo kinases in the pseudo reference file ($MOE/lib/kin_refpseudo.fam.gz). If the sequence identity is higher than 35%, the sequence group is labeled as Pseudo-Kinase. Sequences that are positive with respect to kinase pattern detection (G-Loop, DFG and HRD) but lack sufficient homology with the reference sequences are not given any classification for a kinase group, family, or subfamily. This might apply for distant species like bacteria, parasites, nematodes etc., since the reference collection is biased towards the detection of human kinases. In addition to classifications based on sequence, the conformational states of distinct topologies such as the DFG segment or alpha-C helix are also stored in the database.

For more information, see the MOE Kinase Database section.

GPCR Structure Database. The G protein-coupled receptors (GPCRs) comprise the largest family of integral membrane proteins and one of the most important classes of drug targets. In an effort to assist in 3D structure modeling and function annotation, the GPCR Class A structure repository is provided with the other MOE Project databases in $MOE/project/GPCR Class A.mdb.

For more information, see the MOE GPCR Database section.

MOE Project Databases. A number of protein family databases — including antibodies, kinases and GPCRs — are provided in the $MOE/project directory. These databases have been created using the MOE Project framework, and are all very similar in construction and appearance. Each is superposed into a common frame of reference based on the core region of the reference .moe.gz files, which are also found in $MOE/project. The databases can all be used with the Project Search panel as the repository for all relevant information used in this application.

For more information, see the MOE Project Database section.

Rotamer Library. The Rotamer Library in MOE is a collection of all-atom sidechain conformations created using LowModeMD. These conformations differ from traditional rotamer libraries in that bond length and angle variations are permitted. The intent is to provide adequate coverage of the conformational space available to a sidechain and not just encode dihedral preferences. The rotamers are located in $MOE/opt/rotamers.mdb in a packed conformation field associated with each sidechain.
