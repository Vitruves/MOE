Opening Files

The MOE open file system is used to perform a variety of "open" operations on files of different formats. File formats are recognized automatically, but forcing a particular format interpretation is possible if necessary. Where additional options are available, a secondary panel will open. For example, when molecular data from a PDB file are being loaded into MOE, the Load PDB File panel will be raised, or when an SKC file (sketch file) is opened, the Sketch View panel is raised.

In MOE, the notion of opening a file has been extended beyond such simple operations as opening a molecule file into the system or opening a database file into a database viewer. MOE's system for opening files allows you to browse files, to operate on multiple files, and to select from a variety of operations. For example, you can choose to either load a molecule into MOE or add it to a database file. The Open system is not limited to files containing molecular data. Opening an SVL file for example, allows you to load or run the SVL module.

To open a file, choose File | Open in either the MOE Window or the Sequence Editor, or use MOE | RHS | Open.

Filters

The panel is made up of different pages, for opening files from folders, or from obtaining structure files from PSILO or the PDB. Navigation controls are found at the top of the panel.

The first page of the Open panel presents a list of files from which you can select one or more to open, and a list of operations, common to all selected files, from which you can choose the operation to perform. For information on browsing files and directories, see the documentation on the File Selection Box.

Choose All Files to filter the list by file extensions of common types. In addition to the given file extensions, arbitrary file patterns can be entered in the filter text field.

Tip  File patterns are recognized only in the filter text field. File patterns will not be recognized if entered in the file text field. Items in the file text field must match an existing file to allow for any open operation to proceed.


To open files, select them in the list, or type their names into the file text field. If a single file is selected, its type will be shown in the Open As field, and the default action for that type will be shown in the Action field. If MOE does not correctly identify a file's type, you can enforce a file type by choosing from the Open As option list. Similarly, you can choose a non-default operation from the Action list.

If multiple files are selected, they will be shown in the text field as a comma-separated list. If all the selected files are the same type, that type will be shown in Open As option list, otherwise the type will be blank. The Action option list will initially show the default operation for the current type. If more than one type of file is selected, the Action option list will be populated with the operations (if any) common to all the selected files.

If the selected operation is Open Molecule, then the Close System checkbox will be available. If enabled, the current molecular system will be closed, and the new molecular data will be centered in the MOE Window.

To perform the currently chosen action on the selected files, press OK or Apply. Double-clicking in the list is the same as selecting a single file and pressing OK, in which case the default action associated with the selected file will be performed. Press Edit to load all the selected files into individual SVL Text Editors. When either Apply or Edit is pressed, the panel remains open.

Downloading PDB Files from PSILO

The PSILO page of the Open panel is a direct link between MOE and a PSILO server.

When selected, the PSILO page immediately checks for an index file in the DOWNLOADS directory and uses its contents when reporting search results in the Open panel. This index file will be created if necessary and automatically updated using web services.

Note: Downloading files from PSILO may require configuring MOE to work with a proxy server. Depending on the server, a dialog may pop up requesting a username and password for the proxy server to continue. Please read instructions on Proxy Settings if there are problems opening structures.

The version of PSILO being run on the PSILO server and the server's URL are indicated in the titlebar of the Open panel.

Querying PSILO

The PSILO query widget is found at the top of the panel. To perform a search:

  1. Choose which field or fields are to be searched (default is All Text Fields). Depending on the field, different types of searches (keywords, begins with, etc.) may be available. The data source for the query, e.g. MOE, Sketch or Manual, might also be specifiable. Note: The list of possible search fields is dynamically obtained from the PSILO server and may vary depending on the server.
  2. Enter or choose a search term. The exact format varies, depending on the chosen search field. With some fields, such as All Text Fields or GO Annotation, after a few letters have been typed, possible autocomplete options become available. Press the (textfield shortcut) button to access these options. If the Too many matches message is displayed, type a few more letters and try again. Other fields, such as Space Group, produce a drop-down list of search terms from which to choose. Still others, such as, Ligand | SMILES search, allow choosing MOE (to use selected atoms in the MOE Window as the query) or Sketch (to define a query with a molecular sketcher accessible through the button). Most fields support a Manual entry mode which allows, for example, to enter a SMILES string directly.
  3. Press the Search button at top right to execute the search.

3D Interaction Queries

3D interaction queries can be specified from the MOE Window if the query data source is set to MOE from the drop-down, and the (restraint) button to build distance, angle and/or dihedral restraints is pressed. The query is specified by selecting all restrained atoms and any other relevant atoms. Unselected atoms will be ignored. Each molecular fragment is defined to be either ligand or receptor, as indicated in the MOE System Manager. In general, there should be one ligand and one receptor fragment in each Interaction query. Solvent atoms are considered 'ligand' for this purpose. Atoms can be explicitly designated as ligand atoms by selecting them and then choosing MOE | RHS | Ligand | Choose Ligand. Note that any query fragment which contains a mixture of ligand and receptor atoms is considered to be neither.

If an interaction search involves a ligand and a receptor fragment, upon completion of the search, a button will appear in the bottom right of the panel to open an MDB file containing clustered ligands along with their environment. The ligands in each hit are first grouped by protomer, and then each protomer is clustered with a very tight RMSD cutoff. This results in a number of clusters of ligands with very similar conformations. Each row of the MDB represents one cluster and includes the superposed ligand fragments, as well as the interacting receptor atoms. This allows one to see if there are any patterns to the distribution of receptor atoms in 3D space around the ligand, possibly dependent on ligand conformation. When clustering the ligands, possible symmetries are taken into account.

Ongoing Search

After launching a search, a progress bar replaces the query at the top of the window. While the search is ongoing, the Search button is replaced by an button. Pressing this button will interrupt the current search. A confirmation dialog will be displayed, and upon confirmation, the current search is canceled and the panel will display the previous state.

Completed Search

When the search is complete, the number of hits returned is reported at the bottom right, and the hitlist is displayed. Hovering the mouse over any of the hitlist entries will show a 2D depiction of the primary ligand for that structure, if any. Select hits from the hitlist by clicking on them in the list. The total number of selected hits is reported next to the total number of hits. Buttons below the hitlist give access to additional operations:

  • Open the Favorites prompter: Enter a unique name for the query, then press Save to save the query as a favorite on the PSILO server. Use the button to clear the Query Name field, and to cancel without saving. A previously defined favorite can be retrieved from the Query | Favorite | Favorite Query menu. Press Search to perform a search with the favorite query, or press to permanently delete the favorite from the PSILO server.
  • Open the Subset prompter. The Subset prompter is available when there are selected entries in the hitlist. These selected entries can be saved under a name for later access. Enter a unique name for the subset, then press Save to save the hitlist selection to the PSILO server. A previously defined subset can be retrieved from the Query | Favorite | Subset menu. Press Search to retrieve the entries of the selected subset, or press to permanently delete the subset from the PSILO server. Note: Subsets not available for servers running a 2019 version of PSILO or earlier.
  • Browse in PSILO... Launch PSILO in a web browser. The PSILO main search page will open with the current search, if any, already loaded and with results displayed.

Downloading Hits

To open hits from a search hitlist into MOE, select the desired hits in the hitlist and then press either Apply or OK. Both will open the PSILO Download sub-panel to download the selected hits in various formats. After successfully downloading the hits, OK will additionally close the Open panel; Apply will leave the Open panel raised.

The PSILO Download panel is used to transfer the selected search results from the PSILO server into MOE. Use Open as to select how structures are transferred:

  • PDB. Structures are downloaded as PDB files and opened in MOE. Those with primary ligands defined also fetch a ligand CIF file, which is used for dictionary correction to ensure the bond order and formal charges match those defined in PSILO. If Load Electron Density Map is on, the electron density is computed for those records which have structure factors in PSILO, and loaded as well.
  • MOE MDB. Structures are downloaded into the MOE Database (MDB) specified in the panel, which will be opened into a Database Viewer. The MDB will include additional fields for each record, such as header, compnd, source, as well as a separate field for the primary ligand, if any.
  • MOE MDB Biological Unit. Similar to MOE MDB, but here the first Biological Unit is created for each structure when building the database.
  • Ligands. Open only the ligands from each structure into MOE.
  • Sequences. Open only the polymer sequences into MOE; no atoms are created.

Superpose by provides several options for how structures will be superposed after opening into MOE, as follows. Note: Superposition is not supported for MOE MDB Biological Unit or Sequences.

  • None. Coordinates are left as-is, with no superposition performed.
  • Structure. A standard sequence alignment followed by superposition is performed. This works well when all the sequences are fairly similar.
  • Family. This option works best when all the structures belong to a single PSILO Family. The precalculated matrices which superpose each structure onto the Family Reference are used to superpose all structures into the same frame of reference. Any structures which do not belong to the largest common Family will be left at their original coordinates.
  • Ligand. Superposition is performed using a rigid version of the Flexible Alignment algorithm on the primary ligands only. This is normally limited to a maximum of 5 structures as performance can degrade appreciably above this limit.
  • Pocket. Superpose based on the Cα atoms of the pocket in PSILO's pocket database for each structure. Records which do not superpose well onto the other pockets will be left at their original coordinates. If the query was a pocket search using atoms in MOE, the loaded structures will also be superposed onto the original query.
  • 3D Query. This option is only available when an Interaction search is performed. Structures will be superposed based on the atoms matching the 3D query, and their correspondence to atoms in the query. If the query was From MOE, the loaded structures will also be superposed onto the original query.

Note for PSILO 2017 users: The 2020.09 Open panel is no longer compatible with PSILO 2017. An upgrade to PSILO 2020 is highly recommended.

Downloading PDB Files from the RCSB

The RCSB page of the Open panel is a direct link between MOE and the RCSB Protein Data Bank website.

When selected, the RCSB page immediately checks for an index file in the DOWNLOADS directory and uses its contents to initialize the Open panel file list. This index file will be created if necessary and automatically updated using web services. To download files, select their entries in the list or type their codes into the Codes text field, then press OK or Apply. Double-clicking in the list is the same as selecting a single entry and pressing OK. The RCSB Chemical Component dictionary CIF files for ligands in each structure will be downloaded automatically when available, and used to correct bond orders and/or charges in the ligands. In doing so, the local 3D environment of the ligand is taken into account, so a different tautomer or charge state may result compared to that in the dictionary entry. Note that files can be downloaded even if the index file is not available, or if its code is not in the current index file. There is no need to wait for the index to finish updating before downloading files this way.

mmCIF files are downloaded by default, however this can be changed with the Load drop-down to PDB format if desired. For structures not available in PDB format, mmCIF will be fetched regardless of this setting. If the Electron Density Map box is checked, the structure factors for the PDB file will be downloaded as well, and used to calculate the Electron Density maps. These can then be toggled on and off using the Surfaces and Maps application.

Note:   Downloading files from the RCSB may require configuring MOE to work with a proxy server. Depending on the server, a dialog may pop up requesting a username and password for the proxy server to continue. Please read instructions on Proxy Settings if there are problems using the download tool.

The index list displays the code, (accession) date, resolution (blank if NMR) and the entry name (title). The Codes and Filter fields can be used to search for specific database entries. The Codes field restricts the visible entries to those that match at least one code or pattern entered in the text field. The Filter field searches the index file for words, word fragments or quoted phrases. If "Or" with Filter is on, filter matches will be added to the list of visible entries; otherwise the Filter field will be used to further restrict which entries are displayed in the list.

The Codes and Filter text fields both admit the use of wildcards:

* matches 0 or more of any character
? matches any single character
@ matches any single character in [a..z, A..Z, _ ]
# matches any single digit [0..9]
[...] matches any character in the set; ranges (using dash (-) symbol) are allowed
[^...] matches any character not in the set

Note that an implicit * is placed at the beginning and end of each word. For more on wildcards, see findmatch.

Browsing the MOE Protein Database

The PDB page is used to load molecules from MOE's internal protein database. Note that this database consists of individual protein domains however, not complete structures, so it may be of limited use for this purpose. Alternatively, the PDB page can be used to browse any molecular database file with a code (or chain name) field and a molecule field.

When selected, the PDB page immediately loads index data from the specified MDB file (use Browse to select a different file). To load molecules into MOE, select their entries in the list and press OK or Apply. Double-clicking in the list is the same as selecting a single entry and pressing OK.

The index list displays the code, (accession) date, resolution (blank if NMR) and the entry name (title). The Codes and Filter fields can be used to search for specific database entries. The Codes field restricts the visible entries to those that match each at least one code or code pattern entered in the text field. The Filter field searches the index file for words, word fragments or quoted phrases. If "Or" with Filter is on, filter matches will be added to visible list; otherwise the visible list will be further restricted to entries matching both fields.

The Codes and Filter text fields both admit the use of wildcards:

* matches 0 or more of any character
? matches any single character
@ matches any single character in [a..z, A..Z, _ ]
# matches any single digit [0..9]
[...] matches any character in the set; ranges (using dash (-) symbol) are allowed
[^...] matches any character not in the set

Note that an implicit * is placed at the beginning and end of each word. For more on wildcards, see findmatch.

If Overwrite is on, a new database will be created, otherwise the selected entries will be appended to the target file.
