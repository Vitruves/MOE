Introduction

The Sequence Editor serves as the primary launch point for MOE's protein modeling and protein structure analysis commands. It can also be used to organize and manage residues, chains, and tag groups. To open the Sequence Editor from the MOE Window, select Sequence Editor from the Window menu. Alternatively, press the SEQ button in the upper right corner of the MOE Window.

In MOE, a molecular system consists of one or more chains, possibly organized by chain tag into tag groups. Chains can be further organized by subunit, such that corresponding chains in each tag group which are assigned to the same subunit can be aligned and superposed together. Each chain consists of zero or more residues, and each residue consists of zero or more atoms. The Sequence Editor can be used to organize and manage residues, chains, tag groups, and subunits. Note that chains and tag groups can also be managed using the System Manager.

The Sequence Editor and the MOE Window work in concert to present complementary views of the molecular system. The MOE Window enables visualization and manipulation of the system at the atomic level, whereas the Sequence Editor allows for visualization and modification of molecules at the chain and residue levels. Special annotation operations, accessible from the Sequence Editor, act at both the atom and residue levels, visually linking the atom and residue views. Atom, residue, and chain selection are by default synchronized so that selecting a residue in the Sequence Editor will automatically cause the atoms of the residue to also be selected.

The Sequence Editor displays the current molecular system as a residue schematic, where each chain is displayed as a list of residues. For instance, the figure below shows a residue schematic of $MOE/sample/mol/serine_prot.moe, a collection of serine proteases.

Chain numbering is reflected in the order in which chains are displayed in the Sequence Editor. Chain numbers are shown in the column labeled Chain and are always displayed in ascending order. Consecutive chains having the same chain tag are shown associated with a common tag block. If more than one subunit is present in the system the subunits are also indicated in the chain column as greyed-out numbers. The chain label always displays the chain number, but can also include the chain name and header. Control of the contents of chain label can be found in the Chain popup menu (right-click on Chain).

Residues composing the chains are drawn with either 1- or 3-letter residue codes, depending on the selected display mode. The horizontal alignment ruler in the Sequence Editor indicates the alignment position of residues, and can be made to wrap to fit the current Sequence Editor window size. A given alignment position in the residue schematic is referred to as a residue column. Residue UID numbers and insertion codes can be added to the display, along with a variety of other annotations.

To close the Sequence Editor, choose SE | File | Quit. The Cancel menu, in the top right corner of the Sequence Editor window, allows you to cancel tasks running in MOE. Note that the MOE Window and Database Viewer also provide Cancel menus. Next to the Cancel menu, the SVL and MOE buttons raise the SVL Commands window and MOE Window, respectively.

Components of the Sequence Editor

The Sequence Editor comprises a menu bar, SVL Command Line, residue display area, and footer.

The Sequence Editor also has horizontal and vertical sliders for scrolling the residue display. These sliders each provide a popup menu for moving to the beginning or end positions of the slider, and for restoring the previous (most recent) scroll position.

Command Line

The Command Line Interface (CLI) is where SVL commands are typed and issued. While the Sequence Editor has the focus, all keystrokes are automatically directed to the CLI regardless of where the mouse is positioned within the window.

The command line area is also used as a prompt area for several MOE applications and functions. For example, the Selection Prompter occupies this area. The prompt area may also be for entering data or simply for displaying messages. In all cases, the prompt may be canceled by pressing either the red 'X' or Esc. While the Sequence Editor's CLI is occupied by a prompt, the CLIs in other windows will function as usual.

For more information on the Command Line Interface see the SVL Commands window.

Alignment/Superposition

The Protein Align/Superpose panel can be opened by pressing the Align/Superpose button in the SE toolbar (to the left of the SVL button). This application provides all the functionality required to specify and carry out both sequence and structure alignments in a unified interface in the footer area of the Sequence Editor.

Tags and Chain Numbers

Every chain is associated with a chain tag. Chain tags are located in the column under the Tag Header (Tag). If consecutive chains share the same tag, the tag box will span the height of the chains. Pressing the Tag Header will successively select then deselect all tags and hence all chains. Available on the Tag Header and all tags is the Tag popup menu.

Chain numbers are the boxes to the left of the residue display area in the column headed by the Chain Header (Chain). A chain is selected by pressing its chain number; this also results in the associated chain tag being selected. When the Chain Header is pressed successively, all chains are alternately selected then unselected. Available on the Chain Header and on all chain numbers is the Chain popup menu.

The chain tag and number/name columns may be resized by click-dragging the edge of the column. Double-clicking the column edges will auto-resize the columns to fit the longest chain/tag label.

Chain Color

Chains are always numbered sequentially, and each chain number is colored by its chain color. When atoms are colored by chain color (MOE | Render | Atoms), this is the color that will be used. Chain color can be set using the Chain Color command in the Chain popup menu. The T option colors the chain by tag; chains of the same tag will thus be visually associated by color. # colors the chain by number; i.e. chains will have different colors, whether or not they have the same tag.

Hiding/Showing Chains

Chains recognized to contain only solvent or ligand residues can be hidden by pressing the - button next to the Tag Header. Hidden chains are denoted by a black fletch in the Chain column. Pressing + restores the hidden chains to view.

Any chain may be hidden using the Hide/Show submenu in either the chain or tag popup menu. The Show in Tag and Show All items in the submenu will reshow hidden chains. Note that the + and - buttons only apply to ligand and solvent chains. Right-clicking on the black fletch opens the hidden chain popup menu, indicating the number of hidden chains represented by the fletch and the possibility to reshow just these.

Hiding/Showing Dendrogram

The Sequence Editor dendrogram can be opened with the top left tree icon button (next to the +/- Buttons to the left of the "Tag" button). Once it is open, one can create/show dendrogram trees by selecting the appropriate mode in the tree action menu opened by the button next the tree icon. This button also identifies the current tree type. If the values determining the tree creation/clustering (UPGMA clustering) change the tree will change as well. The tree is not auto-sorting, so it can look "jumbled" after creation. The cluster values of the tree branch nodes are shown in the sequence editor footer if one hovers with the mouse over the branches. When clicking on tree branches the chain (row) pairs of the branches are selected in the SE, to select multiple separate nodes use Ctrl-click.

Residue Display Area

The residue display area comprises the column ruler, the header, and the residue drawing area, including the residues themselves and the empty space between the drawn residues. Residue annotations, such as predicted secondary structure indicators, residue UID numbers and INS codes, chain breaks, disulfides, and constraints, are all displayed in the drawing area. Consensus, similarity, and RMSD plots are displayed in the header area. The Residue popup menu is located in the drawing area, and the residue Column popup menu in the alignment ruler and header.

Split Subunit Display

The multi-subunit display mode can be toggled on and off using the Multi-Subunit Settings panel from the icon. When this mode is active, multiple chains of a tag can be laid out horizontally in a row, in multiple SE panes, and treated by Align/Superpose as one "concatenated" entity. Each pane of the Residue Display area has its own Alignment Ruler, Column plot area, and horizontal scrollbar.

Individual chains can be assigned to different subunits (e.g. all A chains to the left and all B chains to the right) by using either the SE Chain pop-up menu or the Set Up Chains button in the Align/Superpose panel.

If the multi-subunit display is deactivated and multiple subunits are present in the system (i.e. some chains have been assigned to subunits other than 1), the subunit numbers are listed right next to the chain names (in the chain buttons) slightly red and grayed out.

The multi-subunit display can show up to 8 subunit panes. The panes can be resized horizontally by dragging the separator in between them.

Residue Popup Information

Hovering the mouse over a residue or chain while the Shift key is pressed will pop up a bubble help with information about the specific residue or chain. The information includes residue name, number, type, chain name, tag, etc. For ligand residues and non-standard amino/nucleic acids a 2D diagram is also displayed. The modifier key which needs to be pressed to pop up the information can be configured in the Settings Menu.

Header

The area of the window between the alignment ruler and the residue drawing area is referred to as the header and is used to display similarity/percent identity, consensus, consensus structure, gap count, and RMSD annotations. These annotations are enabled from the footer. When the annotations are all disabled, the header area is hidden.

The plot area in the Header is selectable. By doing a rectangular selection across the peaks in the plot, one can select the associated columns in the Residue Display. During the selection, one sees the range values of the selection rectangle in the footer in purple. The value of the current column is shown in the footer in black. When hovering the y-axis value at the current mouse position is also shown in purple (the columns real value in black).

Note: If the selection goes outside of the current SE window, it automatically pans the window. The same is true for the SE wrapped view, so it is possible to select within the full wrapped view across multiple wrapped rows.

Footer

The bottom part of the Sequence Editor window is referred to as the footer and is used to control the display of residues and of residue annotations both in the residue drawing area and in the Sequence Editor header. The components of the footer are described below.

Info Area

In the leftmost part of the Sequence Editor footer, informational messages are displayed. If the mouse cursor is positioned over a chain number or tag or over a residue, the name and number of the chain or the name of the residue is given, respectively. Otherwise, the number of residues and chains selected is reported. Additional information is also shown if the mouse is over residue and header annotations (e.g. specific type of secondary structure, or per column values of RMSD plot, ...).

The commands will be applied to all selected residues, or all residues if there are none selected. If the <Alt> key modifier is pressed, the chain selection (if any) is intersected with the residue selection.

Residue Scalar Plots

Residue scalar plots display numerical residue attributes on a bar graph aligned with and displayed beneath the associated residue. Up to three scalar values may be plotted at the same time. Plotted values range from -1 to 1, with a horizontal black line marking 0. Plotted values exceeding this range are "clamped"; i.e. the values are clipped. Colors assigned to the plots can be configured in the Plot popup in the Sequence Editor footer.

The SE | Annotate | Plot menu includes three built-in measurements:

  • Structure Propensity. Probability of the sequence having a particular secondary structure type.
  • Surface Accessibility. Ratio of the solvent accessible surface areas of all amino acid residues in the system to pre-calculated values for each amino acid type in a GLY-X-GLY tri-peptide in extended conformation. The midline represents exactly 0.25 of the standard exposed surface area for each residue type. Bars above the midline (pointing up) indicate buried residues, below the midline, exposed residues.
  • Stereochemical Quality. Histograms denoting backbone geometry, Ramachandran outliers and rotamer energies. The geometry score is based on a Z-Score combination of all backbone bond length, angles and dihedral components (13 in total). The data of all categories are scaled between -1 and 1 for display. After scaling, Ramachandran outliers have values above 0.5, the allowed set, values between -0.5 and 0.5, and the core, values between -0.5 and -1, using a upper cutoff of 0.1. The rotamer energies are scaled between -0.5 and -1 if lower than 0, between -0.5 and 0.5 for energies from 0 and 3 kcal/mol, and between 0.5 and 1 for energies above 3 kcal/mol (upper cutoff is 5 kcal).

When any of these built-in measurements is chosen, the Plot popup checkbox labels are adjusted to reflect the quantity being plotted. Toggling the checkboxes hides/displays the corresponding plot in the Sequence Editor residue drawing area.

Secondary Structure Annotation

Display the DSSP secondary structure assignments [Kabsch 1983] as color-coded annotations above the amino acid residues to which they apply.

  • Yellow arrows for β-sheets.
  • Red cylinders for α-helices.
  • Blue thin cylinders for turns.

Tip   Moving the mouse cursor over the secondary structure element indicator will show its exact type (e.g. 5-turn) in the Info Area;

Residue Number/UID Labeling

The residue numbers and insertion codes can be displayed above the corresponding residues in the Sequence Editor. Successive clicks cycle between turning off residue numbering, displaying both residue UID numbers and insertion codes, and displaying the residue number (sequential index). In multi-subunit mode the sequential indices start at one independently in each pane. The button icon changes to reflect the current display mode.

By default the current UID numbers and insertion codes are shown. Structures read in from PDB and CIF files store the original author UID/INS values in a separate space. If the residues are renumbered, the original UID/values can be shown instead of the current values with SE | Annotate | Chain Info | Author UID/INS. The original values are shown in blue, but only if different from the current UID/INS values.

Tip   For space reasons, not all residue numbers will be shown. Only the numbers of every fifth residue or a residue at a chain break will be shown. Insertion codes will be shown by default in 3-letter but not 1-letter mode. To determine the UID number of any residue move the mouse cursor over the residue in question; the UID number and INS code will appear with the residue name in the Info Area. If original author UID/INS values are stored and different from the current value, they are shown in parentheses.

Single/Three Letter Codes

Switches between 1- and 3-letter residue codes. In single-letter mode, all non-amino-acid/non-nucleic-acid residues are marked with vertical brackets. Waters are represented by W, other solvents by S, ligands by L, metals by M, anions by -, cations by +, and dummies by *.

Note. The single/three letter code setting is remembered between MOE sessions.

Wrap Mode

Switches the display between linear and wrapped modes. In wrapped mode, the horizontal extent of the residue drawing area is sized to match the width of the Sequence Editor window.

Note. The wrap mode setting is remembered between MOE sessions.

Split Subunits

Switches the display between regular mode (all chains listed in a single column and subunit indices shown as red greyed-out numbers if defined) and split mode (chains in each subunit listed in separate Residue Display panes, with as many panes as defined subunits).

The Display toggle enables/disables the multi-subunit view. A Shift-click on the button in the Sequence Editor footer will switch the subunit mode directly without opening the settings panel. The Layout of the panes can be set to be either horizontally or vertically stacked. A Ctrl-click on the footer button switches between the two layouts.

By default, the Sequence Editor shows all (and only) the subunits present in the current molecule system. The display can be explicitly set to show a dedicated set of subunits. If Select Visible Subunits Explicitly is enabled, the footer button is colored red to indicate that some subunits of the molecule system might not be visible in the Sequence Editor.

Note. The multi-subunit Display setting is remembered between MOE sessions, but not the Layout or explicit subunit visibility.

Font Size

Changes the display font size. The alignment ruler will also resize. The R button resets the font size to its default value.

Sequence Editor Popup Menus

The Sequence Editor features several popup menus with a variety of commands. Familiarity with these popups will give you a much greater control over the display of atoms in the MOE Window and of residues in the Sequence Editor. Press the mouse popup button over each of the following objects to display their respective popup menus.

The popup menus by default apply to all associated selected objects. For example, the Chain popup menu applies by default to all selected chains. When a popup is raised, the objects to which it applies are highlighted in green to provide a visual cue as to which objects will be affected. The number of affected objects appears in the popup header. Pressing Ctrl while raising the popup menu causes the menu commands to apply only to the object directly beneath the cursor at that time, independent of the current selection.

Popup Menu Location Action
Selected Tags Over any chain tag or the Tag Header. Commands apply to all chains of the selected tags. With Ctrl-click, commands apply to only the chains of the tag beneath the cursor.
Selected Chains Over any chain number/name or the Chain Header. Commands apply to all selected chains. With Ctrl-click, commands apply to only the chain beneath the cursor.
Selected Residues Anywhere within residue drawing area. Commands apply to all selected residues. With Ctrl-click, commands apply to only the residue beneath the cursor.
Residue Column In the alignment ruler. Commands apply to all selected columns. With Ctrl-click, commands apply to only the column beneath the cursor.

The Sequence Editor chain, tag, residue, and residue column popup menus all contain options to open the Atom Rendering Style, Residue Color Style, and Ribbon Rendering Style panels.

Coloring Residues

Protein Similarity Monitor

To start the Protein Similarity Monitor, choose SE | Alignment | Similarity. The Similarity Monitor calculates all-against-all sequence identity, similarity and CA-RMSD distance matrices, and displays them as either color-coded heatmaps or trees calculated using the UPGMA (average distance) clustering algorithm. The heatmaps and trees can be exported to a variety of graphics formats, and the distance matrices and alignments from which they were derived can be written to a text file. The input data for the calculations can be either an entire alignment (excluding hidden chains) or restricted to selected residues.

The pairwise matrix plots and dendrograms may be either exported into a common image format or printed by pressing the Export button, which opens the Export Image panel. The Order Chains button is used to reorder the chains in the Sequence Editor to correspond to the plot. If a matrix is displayed, the chains will renumbered according to the values in the first column – that is, they will be ordered according to their similarity to the first chain. If the plot is a dendrogram, the chains will be renumbered from top to bottom. Press Report to write the current matrix and the associated alignment into an SVL Text Editor. If the calculation is restricted to selected residues, the entire alignment still appears in the editor, but with the selection set highlighted in uppercase.

The options for the Similarity Monitor are as follows:

Metric Specifies the all-against-all distances to be calculated. The matrix values (i, j) for the percentage Identity and Similarity metrics are equal to the number of sequence matches between chains i and j, divided by the number of residues in chain i. Residues are considered identical if their single-letter code is the same (e.g. MSE and MET are "identical"). Residues are "similar" if their BLOSUM62 substitution score is greater than zero. The upper triangular values of the RMSD matrix are equal to the RMSD between all aligned α-Carbons of chains i and j; the lower triangular values are calculated using those α-Carbons in positions with no gaps in any of the chains in the alignment.
Display if Matrix is chosen, the distance matrix is presented as a color-coded heat map. If Tree is chosen, a dendrogram is drawn. In the case of sequence-based scores, the distance between two chains is taken to be the number of matches divided by the longer chain length. The RMSD distances are taken from the lower triangle of the distance matrix.
Chains Specifies which protein chains will be included in the calculations. Choose First in Tag to consider only the first chain (in Sequence Editor order) in a contiguous block of chains with same (non-empty) tag. This option makes it easy to exclude protein ligands. Choose Group by Tag to, in effect, treat contiguous blocks of chains with the same (non-empty) tag as single chains for comparison purposes. For example, if antibodies are loaded and aligned (heavy-chain to heavy chain and light-chain to light-chain), this option will implicitly catenate the chains so as to compare the entire proteins.

Note that chains hidden in the Sequence Editor are always ignored.
Monitor Selected Residues If on, the calculation will be restricted to alignment positions with at least one selected residue.
Settings These items apply to the heatmap only. Enable Values to write the numerical data into the heatmap cells. Enable Chain Names to label the heatmap rows with chain names, and enable Chain Numbers to label the heatmap columns and rows with chain numbers.

The heatmap can be rendered in color or Grayscale, and the heatmap color Legend can optionally be drawn to the right of the matrix.

Joining and Splitting Chains

To join two or more chains:

  1. Select the chains to join.
  2. Choose SE | Edit | Join Chains.

The residues of the selected chains are re-parented to the lowest numbered chain.

To split the system at one or more residues:

  1. Select the residues that will mark the beginning of the new chains.
  2. Choose SE | Edit | Split Chains.

The selected residues each become the first residue of a new chain. Each new chain appears directly below the chain from which it was split.

Tip   Chains can also be split and joined with the Sequence Editor Copy As, Paste, and Paste Fragment commands. When joining chain fragments with the SE | Edit | Paste Fragment set the Coordinates option to Original in order to preserve the current atom positions during joining.

Deleting Chains and Residues

To delete a single chain:

  1. Position the cursor over the number/name of the chain to delete.
  2. Display the Chain popup (use Ctrl if the chain is not selected) and choose Delete.

To delete two or more chains at a time:

  1. Select the chains to delete.
  2. Display the Chain popup and choose Delete. The same functionality is found in SE | Edit | Delete | Chains.

Deleting residues is accomplished similarly. To delete a single residue, you can use the Delete option from the Residue popup. To delete one or more residues, select the residues to delete, then use either the Delete option in the Residue popup, or the SE | Edit | Delete| Residues menu command. To delete all residues in a residue column, select a column of residues by clicking in the alignment ruler. Then proceed as for deleting selected residues. After the residue column is deleted, the alignment is "packed" (all residues to the right of the deleted column are moved left by one column to close the gap).

When deleting chains or residues, MOE displays the total number of chains, residues and atoms that will be deleted, and asks you to confirm or cancel the operation. You also have the option to "pack" the alignment; i.e. eliminate empty residue columns that may result from the deletion operation.

Renaming Tags and Chains

To rename chains open the Rename prompter with the SE | Edit | Rename Tag/Chain menu command. The prompter provides a text field for changing the name of all selected chains. In addition to renaming the chains, also the tag, header and group label can be set for the selected chains or tags, therefore reorganizing the chains as requested. The prompter can also be opened from the Chain and Tag popups.

Renumbering Residues and Chains

The Sequence Editor provides an easy way of modifying the UID/INS number of selected residues and chains. This operation uses a CLI prompt which is accessible both through SE | Edit | Renumber and Residue/Chain popup menu Renumber commands.

Enter/Choose the new numbering scheme to be applied to either selected residues or chains and then press Apply.
Four Renumber schemes are provided:

  • Start from a given number;
  • Set to be same as alignment position;
  • Inherit numbering from a specified chain.
  • Set to original author UID/INS values.

In addition, one can also Save the current residue UID/INS values as the author UID/INS values. Note. If two non-consecutive residues on the same chain are selected, and start from number is used, all the intervening residues will be renumbered as well.

Copying and Pasting Residues and Chains

The Sequence Editor supports copying and pasting selected residues and chains. The SE | Edit | Copy As command copies all selected objects to the clipboard. The SE | Edit | Paste command pastes the entire contents of the clipboard into its own chain.

A special paste operation, accessible both through SE | Edit | Paste Fragment and the Residue popup menu Copy/Paste commands, allows for inserting the clipboard contents at a specific location in a chain. This operation uses a CLI prompt. The insertion can be made to either replace or displace a sequence of residues. First the insertion point or region to replace is chosen in the Sequence Editor using the mouse, then specified whether the fragment should be grafted onto the target chain, and finally Paste is pressed to perform the paste action. Insertion is only possible if the clipboard contents is a contiguous sequence fragment. If multiple chains or separate residue block are on the clipboard a normal SE | Edit | Paste operation creating new chains is performed.

While in the Paste Fragment CLI prompt the mouse cursor is changed to a special shape and can be either placed between two residues to mark the spot where the insertion will take place, or mark a sequence region by left-dragging the mouse, drawing a box around the residues to be replaced.

SE | Edit | Paste Fragment
Command Action
Coordinates Specify how the residues on the clipboard should be inserted:
  • None. Insert as sequence-only residues, discarding atom information.
  • Original. Clone atom information including coordinates. The sequence fragment should be superposed upfront to fit into the designated insertion area.
  • Graft. Fit sequence fragment into chain by superposing the flanking residues of the fragment and the insertion area. After the insertion strained geometry will be relaxed by a short minimization.
Options Specify:
  • If the residues to copy should be renumbered to fit into the target sequence.
  • The number of flanking residues used for the grafting.
  • The preview mode. The incoming fragment is shown superposed onto the flanking residues before the paste operation is performed.
Paste Perform the paste operation. If the fragment does not fit into the designated chain area, the user is notified and can undo the operation.

Examples

Example: Loop Grafting

In this example, the inactive loop conformation of 1M52 will be replaced with the active loop conformation of 1IEP

  1. Load the 1IEP and 1M52 complexes into MOE.

    MOE | File | Open | select the files $MOE/sample/mol/1iep.pdb and $MOE/sample/mol/1m52.pdb then press OK.

  2. Open the Sequence Editor.

    MOE | SEQ (upper right corner)

  3. Delete unwanted chains in the system.

    For clarity, we will delete the following chains: 2,3,4,5,6 from 1IEP and 8,9,10,11,12 from 1M52. To do this select those chains in the Sequence Editor then right click on the Chain Header to bring up the Chain Popup then choose Delete

  4. Render the system.
    • MOE | RHS | Hide | All Atoms
    • MOE | Footer | Ribbon | Line
    • MOE | Footer | Ribbon | Color | Chain

    This should render the system as shown below where 1IEP is colored in orange and 1M52 in magenta.

  5. Superpose the structures.

    Press the Align/Superpose button in the Sequence Editor to open the Align/Superpose panel and the press the Superpose button. This will perform an all alpha-carbon superposition, which could be refined further, but will suffice for this exercise. The two activation loop conformations however should be evident.

  6. Copy the 1IEP loop.

    Select residues HIS 375 to LYS 400 (columns: 153-178) of 1IEP (Chain 1) then right-click on the selected residues to bring up the Selected Residues Popup.

    Next, in the Selected Residues Popup choose Copy/Paste. This will change bring up a prompter in the Sequence Editor as shown below:

  7. Paste the 1IEP loop – Preview

    Now select the corresponding residues HIS 375 to LYS 400 (columns: 153-178) in 1M52. Notice that the loop is now rendered in the MOE window. For clarity, in the Paste prompter do one of the following:

    Paste prompter | Options | Show Preview | Ribbons

    or

    Paste prompter | Options | Show Preview | Atoms

    Below are two different views of the paste preview. The image on the left shows an atomistic view while the one on the left is a loop view. The atomistic view is useful for looking at potential side chain clashes while the loop view is good for looking at the flanking residue overlap.

    Here is how to interpret the paste preview modes:

    Show Preview: Atoms Show Preview: Ribbon
    Orange atoms - original backbone atoms on destination chain Orange ribbon - original loop on destination chain
    Blue atoms - flanking residues on destination chain Blue ribbon - flanking region on destination chain
    Green atoms - replacement atoms Green ribbon - replacement loop
    Cyan atoms - flanking residues on the replacement Cyan ribbon - flanking region of the replacement loop
    Element-colored atoms - nearby sidechain atoms on destination Grey ribbon - rest of the destination chain
     
  8. Paste the 1IEP loop – Replace and Minimize

    Finally press Paste prompter | Paste

    A warning message may appear stating that there is a poor superposition of flanking regions. Ideally one should verify that the flanking regions are well superposed. If not, one may need to either extend the loop regions on either side by a few residues, increase the number of flanking residues or choose Paste prompter | Coordinates | None. In this final case, one would need to use the homology modeler to model the loop coordinates.

    If you get a warning message simply press Yes. Once the loop is grafted, there is a quick relaxation procedure that is done to remove strain in the flanking regions. However this simple method was not designed to deal with sidechain clashes.

    If you get another warning message indicating that there may be clashes, simply press Yes. Afterward one may need to perform some local minimizations or loop conformation sampling to correct the geometric issues. The choice is up to user of how this should be done.

    Here is how to interpret the view of the final result:

    Final result via Show Preview: Atoms Final result via Show Preview: Ribbon
    Blue atoms - flanking residues on destination chain Blue ribbon - flanking region on destination chain
    Green atoms - replacement atoms Green ribbon - replacement loop
