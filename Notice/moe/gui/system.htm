Overview

The System Manager provides information and control over the systems currently loaded in MOE. It is raised as a popup using MOE | RHS | System or MOE | Popup | System. Alternatively, it can be opened as a persistent window using MOE | Window | System Manager, either attached to the right side of the MOE Window or as a separate detached window. From the popup, the System Manager can also be made persistent by pressing the Attach or Detach icons in the popup footer.
The attached System Manager can be detached using the Detach icon in the footer. The separate System Manager window can be attached using the Attach icon in the footer. The attached/detached state is remembered for the next time the System Manager is opened.

Attach System Manager to the right side of the MOE window. Detach System Manager from MOE window.

After one or more clicks of items in the System Manager popup, moving the mouse outside the popup will cause it to be hidden. Press the Esc key or click outside the popup (e.g., with the middle mouse button) to hide the System Manager popup. The System Manager window stays open until closed with the Close button.

The System Manager provides quick access to:

  • Visibility of atoms and surfaces.
  • Active, inert, and selected state of atoms (independent of visibility).
  • Coloring and rendering mode of atoms and ribbons.
  • Creation of molecular and interaction quick surfaces.
  • Render style, transparency, material, and coloring of surfaces.
  • Names of chains and chain groupings (tags, groups, sets).

Additionally, the System Manager also allows one to:

  • Browse visually ligand, solvent, and receptors.
  • Reorder tag groups.
  • Reorder ligand lines within tags.
  • Merge tags and move Receptor/Ligand/Solvent lines between tags.
  • Assign new chain tags and chain groups.
  • Duplicate a ligand, receptor or tag.
  • Find and mark lines with a string search.
  • Find and mark ligands with a descriptor or SMILES search.
  • Delete surfaces, grouped atoms, and sets.

The System Manager presents a multifunctional list of the molecular objects loaded in MOE. In MOE, molecular objects are arranged in a hierarchy: the system is a sequence of chains, each of which is a sequence of residues, each of which is a sequence of atoms. Generally speaking, the chains correspond to the chains of proteins and residues to the amino acids. Small molecules are generally confined to one residue (although not required). Multiple small molecules (ligand or solvent) are often collected into a single chain (to preserve PDB chain identifiers for example). Each chain has an associated chain name and chain tag. The tag is used to group chains (such as all chains from a specific PDB file). The name is used to hold the chain letter and other name information. An additional organizational grouping is the group, which collects chains of different tags. A named set may be used to represent an arbitrary collection of chains, residues, and atoms.

The System Manager is composed of a list of lines corresponding to groups (boldfaced, colored purple), sets (boldfaced, colored orange), chain tags (boldfaced) or collections of atoms (non-boldfaced) listed beneath the group, set, or tag lines; these lines can be hidden or shown in the list (collapsed or expanded). Group lines operate on all tag lines, chains, residues, and atoms in the group. Similarly, tag lines operate on all chains, residues and atoms in the tag. Non-tag lines operate on the corresponding collection of atoms (Receptor, Ligand, etc.). Set lines operate on all atoms in the set, although the controls behave differently in some cases as outlined below. Sequence-only and empty chains, if present, are represented by one single line below the set lines.

Group and Tag Controls

Within a tag group there are three different main classes of lines in the System Manager list: Receptor, Solvent, and Ligand. If the system has dummy atoms they are grouped in additional Other lines (one for each chain with dummy atoms). In the default list view mode there is only one Receptor and one Solvent line per tag group, even if the receptor and solvent molecules are spread over multiple chains. Distinct ligand chains on the other hand are listed as multiple Ligand lines. There is also an extended list view mode, in which multiple receptor and solvent chains in a tag group are split into separate Receptor and Solvent lines, and every ligand molecule is listed in its own Ligand line. To switch between the default basic and extended list view open the System Manager Settings panel from with the icon from the footer of the System Manager:

Basic View One Receptor and one Solvent line (if any) per tag group; each ligand chain in a separate Ligand line.
Extended View Each receptor/solvent chain in a separate Receptor/Solvent line; each ligand molecule in a separate Ligand line.

If there is more than one tag group loaded in MOE, then an additional System header line is available which operates on all objects regardless of tag or group associations. The scope of the System header line can be reduced by pressing the double arrow < > in the System button of the System header line. Successive presses will cycle through different modes of operation in which the header line affects just Receptor, Solvent, Ligand, Other, or Marked lines, depending on what is present in the system.

Control Description

Expands / collapses the list to include individual classes of atoms (receptor, solvent, ligand). Collapsed lists reduce to the tag line alone but are still active (controlling all objects in the tag). Keyboard modifiers affect the behavior:
  • <Ctrl>. Collapses all other tags and expand the current tag.
To expand/collapse all group and tag lines at once, use the arrow button in the System header line.

The expansion state is preserved when the system is saved in .moe files.


Expands/collapses group lines. The expand is done in two stages: The first click will expand the group line into its tags, and the second click will expand all tags within the group. The objects within a collapsed group are still active and can be manipulated by the group line controls. Keyboard modifiers affect the behavior:
  • <Ctrl>. Collapses all other groups and expands the current group.


Toggles the selection of all atoms represented in the line. Visibility is not affected. The circle is completely filled in when all atoms associated with the line are selected, half-filled when the atoms are partially selected, and empty otherwise. Keyboard modifiers affect the behavior:
  • <Ctrl>. Deselects all other atoms in the same class (Tag, Receptor, Solvent, Ligand, Other) and select the current line.
  • <Ctrl>+<Shift>. Deselects all other atoms and select the current line.





Toggles the active/inert state of the atoms represented in the line. Visibility is not affected. When at least one such atom is active, the button shows an A on a green background; otherwise, the button shows an I. If all atoms are active the complete background is green, otherwise only a partial green box is shown behind the A. Keyboard modifiers affect the behavior:
  • <Ctrl>. Sets all other atoms in the same class to inert (Tag, Receptor, Solvent, Ligand) and activates the current line.
  • <Ctrl>+<Shift>. Sets all other atoms inert and activates the current line.
The partial inert state of tags and groups is remembered and preserved when tag/groups are hidden/shown with the named Show/Hide button (see below).

The active/inert control also indicates the fixed/tethered state of atoms represented in the line, if all atoms are fixed the control has a red border, if only some atoms are fixed the red border is dotted. Tethered atoms are indicated in light red. This fixed/tethered atom indicator can be enabled/disabled in the System Manager Settings panel.
The Show/Hide button containing a Group or Tag name, or Receptor, Solvent, Ligand, or Other chain name, controls the visibility and activity state of the atoms it represents. Group and tag lines affect all atoms in the group/tag while the other non-tag lines only affect the associated atoms. A colored (blue) background on the line indicates that at least one atom or ribbon segment is visible. A gray background on the line indicates that no atoms or ribbons are visible. Showing a line makes it active; hiding a line makes it inert (inactive). Keyboard modifiers affect the behavior:
  • <Shift>. Toggles the visibility without affecting the active/inert state.
  • <Ctrl>. Hides and Deactivates all other atoms in the same class (Tag, Receptor, Solvent, Ligand, Other) and Show+Activate the current line.
  • <Ctrl>+<Shift>. Hides and Deactivates all other atoms and Shows and Activates the current line.
  • <Alt>. Manually marks/unmarks current line. Marked lines can be manipulated with the system header line in Marked mode.
  • <Alt>+<Shift>. Manually marks a range of lines.
  • <Alt>+<Ctrl>. Manually marks current line and unmarks all other.

If the name label is too long for the button, it is truncated by default on the right side. To truncate on the left side, set the RC variable SystemManager.shorten_right to 0 in your moe-rc file.

When hovering the mouse over a Ligand line a bubble help info box with the ligand 2D structure drawing and its name is shown. Alternatively, one can use System Manager | Popup | Show 2D to show the ligand molecule associated with this line in the 2D Ligand Overlay of the MOE main window. If the 2D Overlay is not yet visible, it will be activated.

Clicking on the lower right corner of the named Show/Hide button opens a popup window to change the Group, Tag, or chain Name. In group lines, only the group name can be set for all its associated tags and chains. In a tag line, the group and tag name are settable for all its descendant chains. Non-tag lines allow additionally to set the chain name (displayed on the named Show/Hide button for ligands, and displayed for other chains when the full expansion mode is turned on).

For non-tag lines, setting a new tag or group will change the grouping: If the new tag name is the same as the tag group below or above, the line will be moved to that tag, otherwise a new tag is created. If the new group is the same as an already existing group, the line is merged into that existing group.

Note: Setting a new Group, Tag, or Name can cause new chains to be created in MOE; for example, if two ligands are in the same chain and only one of them is renamed.
Switches the System header line mode: Depending on the mode, the header line buttons operate on all atoms in the System or all Receptor, Solvent, Ligand, Other, or Marked lines (if any).

The switch is only available on the named Show/Hide button in the System header line.

The color of the button reflects the most frequent color of ribbons (right half) and carbon atoms (left half) represented by that line (heteroatoms if no carbons are present). Click to open the color panel for coloring the associated atoms. This popup panel is similar to the lower part of the atom rendering panel. If the C checkbox is on, only carbon atoms will be affected. On the top of the color panel one can select if only atom colors, only ribbon colors, or both atom and ribbon colors will be set. Keyboard modifiers affect the behavior (of the color button click in the raised color panel):
  • <Ctrl>. Sets the color of all atoms/ribbons in the same class (Group, Tag, Receptor, Solvent, Ligand, Other).
  • <Ctrl>+<Shift>. Sets the color of all atoms/ribbons in the system; this is most useful for setting Color by Element mode.
The color button in the System header line can also be used to set the color of all atoms and ribbons in the system.
The graphic of the button reflects the most frequent rendering mode of the atoms represented by the line. Click to open the render mode panel for the associated atoms. This popup panel is similar to the upper right part of the atom rendering panel. To hide and show all atoms of the current line without affecting the atom render state, push the respective buttons on the left. Keyboard modifiers affect the behavior (of the render button click in the raised render panel):
  • <Ctrl>. Sets the render mode of all atoms in the same class (Group, Tag, Receptor, Solvent, Ligand, Other).
  • <Ctrl>+<Shift>. Sets the render mode of all atoms in the system.
The render button in the System header line can also be used to set the render mode of all atoms in the system.
The graphic of the button reflects the most frequent ribbon rendering mode of the backbone represented by the line. Click to open the ribbon render mode panel for the associated backbone. This popup panel is similar to the upper left part of the ribbon rendering panel. If no ribbons are visible they are all shown when changing the ribbon render mode. To keep them hidden, enable the Set Style Only toggle. Keyboard modifiers affect the behavior (of the render button click in the raised ribbon panel):
  • <Ctrl>. Sets the ribbon render mode of all backbones in the same class (Group, Tag, Receptor, Solvent, Ligand, Other).
  • <Ctrl>+<Shift>. Sets the ribbon render mode of all backbones in the system.
The ribbon button in the System header line can also be used to set the ribbon render mode of all backbones in the system.
Create, modify, and toggle the visibility state of surfaces related to the atoms of the line. The background is colored (blue) if there is a visible associated surface and gray if all related surfaces are hidden. The button text appears insensitive (gray) if there are no surfaces associated with the line. Note that alternatively to using the Quick Surface popup, "quick" surfaces can also be created with the MOE | RHS | Surface. For general surfaces, use MOE | RHS | Surfaces and Maps, or, equivalently, the MOE | Compute | Surfaces and Maps submenu. Note that only Molecular, Interaction, Patch or Electron Density Surfaces created using the Surfaces and Maps panel or the Protein Patch Analyzer can be recognized and modified in the System Manager.

Keyboard modifiers affect the behavior of the Quick Surface popup:

  • <Shift>. Shows and hides surfaces related to the line.
  • <Ctrl>. Shows surfaces related to the line and hides other surfaces in the same class (Group, Tag, Receptor, Ligand).
  • <Ctrl>+<Shift>. Shows surfaces related to the line and hides all other surfaces.
Deletes the atoms, residues, chains and surfaces represented by the line. A confirmation window will be raised providing control over what is deleted. Keyboard modifiers affect the behavior:
  • <Shift>. Deletes without raising the confirmation panel.

Set Controls

Sets are named collections of chains, residues or atoms. The system manager shows by default sets as separate line items at the bottom of the list. Additionally, one can configure in the System Manager Settings panel to show set lines for each tag, these additionally lines will then control only the objects which belong to both the set and the tag. Set lines behave in a mostly analogous manner to group and tag lines, with some minor differences as specified below:

Control Description

Expands / collapses the list of sets. The collapsed set line controls all objects in the set. Keyboard modifiers affect the behavior:


Toggles the selection of all atoms which are in the set of that line. Keyboard modifiers affect the behavior:
  • <Ctrl>. Deselects all atoms (all other lines) and then selects all the atoms which are in the set of that line.





Toggles the active/inert state of the atoms which are in the set of that line, depending on the active/inert state of the tags in the system manager. The active/inert state of sets is remembered and preserved when tag line visibility is toggled on or off. Keyboard modifiers affect the behavior:
  • <Ctrl>+<Shift>. Sets the active/inert state of all set atoms regardless of their current tag line state.

The active/inert control also indicates the fixed state of atoms represented in the line, if all atoms are fixed the control has a red border, if only some atoms are fixed the red border is dotted. This fixed atom indicator can be enabled/disabled in the System Manager Settings panel.
The named Show/Hide button containing a Set name controls only the visibility of the atoms it represents, depending on the visibility of the tags in the system manager. The visibility of sets is remembered and preserved when tag line visibility is toggled on or off. Keyboard modifiers affect the behavior:
  • <Shift>. Toggles only ribbon visibility.
  • <Ctrl>. Toggles only atom visibility.
  • <Ctrl>+<Shift>. Shows all atoms and ribbons of that set regardless of the tag line visibility.
  • <Alt>. Manually marks current line. Marked lines can be manipulated with the system header line in Marked mode.
  • <Alt>+<Shift>. Manually marks a range of lines.
  • <Alt>+<Ctrl>. Manually marks current line and unmarks all other.
Clicking on the lower right corner of the named Show/Hide button opens a popup window to change the set name. Choosing the name of an existing set will merge the sets. An empty set name will delete the named set.

The color button works on all atoms and/or ribbons of that set. Keyboard modifiers affect the behavior:
  • <Shift>. Sets the color of only those atoms and ribbons which are either active or visible in the system manager.
Changes the atom/bond style of all atoms of that set and handles the visibility (showing/hiding) like the Show/Hide button. Keyboard modifiers affect the behavior:
  • <Shift>. Sets the rendering style of only those atoms which are either active or visible in the system manager.
Changes the ribbon style of all residues of that set and handles the visibility (showing/hiding) like the Show/Hide button. If no ribbons are visible they are all shown when changing the ribbon render style. To keep them hidden, enable the Set Style Only toggle. Keyboard modifiers affect the behavior:
  • <Shift>. Sets the rendering style of only those residues which are either active or visible in the system manager.
Deletes the set represented by the line. A confirmation window will be raised providing the option to delete just the named set definition or the set definition and all associated atoms, residues and chains.

System Manager Settings

The System Manager Settings panel is accessible through the button in the footer.

Control Description
View
  • Basic: One Receptor and one Solvent line (if any) per tag group; each ligand chain in a separate Ligand line.
  • Extended: Each receptor/solvent chain in a separate Receptor/Solvent line; each ligand molecule in a separate Ligand line.
Sets
  • Hide: Do not show set lines in the System Manager.
  • Show: Show set lines at the bottom of the System Manager.
  • Per-Tag: Show set lines at the bottom of the System Manager and after each tag the list of associated sets.
  • Show RSL Atom Sets: If on, the Receptor, Solvent, Ligand sets, which override the automatic RSL classification, are shown in the set list.
Fixed If on, display a red frame around the active/inert button to indicate a structure having fixed atoms. Tethered atoms are indicated in light red. If not all atoms are fixed or tethered, the frame is dashed.
Labels If on, the Receptor, Solvent, and Ligand line labels are shortened to R, S, and L.

Create and Modify Surfaces

With the quick surface popup one can create surfaces for atoms associated with Group, Tag, Receptor, and Ligand class System Manager lines. The default settings for the surface creation options (style, coloring, etc.) can be set in MOE | Settings | QuickPrep. If surfaces are already present, they can be modified, shown, hidden, and deleted. A popup with special options is available for electron density surfaces or patch surfaces created with the Protein Patch Analyzer.

Surface Chooses surface type to be created. Depending on the class of the current System Manager line, up to five different quick surfaces can be created: Receptor, Interaction, Protein, Protein Patches, Ligand.
Style Selects the style (Dot, Line, or Solid), material+transparency, and constant color (if Color is set to Constant) of the surface to be to be created or modified.

Note: The surface style options are available only for quick surfaces and molecular or interaction surfaces created using the Surfaces and Maps dialog.
Color Specifies the coloring method of the molecular or interaction surface: ActiveLP, Atom Color, Electrostatics, ElectrostaticsPB, Lipophilicity, Pocket, B-Factor, Carbon Color, Residue Color, Chain Color, and Constant.

Note: Only molecular or interaction surfaces can be recolored.
Create Surface Creates a surface for the atoms associated with the current System Manager line. If a quick surface is already present, it will be replaced. Additional surfaces per line can be created with the Surfaces and Maps panel.
Create
Modify		 Creates a new surface, or, if surfaces are already associated with the current System Manager line, modify an existing surface. The radio switches between the Create and Modify pages of the popup.
Surface List The surfaces associated with the current System Manager line are displayed in a list. Selecting a surface in the list allows that surface to be modified, shown, hidden, or deleted. Upon selection, the surface will flash briefly in the MOE Window for purposes of identification.
Hide
Show		 Hides or shows the selected surface.
Delete
Delete All		 Deletes the selected surface, or all surfaces associated with the current system manager line.

Patch Surface Popup
Type Selects the protein patch surface types (Hydrophobic, Positive, and Negative) to be created/shown.
Style Selects the style (Dot, Line, or Solid), material+transparency, and color of the patch surface types to be created or modified.

The vertical slider on the right side sets the transparency of all three patch surfaces types together.

Electron Density Surface Popup
Style Selects the style (Dot, Line, or Solid), material+transparency, and color(s) of the surface. For difference maps, the first color applies to positive density and the second to negative density. For other map types only the first color setting is used.
Level Adjusts the sigma level (contour) of the surface measured in standard deviations above the mean density.

Duplicate Objects

With the System Manager | Popup it is easy to duplicate ligands, receptors and whole tags: Click the right mouse button over the ligand, receptor or tag line to duplicate and choose Duplicate in the popup menu. This will then duplicate the atoms of the selected System Manager line. The new atoms will be shown, activated and automatically selected; the chain or tag name of the new atoms will be appended with (2) or a subsequent number. The original atoms of the duplicated System Manager line will be hidden and set inert. Additional duplication options can be set in the MOE Settings Panel.

Constrain Objects

The System Manager | Popup allows to constrain and unconstrain ligands, solvent, receptors and whole tags, groups or sets: Click the right mouse button over the System Manager line to constrain and choose Fix or Tether in the popup menu. This will then fix or tether all the atoms of the selected System Manager line. Conversely one can unfix and free the atoms with the options Unfix and Free. Fixed and tethered atoms will be indicated by a red or light red frame around the active/inert button in the object lines.

Browse System

With the System Manager, it is easy to analyze and compare complex systems. In docking experiments, one often has a single receptor structure loaded and desires to browse through a series of docked ligands, possibly also comparing each docked ligand to a reference ligand structure. With a <Ctrl>-click onto the named Show/Hide button of a ligand line only that ligand becomes visible and all others are hidden.

For large numbers of ligands, the footer of the System Manager has a next and previous arrow button to step through all the ligands loaded in MOE. The RSL button next to the arrow buttons opens a popup in which one can specify whether the arrow buttons step through the ligand, solvent, or receptor lines of the System Manager list. If Receptor, Solvent, and Ligand are all activated, the arrow buttons step through entire tag groups (equivalent to <Ctrl>-click on the tag lines). A <Shift>-click onto an arrow button jumps to the first or last list entry. Alternatively, one can use the Up and Down arrow keys on the keyboard to step through the system.

By default, the expansion state of the tags and groups is not changed during browsing. To automatically expand tags of browsed structures, <Ctrl>-click on the arrow buttons.

In order to keep one or more ligand lines visible during the ligand browsing, for reference comparisons, one has to make sure that the lines are visible but inactive before browsing. These lines will then not be included in the browsing and the associated structures will be kept visible. The same applies to receptor browsing.

Reorder and Organize System

It is possible to reorder the members of groups and tags in the System Manager with the mouse. In order to move an entire tag group, select the tag line with the mouse, and then press and hold the middle mouse button. A red rectangle appears around the tag line, indicating that it will be moved. To select the target of the move operation, continue to hold the middle mouse button and drag the mouse cursor to the list location where the tag should be moved to. While dragging, a red line will indicate the currently selected target. To move the tag, the destination must be between two tags. If the target is within a tag itself (receptor, solvent, ligand lines), then it is assumed that a merge of the source tag with the destination tag is requested. Once the middle mouse button is released, the move or merge is performed. In the case of a merge request, a confirmation panel will pop up.

Similarly, groups can be reordered and merged. To move an entire group, select the group line and drag it to the desired position. If the target location is within a group, a group merge action is possible, in which a confirmation dialog will open.

Receptor, solvent, or ligand lines can be moved between tags or groups using the middle mouse button in the same manner. Such a move and merge operation will also pop up a confirmation panel before the merge is performed.

Note: Similarly to setting a new Group, new Tag, or new Name, moving a single non-tag line can cause new chains to be created in MOE: for example, if two ligands are in the same chain and one of the ligands is moved.

If a tag has multiple ligand lines (or multiple receptor/solvent lines in expanded-list mode) the ligand lines can be reordered within the group by dragging them with the middle mouse as described above. Note: The order of the classes within the tag (1. Receptor, 2. Solvent, 3. Ligand, 4. Other) cannot be changed, only the order of lines within its own class.

The System Manager also displays any named sets that exist in the system. The set name can be edited in the popup raised by pressing the lower right corner. The sets lines are always sorted alphabetically.

A Group By option is available in System Manager | Popup when auto grouping is available. Auto grouping can be set by adding delimiting group names using braces { } in chain headers. Structures loaded from project family databases have already chain headers for auto grouping set.

Search System

The leftmost button in the footer is the Search button. It pops up a text search line. The entered text phrase will be searched within the chain names, groups, and tags of the System Manager list lines. Lines associated with matching structures are marked with blue text. Collapsed groups or tags with marked lines are automatically expanded. Repeated search for the same text string will show the next matching line with a temporary blue rectangle and scroll the list if needed to make the line visible.

Marked lines can be assigned to a group name by pressing the Group button. This opens a dialog box where a name for the group can be entered. After clicking OK, the grouped items will appear under the name of the group in the System Manager panel. Grouped items can be searched in the search line via group=pattern.

The search also accepts QuaSAR descriptor expressions and SMARTS strings; but only searches for these in the ligand lines.

The marked state of lines can also be changed manually by pressing <Alt>-click on named Show/Hide button. A range of lines can be marked with <Alt>+<Shift>. <Alt>+<Ctrl> will mark the current line and unmark all others. Marked lines can be manipulated together by the system header line in Marked mode.
