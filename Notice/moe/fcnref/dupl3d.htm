Duplicate3DWindow Function

Syntax

    Duplicate3DWindow [enable, monitor_id, 'stereo']

    current_options = UISetup []
    UISetup options
    options = [
        smallMonitor: flag
    ]

Description

[enabled, monitor_id, 'stereo'] = Duplicate3DWindow []
Duplicate3DWindow [enable, monitor_id, 'stereo']

The Duplicate3DWindow function makes a duplicate of the 3D view of the main window into a separate window or in fullscreen onto another monitor. The view of both windows are synchronized. This is useful for presentations. The duplicated window can be set to use a different stereo mode than the main MOE window. If no arguments are specified, the current mode parameters are returned.

enable
Flag indicating if the main 3D view should be duplicated/cloned into a separate window or fullscreen.
monitor_id
Number of monitor to open duplicate window in fullscreen mode. If 0 duplicate as resizable window which can be moved around on the desktop. The default value is 0
stereo
The stereo mode is specified by the argument token and can be one of the following. The default is 'none'.
Stereo Mode Description
'none' Disables stereo projection.
'quad-buffer' Enables quad buffer stereo video. Each eye is rendered into a separate image buffer and special hardware (e.g. NVidia 3DVision Por Kit or StereoGraphics shutter glasses) is used to rapidly alternate the display of each image. On some computer platforms, the graphics driver can be configured for passive stereo equipment (polarized glasses) by outputting the images for both eyes to separate outputs (e.g. two projectors). Other stereo devices might be supported by the graphics drivers as well.
'left-right' Enables left/right or split pair stereo. The left eye is rendered on the left half of the screen and the right eye on the right. By relaxing one's eyes ("looking through the monitor") one can overlay the two images and see stereo.
'over-under' Enables over/under stereo video. The left eye is rendered on the top half of the screen and the right eye on the bottom half. This mode is intended for fullscreen use and is supported by some Stereo TV systems. No operating system drivers are required.
'side-by-side' Enables side by side stereo video. The left eye is rendered on the left half of the screen and the right eye on the right half. This mode is intended for fullscreen use and is supported by some Stereo TV systems. No operating system drivers are required.
'interlace' Enables line-by-line interleaving of left and right eye images used by some polarized filter stereo systems (e.g. Zalman and SeeReal 2D/3D monitors). Some DLP projectors support the interlace mode with a checker pattern.
'anaglyph' Enables anaglyph stereo. Anaglyph stereo colors the left and right eye images complementary colors and superposes the two images on the screen. The 3D effect is then achieved by wearing glasses with one lens of each complementary color. MOE supports two types of colored glasses: green-magenta (green on the left eye) and red-cyan (red on the left eye). The video card you are using must support GLSL shaders.
'parallel' Enables parallel projection mode. This is not really a stereo mode but a mode in which the molecular system is rendered without perspective.
'zspace' Enables zSpace compatible stereo mode. A zSpace display and 3D glasses are required to view the images properly. This mode uses head tracking data to calculate accurate parallax movements.

Examples:

    svl> Duplicate3DWindow [1, 0, 'none'];     // open duplicate 3D window
    svl> Duplicate3DWindow [1, 2, 'anaglyph']; // duplicate 3D fullscreen on second
                                               // monitor in anaglyph stereo mode
    svl> Duplicate3DWindow 0;                  // close duplicate window

UISetup options
current_options = UISetup []

Set user interface configuration options for the MOE main window.

options is a tagged vector of attribute-value pairs:

smallMonitor
Flag indicating if the small monitor mode is enabled or not. In small monitor mode some user interface elements are shrunken or rearranged to make it possible to run the MOE user interface on monitors with a very low resolution (e.g. below 1024x768 pixels).
