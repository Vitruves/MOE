MOL2 Database Import/Export Functions

Syntax

    db_ImportMOL2 [ 'MOL2_file', 'mdb_file', 'mol_field', options ]

where options is 

[
    overwrite:      flag,
    file_field:     flag
]

    db_ExportTriposMOL2 ['mdb_file', 'MOL2_file', 'mol_field', options]

where options is 

[
    esel:   flag
]

Description

db_ImportMOL2 [ 'MOL2_file', 'mdb_file', 'mol_field', options ]

Imports molecules from a MOL2 formatted file to the specified molecule field in a MOE molecular database file.

'MOL2_file'
Name of the source MOL2 format file to import, specified as a token.
'mdb_file'
Name of the destination MOE molecular database to which the imported structures will be written, specified as a token.
'mol_field'
The name (a token) of the molecule field in the destination database where the molecular data will be written. The default is mol. If this field name currently exists, the import operation may overwrite the contents of the field. Otherwise, a new field will be created.
options
The import options are attribute-value colon-separated pairs:
overwrite:flag
If flag is zero, the imported molecules are appended to the destination database. Otherwise, molecules are added to the specified database field starting from the first entry; if the destination molecule field already exists, any molecule data already present will be overwritten. The default value is 0.
file_field:flag
If flag is non-zero, then the name of the source database file for each entry is written to the destination database file. The default is 0.

The following example illustrates how to import the molecules in file molpack.mol2 into the field mol of database molpack.mdb starting at the first entry:

db_ImportMOL2 [ 'molpack.mol2', 'molpack.mdb', 'mol', [overwrite:1] ]

db_ExportTriposMOL2 ['mdb_file', 'MOL2_file', 'mol_field', options]

Exports a MOE molecular database file to a Tripos MOL2 database file.

'mdb_file'
Name of the MOE molecular database file, specified as a token, to be exported to the Tripos MOL2 format.
'MOL2_file'
Name of the file to be created, specified as a token. If the file already exists it will be overwritten.
'mol_field'
Name of the molecular field to use for the export. If no field name is given then the first molecule field in the database will be used.
options
Output options are specified as a tagged vector:
esel
If non-zero, only entries that are selected in the Database Viewer will be exported. If no database viewer is currently displaying mdb_file then this option will be ignored.
