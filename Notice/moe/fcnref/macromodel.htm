MacroModel Database Import/Export Functions

Syntax

macromodel_ImportMDB [ 'db_file', 'mm_file', 'mfield', options ]

where options is:

    [
        overwrite:          flag,
        file_field:         flag
    ]

macromodel_ExportMDB [ 'db_file', 'mm_file', 'mfield', options ]

where options is:

    [
        esel:       flag,
        maestro:    flag
    ]

Description

macromodel_ImportMDB [ 'db_file', 'mm_file', 'mfield', options ]

Imports data from a MacroModel Database file to a MOE molecular database file.

'db_file'
Name of the destination molecular database file, specified as a token. This file must already exist for the function to work.
'mm_file'
Name of the source MacroModel file to import, specified as a token.
'mfield'
Name of the molecule field in the destination database to use for the imported molecules. A new field is created if necessary.
options
options is an attribute-value tagged vector of options:
overwrite:flag
If flag is non-zero, database entries are appended to the database. Otherwise, data is merged into the database starting from the first entry, i.e. the source data overwrites the destination, beginning at the first entry of the destination database. The default is 1.
file_field:flag
If flag is non-zero, then the name of the source database file for each entry is written to the destination database file. The default is 1.
macromodel_ExportMDB [ 'db_file', 'mm_file', 'mfield', options ]

Exports fields and entries from a molecular database to a MacroModel file (which is overwritten if it exists). If no molecule field is present in the molecular database, one is created. Only the first molecule field encountered is exported.

'db_file'
Name of the source MOE molecular database, specified as a token.
'mm_file'
Name of the destination MacroModel file, specified as a token.
'mfield'
Name of the molecule field to export. If null is given then the first molecule field found in the database will be used.
options
An attribute-value tagged vector of options:
esel:flag
If flag is non-zero, only entries that are selected in the Database Viewer will be exported. If no database viewer is currently displaying db_file then this option will be ignored.
maestro:flag
If flag is non-zero, the Maestro file format is used.
