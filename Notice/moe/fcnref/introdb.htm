MOE Database Functions

A molecular database is a binary file that holds an ordered sequence of entries. Each entry may have any number of data fields; all entries share the same set of field names for their data. Each cell in the database can store up to 2 gigabytes of information (integer, floating point, double, byte, char, molecule). Data compression techniques are used to store 3D conformations of molecules. Topological and conformational molecular data is stored using an average of 7 bytes per atom for small molecules and 8 bytes per atom for biopolymers.

A database can be viewed using the Database Viewer, which is opened using dbv_Open. This viewing tool is used to display and manipulate the data that is stored on disk. Database Viewer functions are prefixed by dbv_.

Every database is identified with a key which is obtained with db_Open, dbv_Open, dbv_OpenFile or dbv_KeyList. The key identifying a database remains valid until the database is closed. In general, database functions accept a database key as a parameter, using it to identify the database to be manipulated. Most database functions also accept the database name as the database identifier.

The entries themselves are referenced with entry keys. An entry key is an integer that uniquely identifies the entry within a particular database. The precise value of a key has no meaning and should not be interpreted in any way. Entry keys are obtained with db_Entries.

Data is read from and written to the database as either a flat vector of values (for integer, real and character data) or a molecule structure, as described in mol_Extract.

DB_FIELD_TYPES is the constant used to enumerate the possible field types allowed in a database.
