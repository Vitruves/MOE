Chain Object Functions

Syntax

    colors          = cColor        chain_keys
    colorby_tokens  = cColorBy      chain_keys
    group_tokens    = cGroup        chain_keys
    header_tokens   = cHeader       chain_keys
    name_tokens     = cName         chain_keys
    chain_indices   = cNumber       chain_keys
    residue_counts  = cResidueCount chain_keys
    colors          = cRGB          chain_keys
    flags           = cSelected     chain_keys
    subunit_numbers = cSubunit      chain_keys
    tag_tokens      = cTag          chain_keys
    tag_id_numbers  = cTagId        chain_keys
    flags           = cVisibleSE    chain_keys

    cSetColorBy  [ chain_keys, colorby_tokens ]
    cSetGroup    [ chain_keys, group_tokens ]
    cSetHeader   [ chain_keys, header_tokens ]
    cSetName     [ chain_keys, name_tokens ]
    cSetNumber   [ chain_keys, chain_indices ]
    cSetRGB      [ chain_keys, colors ]
    cSetSelected [ chain_keys, flags ]
    cSetSubunit  [ chain_keys, subunit_numbers ]
    cSetTag      [ chain_keys, tag_tokens ]
    cSetHiddenSE [ chain_keys, flags ]

Description

In MOE, molecular data is organized as a hierarchy of molecular objects or objects. A chain object can own zero or more residues. MOE stores chain information in separate internal data structures. The following functions obtain or modify basic chain properties. Chains are referenced with an object key, which is a unique number identifying the chain object.

Important! All functions described herein can be used on arbitrarily nested vectors.

The following functions can be used to obtain information from chain objects:

Function Description
cColor Returns the display color for the chain. This is the (computed) color that is actually used for the chain label in the Sequence Editor. The integer color format is 0xRRGGBB encoding red, green and blue.
cColorBy For each chain in the argument, this function returns a token describing how the chain is colored. Legal values are
  • 'rgb' - color by chain's cRGB color
  • 'c:rgb' - color by the chain's cRGB color
  • 'c:number' - color by the chain's index
cGroup Returns the chain group token for each chain in the argument. The chain group is used as additional grouping layer in the System Manager.
cHeader Returns the chain header token for each chain in the argument. The chain header is typically the HEADER line from a PDB file.
cName Returns the chain name token for each chain in the argument.
cNumber Returns the index in the list of chains of each chain specified in the argument and is equivalent in functionality to oIndex.
cResidueCount Returns the number of residues in each chain and is equivalent in functionality to oChildCount.
cRGB Returns the internal color of the chain. This is the color used when the chain's ColorBy mode is rgb or c:rgb. The internal color is an integer representing the RGB color in the format 0xRRGGBB encoding red, green and blue.
cSelected For each chain in the argument, returns a value of 1 if the chain is selected in the Sequence Editor, and zero otherwise.
cSubunit For each chain in the argument, returns the Sequence Editor subunit pane number.
cTag Returns the chain tag token for each chain in the argument. The chain tag is used to identify a group of chains.
cTagId Returns the chain tag id number for each chain in the argument. The chain tag id is related to the chain tag token and is used to split chains into groups as in the System Manager and Sequence Editor.
cVisibleSE For each chain in the argument, returns a value of 1 if the chain is visible in the Sequence Editor, and zero if hidden.

The following functions can be used to modify information in chain objects:

Function Description
cSetColorBy For each chain in the argument, this function sets the mode by which the chain is colored in the Sequence Editor. Legal values are
  • 'rgb' - color by chain's cRGB color
  • 'tempfactor' - color by chain's atoms' average temperature factor
  • 'c:rgb' - color by the chain's cRGB color
  • 'c:number' - color by the chain's index
cSetGroup Sets the chain group token for each given chain. The group token is used in the System Manager as additional grouping layer.
cSetHeader Sets the chain header token for each given chain. Typically, this represents the PDB header (for proteins).
cSetName Sets the chain name token for each given chain. For proteins in the PDB, this field usually contains the PDB code and chain letter.
cSetNumber Sets the order of chains in the system. Chain numbers start from 1. This function can be used to re-order or re-number chains.
cSetRGB Sets the internal color of the chain. This is the color used when the chain's ColorBy mode is rgb or c:rgb. The internal color is an integer representing the RGB color in the format 0xRRGGBB encoding red, green and blue.
cSetSelected Assigns the selected flag for each given chain. The selected flag controls the selection state of the chain in the Sequence Editor.
cSetSubunit Assigns the Sequence Editor subunit pane numbers of the specified chains for the split multi-subunit display.
cSetTag Sets the chain tag token for each given chain. The tag token is used to identify groups of related chains. For example, for PDB files, the tag is set to the PDB code.
cSetHiddenSE Hides a given chain in the Sequence Editor. Hiding the chain will also unselect it.

For example, suppose insulin is the only molecule loaded. We can obtain the chain keys and find out the indices of each chain loaded with:

    svl> c = Chains[];
    svl> cNumber c
    [1,2]

The following code sets the name field of chains to '91NS.A' and '9INS.B' respectively and verifies the modification:

    svl> cSetName [c, ['9INS.A','9INS.B']]
    svl> cName c
    ['9INS.A','9INS.B']

We now set the tag field of both chains to '9INS':

    svl> cSetTag [c, '9INS']
    svl> cTag c
    ['9INS','9INS']
