XYZ File Format I/O Functions

Syntax

    keys = fread_XYZ ['filename', opt]
    opt = [
        def_molname: 'name_tok',
        center: flag
    ]

    fwrite_XYZ ['filename', obj_keys, opt]
    opt = [
        center: flag,
        split: flag,
        fileSplit: flag,
        base: '/path/file_prefix',
        multipleFnames: ['filename1', 'filename2', ...]
    ]

Description

These functions provide means for reading and writing XYZ files.

keys = fread_XYZ ['filename', opt]

Reads a XYZ format file and returns chain keys.

opt provides the following options:

def_molname: 'name_tok'
This option allows for specifying a default molecule name that will use used if the XYZ comment line is empty.
center: flag
This flag controls the centering of molecules when reading an XYZ file:
  • If flag is 0, then the molecules are not centered.
  • If flag is 1, then the molecules are centered. This is the default. 
fwrite_XYZ ['filename', obj_keys, opt]

Saves the specified molecular objects to a XYZ format file. obj_keys specifies the keys of the molecular objects loaded in the MOE system that are to be saved.

opt provides the following options:

center: flag
This flag controls the centering of molecules when writing an XYZ file. If true, then the molecules are recentered before writing to the file. (default 1)
split: flag
This flag controls the separation of molecules when writing an XYZ file. If true, then each molecule will be written in a separate entry.
fileSplit: flag
This flag controls the separation of entries into different XYZ files. If true, then molecules will be written individually into separate files, with their filenames either taken from multipleFnames or generated from base and an incrementing index. Otherwise all the molecule entries will be written to a single XYZ file.
base: '/path/file_prefix'
This option allows for specifying a filename prefix for writing to multiple files when multipleFnames is not specified. If neither is specified, filename is used. An incrementing index is used as a suffix. Homonymous existing files will be overwritten.
multipleFnames: ['filename1', 'filename2', ...]
This option allows for specifying filenames for writing to multiple files. If not specified, base is used. Homonymous existing files will be overwritten.
